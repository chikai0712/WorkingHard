# 安全掃描與訪問控制策略

## 📋 概述

本文件定義了代碼安全掃描、容器鏡像掃描，以及生產環境訪問控制機制，確保維運人員無法直接接觸生產環境，避免內部資安風險。

## 🔍 安全掃描策略

### 1. 代碼安全掃描 (SAST - Static Application Security Testing)

#### 掃描工具
- **Gosec**: Go 語言安全掃描
- **Semgrep**: 通用代碼安全掃描
- **GitLab SAST**: GitLab 內建掃描

#### 掃描內容
- SQL 注入風險
- XSS 漏洞
- 敏感信息洩露
- 不安全的加密實現
- 命令注入風險
- 路徑遍歷漏洞
- 不安全的隨機數生成

### 2. 依賴掃描 (Dependency Scanning)

#### 掃描工具
- **Trivy**: 容器和依賴掃描
- **Snyk**: 依賴漏洞掃描
- **GitLab Dependency Scanning**: GitLab 內建掃描

#### 掃描內容
- 已知 CVE 漏洞
- 過時的依賴版本
- 許可證合規性
- 直接和間接依賴

### 3. 容器鏡像掃描 (Container Scanning)

#### 掃描工具
- **Trivy**: 容器鏡像掃描
- **Clair**: 容器漏洞掃描
- **Anchore**: 容器安全掃描

#### 掃描內容
- 基礎鏡像漏洞
- 安裝的軟件包漏洞
- 配置安全問題
- 敏感信息洩露

## 🚫 生產環境訪問控制

### 原則：零信任架構

1. **最小權限原則**：只授予必要的權限
2. **職責分離**：開發、測試、生產環境分離
3. **審計追蹤**：所有操作必須記錄
4. **自動化優先**：減少人工操作

### 訪問控制架構

```
開發人員
  ↓
GitLab CI/CD (自動化)
  ↓
構建 & 掃描 (必須通過)
  ↓
鏡像 Registry (簽名驗證)
  ↓
Kubernetes (RBAC + Policy)
  ↓
生產環境 (只讀監控)
```

## 🔐 實施方案

### 1. GitLab CI/CD 安全掃描

所有代碼變更必須通過安全掃描才能部署。

### 2. 鏡像簽名與驗證

使用 Docker Content Trust 確保鏡像完整性。

### 3. Kubernetes RBAC

嚴格控制生產環境訪問權限。

### 4. 操作審計

所有操作記錄到審計系統，無法刪除或修改。

### 5. 緊急訪問流程

即使緊急情況，也需要審批和記錄。
