# XE.com 登入方法說明

## 🔐 兩種登入方式

由於 XE.com 可能有反爬蟲機制，我們提供了兩種登入方式：

---

## 方法 1: 使用 Cookies（推薦 ⭐）

**適用情況**：自動登入失敗、需要處理驗證碼、或網站有嚴格的反爬蟲機制

### 步驟

1. **手動登入並保存 cookies**：
   ```bash
   python3 manual_login_helper.py
   ```
   
   這會：
   - 打開瀏覽器
   - 讓您手動完成登入（包括驗證碼等）
   - 保存登入後的 cookies 到 `xe_cookies.json`

2. **使用保存的 cookies 執行抓取**：
   ```bash
   python3 xe_scraper.py --use-cookies
   ```

### 優點

- ✅ **最可靠**：完全繞過自動化檢測
- ✅ **可處理驗證碼**：手動完成所有驗證
- ✅ **穩定**：cookies 可以重複使用（直到過期）
- ✅ **簡單**：只需手動登入一次

### 注意事項

- Cookies 會過期，過期後需要重新執行 `manual_login_helper.py`
- `xe_cookies.json` 包含敏感資訊，已在 `.gitignore` 中

---

## 方法 2: 自動登入（已改進）

**適用情況**：網站沒有嚴格的反爬蟲機制，或自動登入可以成功

### 改進內容

已添加以下改進以提高成功率：

1. **模擬人類輸入**：
   - 逐字輸入（而非一次性輸入）
   - 隨機延遲（0.05-0.2 秒/字元）
   - 輸入後思考時間（0.3-1.5 秒）

2. **增強反檢測**：
   - 隱藏 webdriver 標記
   - 設置真實的 User-Agent
   - 使用 ActionChains 模擬真實點擊

3. **驗證碼處理**：
   - 自動檢測驗證碼
   - 非 headless 模式下可手動完成

### 使用方式

#### 方式 A: 環境變數

```bash
export XE_EMAIL=your-email@example.com
export XE_PASSWORD=your-password
python3 xe_scraper.py
```

#### 方式 B: .env 檔案

```bash
# 1. 創建 .env 檔案
cp .env.example .env

# 2. 編輯 .env，填入帳號資訊
nano .env

# 3. 執行
python3 xe_scraper.py
```

#### 方式 C: 命令列參數

```bash
python3 xe_scraper.py --email your-email@example.com --password your-password
```

### 如果自動登入失敗

如果自動登入失敗（被重定向到 `/signin/error`），建議：

1. **使用 `--no-headless` 查看實際情況**：
   ```bash
   python3 xe_scraper.py --email xxx --password xxx --no-headless
   ```

2. **檢查是否需要驗證碼**：
   - 如果看到驗證碼，程式會提示手動完成
   - 完成後按 Enter 繼續

3. **如果仍然失敗，改用方法 1（Cookies）**：
   ```bash
   python3 manual_login_helper.py
   python3 xe_scraper.py --use-cookies
   ```

## 🎯 建議

### 首次使用

1. 先嘗試**方法 1（Cookies）**，確保可以成功
2. 如果成功，之後可以嘗試**方法 2（自動登入）**

### 如果自動登入一直失敗

- 使用**方法 1（Cookies）**，這是最可靠的方式
- Cookies 通常可以持續使用一段時間（幾天到幾週）

### 定期更新

- Cookies 會過期，過期後需要重新執行 `manual_login_helper.py`
- 建議在每次執行前檢查 cookies 是否仍然有效

---

## 🔧 故障排除

### 問題：自動登入失敗，被重定向到 `/signin/error`

**解決方案**：
1. 確認帳號密碼正確
2. 使用 `--no-headless` 查看是否有驗證碼
3. 改用方法 1（Cookies）

### 問題：Cookies 過期

**解決方案**：
```bash
# 重新執行手動登入
python3 manual_login_helper.py
```

### 問題：找不到驗證碼元素

**解決方案**：
- 使用 `--no-headless` 模式
- 手動在瀏覽器中完成驗證
- 或改用方法 1（Cookies）
