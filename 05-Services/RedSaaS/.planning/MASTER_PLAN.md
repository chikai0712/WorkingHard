# RedSaaS — 組織架構與完整執行計畫

## 願景

打造博弈產業第一個「攻防一體的 AI 資安平台」：
白天是替客戶找漏洞的紅隊引擎，晚上是幫客戶守住金流的 AI 防禦網。
先把內部能力練紮實，再對外輸出成可訂閱的 SaaS 服務。

## 執行階段

### Phase A — 奠基（內部工具可用）

**目標**：讓團隊自己用得起來，完成一次端到端的完整攻防循環。
**完成標準**：從 Web UI 觸發掃描，到 DefectDojo 看到結果，到 Word 報告輸出——全程一鍵完成，不需手動介入。

**紅隊側**
- secureCodeBox 完整管線通（Nmap → Nuclei → DefectDojo Done，非 Errored）
- 博弈漏洞 Template 庫：5 種核心場景，靶場 TP 率 > 80%
- Sliver C2 部署，靶場機器可上線 beacon，Web UI 可見連線狀態
- AI 攻擊建議面板：根據 findings 自動輸出下一步攻擊手法 + ATT&CK 對應

**藍隊側**
- DefectDojo 穩定運行，findings 匯入不再 Errored
- Log 收集管線定義完成（API 呼叫鏈 + UI 操作時序格式統一）
- SIEM 告警基礎規則上線（已知攻擊特徵 100% 觸發）

**AI 側**
- Ollama 本地跑通報告生成（llama3.2:3b → Word 報告）
- 輕量哨兵（8B）MVP 部署：能分辨人類操作 vs 程式化 API 暴力

**DevSecOps**
- Docker Compose + K8s 環境穩定，一鍵啟動靶場
- 博弈 UI 行為基準初版定義（存款/提款/登入三條關鍵流程）

---

### Phase B — 閉環（攻防自動互饋）

**目標**：紅隊找到的東西，藍隊和 AI 能在 24 小時內自動學會防。
**完成標準**：跑一個 BAS 劇本 → 5 分鐘內出防禦缺口報告 → SIEM 規則自動更新。

**紅隊側**
- MITRE Caldera 中控台部署完成，2 台靶場機器安裝 Agent
- 博弈商業邏輯攻擊劇本庫（初版 5 個劇本）：
  - 存提款時序競爭（Race Condition）
  - 代理商後台 API 暴力
  - 跨系統餘額不一致利用
  - 優惠碼重複使用
  - 玩家 Token 橫向移動
- Atomic Red Team 整合：從 ATT&CK 撈取 Payload，由研究員改寫以規避 EDR

**藍隊側**
- BAS 閉環比對機制：攻擊完成 → 自動比對 SIEM 是否觸發 → 產出缺口報告
- 每日防禦缺口日報自動送到看板
- IR playbook 初版：針對每個 BAS 劇本，定義藍隊標準應變流程

**AI 側**
- 輕量哨兵（8B）正式版：延遲 < 50ms，BAS 暴力 API 觸發率 > 95%
- 70B 模型地端部署（Qwen2.5-72B 或 Llama3.3-70B）
- RAG 知識庫 V1：博弈業務規則 + 已知攻擊模式 + ROADMAP 向量化
- BAS 觸發 → 70B 自動調取 24 小時日誌 → 中文根因報告，5 分鐘內完成

**DevSecOps**
- 算力平台：地端 GPU 伺服器（或私有雲）穩定支撐 70B 推論
- BAS 排程自動化：每週跑一輪全場景劇本，報告自動存檔

### Phase C — 產品化（對外可賣）

**目標**：把內部能力包裝成客戶看得懂、用得起、裝得下的產品。
**完成標準**：一個從未接觸過此平台的資安主管，能在 1 小時內看懂演示，並知道月費在買什麼。

**產品化方向（兩條服務線）**

```
服務線 A：紅隊掃描訂閱
  每週 / 每月自動對客戶目標執行 secureCodeBox 掃描
  新 CVE 出現時主動通知 + 自動補測
  輸出 MGA / PAGCOR 合規格式中文報告

服務線 B：AI 行為防禦部署
  輕量哨兵（8B）打包為客戶可部署 Docker image
  交易日誌 sidecar agent（客戶側安裝，資料不出內網）
  每日行為基線報告（正常 vs 異常比例）
  客戶自定義 UI 流程基準的設定介面
```

**紅隊側**
- 合規報告模板完成：MGA 附錄格式 + PAGCOR 安全審查章節
- 第三方 SDK 追蹤：Evolution、Pragmatic、Playtech 等主要遊戲商
- 外網資產持續監控（secureCodeBox CronJob，每週自動跑）

**藍隊側**
- 客戶版 IR playbook：可交付的應變流程文件
- 防禦評估報告：每季給客戶出一份「防禦成熟度」評估

**AI 側**
- 哨兵（8B）客戶版打包完成，支援客戶自備算力部署
- LoRA 微調流程文件化：客戶可提供自家日誌進行客製化微調
- 70B 部署指南：客戶自備 GPU 伺服器的安裝與設定文件

**DevSecOps**
- 多租戶基礎架構：每個客戶的掃描在獨立容器內執行
- 動態 IP 輪替（Egress Proxy）避免客戶目標封鎖

### Phase D — SaaS 化（規模擴張）

**目標**：讓博弈業者 IT 主管可以自助操作，不需要透過業務窗口，建立月費制訂閱收入。
**完成標準**：客戶自行在 Web UI 上建立帳號、設定目標、查看報告、支付月費——全程無需人工介入。

**平台**
- 從 Docker Compose 全面遷移到 K8s（GKE 或 EKS）
- Self-service Web UI（React + FastAPI），取代現有 Flask UI
- 多語言支援（繁中 / 簡中 / 英文）
- AI 防禦看板：每日行為基線時序圖 + 異常事件列表

**商業**
- 訂閱方案設計：紅隊掃描版 / 全套攻防版（含 AI 防禦）
- 金流串接
- 客戶自助帳號管理（建立、暫停、升級方案）

**研究**
- 紅隊具備對市面商用軟體挖掘 0-Day 的能力
- 對外發表博弈產業漏洞研究報告，建立技術聲望

## 閉環運作機制（每日 / 每週循環）

```
每日循環：
  ① AI 哨兵持續吞吐交易日誌，更新行為基線
  ② SIEM 自動比對已知攻擊特徵
  ③ 有告警 → 70B 觸發根因分析 → 自動通知

每週循環：
  ① BAS 自動跑全場景劇本（5 個核心劇本）
  ② 比對 SIEM 觸發率，找出防禦缺口
  ③ 產出防禦缺口週報
  ④ 紅隊把新發現的手法 → 寫成新 BAS 劇本 → 加入下週排程
  ⑤ AI 工程師把新攻擊日誌 → 更新哨兵訓練樣本

每季循環：
  ① 紅隊對客戶執行一次限定範圍紅隊演練（含商業邏輯攻擊）
  ② 演練結果 → 更新 BAS 劇本庫 + RAG 知識庫
  ③ 給客戶出一份「防禦成熟度評估」報告
```

## 關鍵設計原則

1. **紅藍共同業績**：紅隊找到漏洞 = 紅藍雙方的功績；藍隊快速修補 = 紅藍雙方的獎金。避免對立，才能真正閉環。

2. **商業邏輯優先於技術漏洞**：不只掃 CVE，要能找出「兩個合法 API 串聯後產生的邏輯矛盾」。這是市場上最難被複製的差異化能力。

3. **資料不出內網**：交易日誌、業務規則、70B 模型全部地端部署。這既是合規要求，也是對博弈業者最有力的銷售論點。

4. **速度與深度分層**：輕量哨兵守即時（<50ms），70B 大腦做審計（分鐘級）。兩層各司其職，不互相等待。

5. **工具不造輪子**：secureCodeBox + Caldera + Atomic Red Team 都是成熟開源專案。精力放在「博弈業務邏輯」的客製化，不放在重新發明掃描器。
