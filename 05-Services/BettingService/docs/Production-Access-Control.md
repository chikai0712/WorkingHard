# 生產環境訪問控制策略

## 📋 概述

本文件定義了生產環境的訪問控制策略，確保維運人員無法直接接觸生產環境，避免內部資安風險。

## 🚫 訪問控制原則

### 1. 零信任架構
- **默認拒絕**：所有訪問默認拒絕，除非明確允許
- **最小權限**：只授予完成工作所需的最小權限
- **職責分離**：開發、測試、生產環境完全分離

### 2. 自動化優先
- **CI/CD 部署**：所有部署必須通過 CI/CD Pipeline
- **無人工操作**：禁止手動 kubectl 操作生產環境
- **審計追蹤**：所有操作自動記錄，無法刪除

### 3. 緊急訪問流程
- **審批機制**：即使緊急情況，也需要審批
- **時間限制**：緊急訪問有時間限制（默認 1 小時）
- **自動撤銷**：時間到期後自動撤銷權限

## 🔐 實施方案

### 1. Kubernetes RBAC

#### CI/CD Service Account
- **用途**：僅用於自動化部署
- **權限**：僅允許部署操作，禁止刪除和修改敏感資源
- **綁定**：僅綁定到 CI/CD Pipeline

#### 維運人員 Role
- **權限**：僅讀取權限（get, list, watch）
- **禁止**：所有寫入操作（create, update, patch, delete）
- **禁止**：Pod 執行（exec, attach）
- **禁止**：日誌訪問（可能包含敏感信息）

### 2. 網絡隔離

#### NetworkPolicy
- **入站**：僅允許來自 Gateway 和監控系統的流量
- **出站**：僅允許訪問數據庫和 Redis
- **隔離**：生產環境與其他環境完全隔離

### 3. 審計日誌

#### 審計策略
- **記錄所有寫入操作**：create, update, patch, delete
- **記錄所有 RBAC 操作**：權限變更
- **記錄所有 Secrets 訪問**：即使只讀
- **記錄所有 Pod 執行**：exec, attach, portforward
- **不可刪除**：審計日誌存儲在外部系統，無法刪除或修改

### 4. 鏡像簽名驗證

#### Docker Content Trust
- **簽名要求**：所有生產環境鏡像必須簽名
- **驗證機制**：部署前自動驗證簽名
- **拒絕未簽名鏡像**：未簽名鏡像無法部署

## 🚨 緊急訪問流程

### 1. 請求流程

```
1. 維運人員提交緊急訪問請求
   ↓
2. 填寫原因和審批人
   ↓
3. 發送審批請求（Slack/Email）
   ↓
4. 審批人審批
   ↓
5. 創建臨時 RoleBinding（時間限制）
   ↓
6. 記錄到審計系統
   ↓
7. 時間到期自動撤銷
```

### 2. 使用腳本

```bash
# 提交緊急訪問請求
./scripts/emergency-access-request.sh \
  betting-service-prod \
  "緊急修復生產問題" \
  "manager@example.com" \
  3600  # 1 小時
```

### 3. 審批要求

- **審批人**：必須是 Manager 或以上級別
- **原因**：必須詳細說明緊急情況
- **時間限制**：最長 4 小時
- **記錄**：所有操作記錄到審計系統

## 📊 監控與告警

### 1. 異常檢測

#### 監控指標
- Pod 執行操作次數
- 配置變更次數
- 異常 ServiceAccount 使用
- 審計日誌異常模式

#### 告警規則
- **異常訪問**：5 分鐘內超過 5 次 Pod 執行操作
- **未授權操作**：檢測到未審批的寫入操作
- **異常時間**：非工作時間的訪問

### 2. 監控腳本

```bash
# 監控生產環境訪問
./scripts/monitor-production-access.sh betting-service-prod
```

## 🔒 安全措施

### 1. 多因素認證 (MFA)
- **要求**：所有生產環境訪問必須啟用 MFA
- **實施**：通過 Kubernetes 認證插件

### 2. 會話管理
- **超時**：會話 30 分鐘無活動自動登出
- **記錄**：所有會話記錄到審計系統

### 3. IP 白名單
- **限制**：僅允許公司 IP 範圍訪問
- **VPN**：必須通過 VPN 連接

## 📝 最佳實踐

1. **定期審查**：每月審查生產環境訪問記錄
2. **權限清理**：離職人員立即撤銷所有權限
3. **最小權限**：只授予必要的權限
4. **自動化**：盡可能使用自動化，減少人工操作
5. **培訓**：定期培訓團隊安全意識

## 🚨 違規處理

### 違規行為
- 未經審批直接訪問生產環境
- 使用未簽名鏡像
- 繞過 CI/CD Pipeline
- 刪除或修改審計日誌

### 處理流程
1. **立即撤銷權限**
2. **記錄違規行為**
3. **通知安全管理團隊**
4. **進行安全審查**
