## 🔄 CI/CD 流程圖

```
┌─────────────────────────────────────────────────────────────────────────┐
│ CI/CD 完整流程                                                           │
└─────────────────────────────────────────────────────────────────────────┘

開發人員提交代碼
    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│ 1. 代碼審查階段                                                          │
├─────────────────────────────────────────────────────────────────────────┤
│  Merge Request → CODEOWNERS 審查 → 至少 1-2 人批准                       │
└─────────────────────────────────────────────────────────────────────────┘
    ↓ (審查通過)
┌─────────────────────────────────────────────────────────────────────────┐
│ 2. 安全掃描階段                                                          │
├─────────────────────────────────────────────────────────────────────────┤
│  SAST 掃描 (Gosec/Semgrep)                                              │
│    ↓                                                                    │
│  依賴掃描 (Trivy)                                                       │
│    ↓                                                                    │
│  敏感信息掃描 (TruffleHog)                                              │
│    ↓                                                                    │
│  [所有掃描必須通過，否則阻止部署]                                        │
└─────────────────────────────────────────────────────────────────────────┘
    ↓ (掃描通過)
┌─────────────────────────────────────────────────────────────────────────┐
│ 3. 構建階段                                                              │
├─────────────────────────────────────────────────────────────────────────┤
│  構建 Docker 鏡像                                                        │
│    ↓                                                                    │
│  鏡像掃描 (Trivy)                                                       │
│    ↓                                                                    │
│  鏡像簽名 (Docker Content Trust)                                        │
│    ↓                                                                    │
│  推送到 Registry                                                        │
└─────────────────────────────────────────────────────────────────────────┘
    ↓ (構建完成)
┌─────────────────────────────────────────────────────────────────────────┐
│ 4. 部署階段                                                              │
├─────────────────────────────────────────────────────────────────────────┤
│  金絲雀部署 (10% 流量)                                                  │
│    ↓                                                                    │
│  自動驗證 (5-10 分鐘)                                                   │
│    ├─ 錯誤率 < 1%                                                       │
│    ├─ P99 延遲 < 100ms                                                  │
│    └─ 業務成功率 > 99%                                                  │
│    ↓                                                                    │
│  漸進式部署 (50% → 100%)                                                │
│    ↓                                                                    │
│  完全切換到生產環境                                                      │
│                                                                          │
│  [如果驗證失敗，自動回滾]                                                │
└─────────────────────────────────────────────────────────────────────────┘
```

## 📊 數據流架構

```
┌─────────────────────────────────────────────────────────────────────────┐
│ 數據處理流程                                                              │
└─────────────────────────────────────────────────────────────────────────┘

客戶端請求
    ↓
API Gateway (限流/認證)
    ↓
Order Service (Goroutine Pool)
    ↓
┌─────────────────────────────────────────────────────────────────────────┐
│ 處理流程                                                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                           │
│  1. 檢查 Redis 緩存 (熱點數據)                                            │
│     ├─ 命中 → 直接返回                                                   │
│     └─ 未命中 → 繼續                                                     │
│                                                                           │
│  2. 檢查 Redis 鎖 (防止重複提交)                                          │
│     ├─ 已存在 → 返回錯誤                                                 │
│     └─ 不存在 → 設置鎖 (5秒)                                            │
│                                                                           │
│  3. 寫入 PostgreSQL (數據持久化)                                         │
│                                                                           │
│  4. 更新 Redis 緩存 (1小時 TTL)                                          │
│                                                                           │
│  5. 發送到 Kafka (異步處理)                                              │
│                                                                           │
└─────────────────────────────────────────────────────────────────────────┘
```
