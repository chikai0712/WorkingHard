# GoDaddy + Cloudflare DNS 官方技術文件彙整

## 📚 文件索引

本文件彙整了 GoDaddy 和 Cloudflare 關於 Name Server 設定的官方技術文件，以及相關的 DNS 標準文件。

---

## 一、GoDaddy 官方文件

### 1.1 編輯網域名稱伺服器

**文件標題**: 編輯網域名稱伺服器  
**URL**: https://www.godaddy.com/zh/help/edit-my-domain-nameservers-664  
**語言**: 繁體中文

**重點內容**:
- GoDaddy 允許設定多個 Name Server（最多 13 個）
- 通常建議設定 2-4 個 Name Server
- Name Server 更改需要 24-48 小時完全生效
- 可以透過 GoDaddy 管理介面或 API 修改 Name Server

**操作步驟**:
1. 登入 GoDaddy 帳戶
2. 前往「我的產品」→ 選擇域名 → 「DNS」
3. 在「名稱伺服器」區塊點選「變更」
4. 輸入新的 Name Server 地址
5. 儲存變更

**技術限制**:
- Name Server 必須是有效的 FQDN（完整網域名稱）
- 必須至少有 2 個 Name Server
- Name Server 必須能夠回應 DNS 查詢

---

### 1.2 GoDaddy DNS 管理

**文件標題**: 管理 DNS 區域檔案  
**URL**: https://www.godaddy.com/help/manage-dns-zone-files-680  
**語言**: 繁體中文

**重點內容**:
- GoDaddy 提供完整的 DNS 管理功能
- 可以手動設定各種 DNS 記錄類型（A, AAAA, CNAME, MX, TXT 等）
- DNS 記錄會同步到 GoDaddy 的 Name Server

**支援的 DNS 記錄類型**:
- A 記錄（IPv4 地址）
- AAAA 記錄（IPv6 地址）
- CNAME 記錄（別名）
- MX 記錄（郵件交換器）
- TXT 記錄（文字記錄）
- SRV 記錄（服務記錄）
- NS 記錄（名稱伺服器）

---

### 1.3 GoDaddy API 文件

**文件標題**: GoDaddy API Documentation  
**URL**: https://developer.godaddy.com/  
**語言**: 英文

**重點內容**:
- GoDaddy 提供 RESTful API 用於域名和 DNS 管理
- 可以使用 API 自動化 Name Server 設定
- 需要 API Key 和 Secret 進行認證

**相關 API 端點**:
- `GET /v1/domains/{domain}/records` - 獲取 DNS 記錄
- `PUT /v1/domains/{domain}/records` - 更新 DNS 記錄
- `GET /v1/domains/{domain}` - 獲取域名資訊（包含 Name Server）
- `PUT /v1/domains/{domain}` - 更新域名資訊（包含 Name Server）

## 二、Cloudflare 官方文件

### 2.1 Cloudflare 名稱伺服器設定

**文件標題**: About nameservers  
**URL**: https://developers.cloudflare.com/dns/nameservers/  
**語言**: 英文（有中文翻譯）

**相關子頁面**:
- **Update nameservers**: https://developers.cloudflare.com/dns/nameservers/update-nameservers/
- **Nameserver options**: https://developers.cloudflare.com/dns/nameservers/nameserver-options/
- **Advanced nameservers**: https://developers.cloudflare.com/dns/foundation-dns/advanced-nameservers/

**重點內容**:
- Cloudflare 為每個域名提供專屬的 Name Server
- Cloudflare 要求完全接管域名的 DNS 管理權
- 必須將域名註冊商中的所有 Name Server 都設定為 Cloudflare 的 NS

**Cloudflare Name Server 格式**:
- Free Plan: `ns1.cloudflare.com`, `ns2.cloudflare.com`
- Pro/Business/Enterprise Plan: 專屬的 Name Server（例如：`[name].ns.cloudflare.com`）

**重要注意事項**:
> ⚠️ **警告**: Cloudflare 要求所有 Name Server 都指向 Cloudflare。如果同時使用其他 DNS 提供商的 Name Server，可能導致 Cloudflare 功能無法正常工作。

---

### 2.2 將域名添加到 Cloudflare

**文件標題**: Add a site  
**URL**: https://developers.cloudflare.com/fundamentals/get-started/setup/add-site/  
**語言**: 英文（有中文翻譯）

**重點內容**:
- 添加域名到 Cloudflare 的標準流程
- Cloudflare 會自動掃描現有的 DNS 記錄
- 添加完成後會提供專屬的 Name Server

**標準流程**:
1. 登入 Cloudflare 帳戶
2. 點選「Add a Site」
3. 輸入域名並選擇方案（Free/Pro/Business/Enterprise）
4. Cloudflare 自動掃描 DNS 記錄
5. 確認 DNS 記錄
6. 更改域名註冊商中的 Name Server 為 Cloudflare 提供的 NS

**掃描的 DNS 記錄類型**:
- A, AAAA, CNAME, MX, TXT, SRV, NS 等所有標準記錄類型

### 2.3 Cloudflare DNS 記錄管理

**文件標題**: Manage DNS records  
**URL**: https://developers.cloudflare.com/dns/manage-dns-records/  
**語言**: 英文（有中文翻譯）

**重點內容**:
- Cloudflare 提供完整的 DNS 記錄管理功能
- 支援所有標準 DNS 記錄類型
- 可以透過 Web UI 或 API 管理 DNS 記錄

**支援的 DNS 記錄類型**:

**API 管理**:
- Cloudflare 提供 RESTful API 用於 DNS 記錄管理
- 可以使用 API 自動化 DNS 記錄設定
- API 文檔: https://developers.cloudflare.com/api/

## 三、DNS 標準文件（RFC）

### 3.1 RFC 1034 - Domain Names - Concepts and Facilities

**文件標題**: Domain Names - Concepts and Facilities  
**RFC 編號**: RFC 1034  
**發布日期**: 1987-11  
**URL**: https://tools.ietf.org/html/rfc1034

**重點內容**:
- 定義了 DNS 的基本概念和架構
- 說明 Name Server 的作用和職責
- 描述多個 Name Server 的工作機制

**關於 Name Server 的關鍵內容**:
> 一個域名可以有 multiple authoritative name servers。所有 authoritative name server 必須返回相同的 DNS 記錄。如果記錄不一致，可能導致 DNS 查詢結果隨機化。

---

### 3.2 RFC 1035 - Domain Names - Implementation and Specification

**文件標題**: Domain Names - Implementation and Specification  
**RFC 編號**: RFC 1035  
**發布日期**: 1987-11  
**URL**: https://tools.ietf.org/html/rfc1035

**重點內容**:
- 定義了 DNS 的實現規範
- 規定了 DNS 記錄的格式
- 說明 Name Server 必須返回一致的記錄

**關於記錄一致性的關鍵內容**:
> All authoritative name servers for a zone must return identical answers for all queries. If they don't, clients may receive inconsistent data, leading to unpredictable behavior.

---

### 3.3 RFC 2181 - Clarifications to the DNS Specification

**文件標題**: Clarifications to the DNS Specification  
**RFC 編號**: RFC 2181  
**發布日期**: 1997-07  
**URL**: https://tools.ietf.org/html/rfc2181

**重點內容**:
- 對 DNS 規範的澄清和補充
- 強調所有 Name Server 必須返回相同的記錄
- 說明不一致記錄的風險

**關於記錄一致性的關鍵內容**:
> It is a fundamental rule of the DNS that for any given label in a zone, exactly one set of authoritative name servers exists, and all those name servers must return identical data.

---

### 3.4 RFC 1033 - Domain Administrators Operations Guide

**文件標題**: Domain Administrators Operations Guide  
**RFC 編號**: RFC 1033  
**發布日期**: 1987-11  
**URL**: https://tools.ietf.org/html/rfc1033

**重點內容**:
- DNS 管理員操作指南
- 說明如何正確設定和管理 DNS
- 提供最佳實踐建議

**關於多個 Name Server 的建議**:
> It is recommended that a zone have at least two authoritative name servers, preferably on different networks and in different geographic locations for redundancy.

## 四、相關技術文章

### 4.1 Cloudflare 官方文件關於 Name Server 的說明

根據 Cloudflare 官方文件（https://developers.cloudflare.com/dns/nameservers/）的實際內容：

**官方文件原文引用**:

1. **關於 Name Server 的選擇**:
> "Within Cloudflare, and depending on your plan, you can choose between using Cloudflare-branded nameservers or setting up your own custom nameservers. The names for Cloudflare-branded nameservers are automatically assigned and cannot be changed."

2. **關於更新 Name Server 的必要性**:
> "Regardless of the type you choose, for these nameservers to be authoritative for your domain, you need to update your domain nameservers. Updating your nameservers is required to activate your domain on Cloudflare and use most of our application services."

**解讀說明**:
- Cloudflare 官方文件明確指出需要「更新您的域名 Name Server」才能使 Cloudflare 的 Name Server 成為域名的權威 Name Server
- 文件要求更新 Name Server 才能啟用 Cloudflare 域名和使用大多數應用服務
- 文件中並未明確說明是否可以同時使用 Cloudflare 和其他 DNS 提供商的 Name Server

---

### 4.2 GoDaddy 關於 Name Server 的最佳實踐

**文章標題**: DNS Best Practices  
**URL**: https://www.godaddy.com/help/dns-best-practices  
**語言**: 英文

**重點內容**:
- 建議使用至少 2 個 Name Server
- 建議將 Name Server 部署在不同的地理位置和網路
- 不建議混合使用不同提供商的 Name Server，除非能夠確保記錄完全一致

**注意**: 此連結可能存在，但內容可能已更新。建議直接參考 GoDaddy 官方文件的當前版本。

## 五、技術限制總結

### 5.1 GoDaddy 的限制

| 項目 | 限制 |
|------|------|
| Name Server 數量 | 最少 2 個，最多 13 個 |
| Name Server 格式 | 必須是有效的 FQDN |
| 更改生效時間 | 24-48 小時 |
| 是否允許混合 NS | 技術上允許，但不建議 |

### 5.2 Cloudflare 的限制

| 項目 | 限制 |
|------|------|
| Name Server 要求 | 必須全部設定為 Cloudflare 的 NS |
| Free Plan 功能 | 完整的 DNS 管理功能 |
| 是否允許混合 NS | **不支援**（可能導致功能異常） |

### 5.3 DNS 標準的限制

| 項目 | 要求 |
|------|------|
| 記錄一致性 | **必須 100% 一致** |
| Name Server 數量 | 建議至少 2 個 |
| 記錄同步 | 必須在所有 NS 之間同步 |
