# DNS 故障切換測試 - 完整執行步驟

## 📋 測試目標

驗證當一個 DNS 伺服器被封鎖時，系統是否會自動切換到其他 DNS 伺服器。

---

## 🚀 完整執行步驟

### 步驟 1：準備工作

```bash
# 1.1 開啟終端機（保持開啟，用於執行命令）

# 1.2 開啟 Wireshark（如果還沒開啟）
# - 選擇網路介面（例如：Wi-Fi 或 en0）
# - 過濾器輸入：dns || icmp
# - 點擊「開始捕獲」

# 1.3 確認測試域名和 DNS
DOMAIN="www.clouddeployment168.site"
AWS_DNS="205.251.197.44"
GOOGLE_DNS="216.239.32.108"
```

---

### 步驟 2：封鎖 DNS 伺服器

```bash
# 2.1 封鎖 AWS DNS
echo 'block drop out quick proto udp from any to 205.251.197.44 port 53' | \
    sudo pfctl -a custom -f -

# 2.2 驗證封鎖規則是否存在
sudo pfctl -a custom -sr | grep 205.251.197.44
# 應該看到：block drop out quick proto udp from any to 205.251.197.44 port 53

# 2.3 測試封鎖是否生效
dig @205.251.197.44 www.clouddeployment168.site +time=2 +tries=1
# 應該失敗或超時（如果成功，表示封鎖未生效）
```

**預期結果：**
- 規則存在
- `dig` 查詢失敗或超時

---

### 步驟 3：清除 DNS 快取

```bash
# 3.1 清除 macOS DNS 快取
sudo dscacheutil -flushcache
sudo killall -HUP mDNSResponder

# 3.2 確認快取已清除（可選）
dscacheutil -q host -a name www.clouddeployment168.site
# 應該沒有快取結果
```

---

### 步驟 4：執行 DNS 查詢（觀察故障切換）

```bash
# 4.1 在 Wireshark 中確認捕獲已開始

# 4.2 執行查詢（不指定 DNS，讓系統自動選擇）
nslookup www.clouddeployment168.site

# 4.3 觀察結果
# - 如果成功解析到 IP，表示系統切換到其他 DNS
# - 如果失敗，表示所有 DNS 都被封鎖或系統沒有切換
```

---

### 步驟 5：在 Wireshark 中觀察

**應該看到的封包順序：**

1. **DNS Query** → 205.251.197.44
   - 時間：例如 0.000s
   - 來源：你的 IP (10.40.3.146)
   - 目標：205.251.197.44

2. **ICMP Destination Unreachable** ← 205.251.197.44
   - 時間：例如 0.001s
   - 表示：DNS 被封鎖，封包無法到達

3. **DNS Query Retransmission** → 205.251.197.44（可選）
   - 時間：例如 1.000s
   - 表示：系統重試查詢

4. **DNS Query** → 216.239.32.108 或 8.8.8.8
   - 時間：例如 2.000s
   - 表示：系統切換到其他 DNS

5. **DNS Response** ← 成功回應
   - 時間：例如 2.050s
   - 包含：IP 位址 44.202.226.49

**時間間隔分析：**
- 第一次查詢到 ICMP 錯誤：< 10ms
- ICMP 錯誤到切換：1-5 秒
- 切換到成功回應：< 100ms

---

### 步驟 6：解除封鎖

```bash
# 6.1 解除封鎖
sudo pfctl -a custom -F all

# 6.2 驗證已解除
sudo pfctl -a custom -sr | grep 205.251.197.44
# 應該沒有輸出（表示已解除）

# 6.3 測試恢復正常
dig @205.251.197.44 www.clouddeployment168.site +time=2
# 應該成功
```

## 🎯 預期結果

### 成功的情況

1. **封鎖生效**
   - `dig @205.251.197.44` 失敗或超時
   - Wireshark 顯示 ICMP 錯誤

2. **故障切換發生**
   - 系統在 1-5 秒內切換到其他 DNS
   - Wireshark 顯示查詢其他 DNS 伺服器
   - 最終查詢成功

3. **時間分析**
   - 第一次查詢失敗：立即
   - 切換時間：1-5 秒
   - 總時間：< 10 秒

### 失敗的情況

1. **封鎖未生效**
   - `dig @205.251.197.44` 仍然成功
   - 沒有 ICMP 錯誤
   - **解決方法：** 檢查 pfctl 規則

2. **沒有切換**
   - 查詢一直失敗
   - 沒有查詢其他 DNS
   - **可能原因：** 系統只配置了一個 DNS

3. **立即切換（< 100ms）**
   - 可能是並行查詢，不是故障切換
   - **解決方法：** 只配置一個 DNS 測試

## 🐛 常見問題

### Q: 封鎖規則沒有顯示？

**A:** 
```bash
# 檢查所有規則
sudo pfctl -sr

# 檢查 anchor
sudo pfctl -a custom -sr

# 如果都沒有，重新封鎖
echo 'block drop out quick proto udp from any to 205.251.197.44 port 53' | \
    sudo pfctl -a custom -f -
```

### Q: dig 查詢仍然成功？

**A:**
```bash
# 1. 確認規則存在
sudo pfctl -a custom -sr | grep 205.251.197.44

# 2. 清除快取
sudo dscacheutil -flushcache

# 3. 使用 +tries=1 強制只嘗試一次
dig @205.251.197.44 www.clouddeployment168.site +time=2 +tries=1
```

### Q: 沒有看到切換？

**A:**
```bash
# 1. 檢查系統 DNS 設定
scutil --dns | grep nameserver

# 2. 如果只有一個 DNS，系統無法切換
# 解決方法：在系統設定中添加多個 DNS

# 3. 確認封鎖生效
dig @205.251.197.44 www.clouddeployment168.site +time=2
```
