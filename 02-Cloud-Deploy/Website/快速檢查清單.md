# AWS 金鑰連線快速檢查清單

## ✅ 前置需求檢查

### 1. 安裝 AWS CLI

**macOS（推薦）：**
```bash
# 使用自動安裝腳本
./install-aws-cli.sh

# 或手動安裝
brew install awscli
```

**驗證：**
```bash
aws --version
# 應該顯示：aws-cli/2.x.x
```

### 2. 配置 AWS 憑證

```bash
aws configure
```

需要輸入：
- AWS Access Key ID
- AWS Secret Access Key  
- Default region（例如：`ap-northeast-1`）
- Default output format（建議：`json`）

**驗證：**
```bash
aws sts get-caller-identity
# 應該顯示你的 AWS 帳號資訊
```

## ✅ Key Pair 設定檢查

### 1. 建立或確認 Key Pair

**在 AWS Console：**
- EC2 → Key Pairs → 確認是否有 Key Pair

**或使用 AWS CLI：**
```bash
# 列出所有 Key Pairs
aws ec2 describe-key-pairs

# 建立新的 Key Pair
aws ec2 create-key-pair \
  --key-name my-ec2-key \
  --query 'KeyMaterial' \
  --output text > ~/.ssh/my-ec2-key.pem
```

### 2. 確認 Key 檔案存在

```bash
ls -la ~/.ssh/*.pem
```

### 3. 設定正確權限

```bash
chmod 400 ~/.ssh/my-ec2-key.pem
```

**驗證權限：**
```bash
ls -la ~/.ssh/my-ec2-key.pem
# 應該顯示：-r--------
```

## ✅ EC2 實例檢查

### 1. 確認實例運行中

```bash
aws ec2 describe-instances \
  --query 'Reservations[*].Instances[*].[InstanceId,State.Name,PublicIpAddress]' \
  --output table
```

### 2. 確認實例使用正確的 Key Pair

```bash
aws ec2 describe-instances \
  --instance-ids i-xxx \
  --query 'Reservations[0].Instances[0].KeyName'
```

### 3. 確認 Security Group 開放 SSH

```bash
# 列出實例的 Security Group
aws ec2 describe-instances \
  --instance-ids i-xxx \
  --query 'Reservations[0].Instances[0].SecurityGroups[*].GroupId'

# 檢查 Security Group 規則
aws ec2 describe-security-groups \
  --group-ids sg-xxx \
  --query 'SecurityGroups[0].IpPermissions[?FromPort==`22`]'
```

**如果需要開放 SSH：**
```bash
aws ec2 authorize-security-group-ingress \
  --group-id sg-xxx \
  --protocol tcp \
  --port 22 \
  --cidr 0.0.0.0/0  # 或你的 IP/32（更安全）
```

## ✅ 連線測試

### 1. 取得實例資訊

```bash
# 取得 Public IP
PUBLIC_IP=$(aws ec2 describe-instances \
  --instance-ids i-xxx \
  --query 'Reservations[0].Instances[0].PublicIpAddress' \
  --output text)

echo $PUBLIC_IP
```

### 2. 測試連線

```bash
# Amazon Linux
ssh -i ~/.ssh/my-ec2-key.pem ec2-user@$PUBLIC_IP

# Ubuntu
ssh -i ~/.ssh/my-ec2-key.pem ubuntu@$PUBLIC_IP
```

## 📋 完整檢查命令

```bash
#!/bin/bash

echo "=== AWS CLI 檢查 ==="
aws --version || echo "❌ AWS CLI 未安裝"

echo ""
echo "=== AWS 憑證檢查 ==="
aws sts get-caller-identity || echo "❌ AWS 憑證未設定"

echo ""
echo "=== Key Pairs 檢查 ==="
aws ec2 describe-key-pairs --query 'KeyPairs[*].KeyName' --output table

echo ""
echo "=== EC2 實例檢查 ==="
aws ec2 describe-instances \
  --query 'Reservations[*].Instances[*].[InstanceId,State.Name,PublicIpAddress,KeyName]' \
  --output table

echo ""
echo "=== 本地 Key 檔案檢查 ==="
ls -la ~/.ssh/*.pem 2>/dev/null || echo "❌ 找不到 .pem 檔案"
```

## ⚠️ 常見錯誤與解決

| 錯誤 | 原因 | 解決方法 |
|------|------|----------|
| `aws: command not found` | AWS CLI 未安裝 | `brew install awscli` |
| `Unable to locate credentials` | 未配置憑證 | `aws configure` |
| `Permission denied (publickey)` | Key 權限錯誤 | `chmod 400 ~/.ssh/key.pem` |
| `Connection timed out` | Security Group 未開放 | 開放 port 22 |
| `Host key verification failed` | IP 改變 | `ssh-keygen -R <IP>` |
