# 🚀 快速開始指南

## 5 分鐘快速部署

### 步驟 1：設定變數

```bash
cd terraform
cp terraform.tfvars.example terraform.tfvars
```

編輯 `terraform.tfvars`：

```hcl
aws_region = "ap-northeast-1"  # 或你偏好的區域
domain_name = "yourdomain.com"  # 改為你的域名
bucket_name = "yourdomain-com-website"  # 必須全球唯一
additional_domains = ["www.yourdomain.com"]  # 可選
create_route53_zone = false  # 如果域名在 Route 53，設為 true
```

### 步驟 2：初始化並部署

```bash
# 初始化
terraform init

# 預覽變更
terraform plan

# 部署（輸入 yes 確認）
terraform apply
```

### 步驟 3：設定 DNS

#### 如果域名在 Route 53（create_route53_zone = true）

DNS 會自動設定，等待 5-30 分鐘即可。

#### 如果域名在其他 DNS 提供商

1. **先設定 ACM 證書驗證記錄**（必須！）

```bash
terraform output acm_validation_records
```

在你的 DNS 提供商新增 CNAME 記錄：
- Name: 從輸出中複製 `name` 欄位
- Value: 從輸出中複製 `record` 欄位

2. **等待證書驗證**（約 5-30 分鐘）

在 AWS Console → Certificate Manager (us-east-1) 檢查證書狀態，變為 "Issued" 後繼續。

3. **設定域名指向 CloudFront**

```bash
terraform output cloudfront_domain_name
terraform output cloudfront_distribution_id
```

在你的 DNS 提供商新增記錄：

**選項 A：CNAME（推薦）**
- Type: CNAME
- Name: `yourdomain.com`（或 `@`）
- Value: CloudFront 域名（從輸出複製）

**選項 B：A Record（Alias）**
- Type: A (Alias)
- Name: `yourdomain.com`
- Alias Target: CloudFront 域名
- Alias Hosted Zone ID: `Z2FDTNDATAQYW2`（CloudFront 的固定值）

### 步驟 4：上傳網站檔案

```bash
# 從專案根目錄執行
cd ../..
aws s3 sync Website s3://$(cd terraform && terraform output -raw s3_bucket_name) \
  --exclude "*.md" \
  --exclude ".git/*" \
  --exclude "terraform/*" \
  --exclude "*.sh" \
  --delete

# 清除 CloudFront 快取
aws cloudfront create-invalidation \
  --distribution-id $(cd terraform && terraform output -raw cloudfront_distribution_id) \
  --paths "/*"
```

### 步驟 5：完成！

等待 DNS 傳播（5-30 分鐘），然後訪問：`https://yourdomain.com`

## 使用部署腳本（更簡單）

```bash
cd terraform
./deploy.sh
```

腳本會自動：
- ✅ 檢查前置需求
- ✅ 初始化 Terraform
- ✅ 部署基礎設施
- ✅ 上傳網站檔案
- ✅ 清除 CloudFront 快取

## 重要注意事項

### ⚠️ ACM 證書區域

**CloudFront 只能使用 `us-east-1` 區域的 ACM 證書**

配置已自動處理此問題，證書會在 `us-east-1` 建立，即使你的其他資源在其他區域。

### ⚠️ S3 Bucket 名稱

- 必須全球唯一
- 只能包含小寫字母、數字、連字號
- 建議格式：`yourdomain-com-website`

### ⚠️ DNS 傳播時間

- 通常 5-30 分鐘
- 最多可能需要 48 小時
- 使用 `dig yourdomain.com` 檢查 DNS 是否已更新

terraform output

## 故障排除

### 問題：ACM 證書驗證失敗

1. 確認 DNS 驗證記錄已正確設定
2. 等待 5-30 分鐘讓 DNS 傳播
3. 檢查記錄：`dig <validation-record-name> CNAME`

### 問題：CloudFront 顯示 403

1. 確認檔案已上傳到 S3
2. 確認 S3 Bucket Policy 允許 CloudFront 訪問
3. 檢查 Origin Access Control 設定

### 問題：域名無法訪問

1. 確認 DNS 記錄已設定
2. 等待 DNS 傳播
3. 檢查：`dig yourdomain.com`
