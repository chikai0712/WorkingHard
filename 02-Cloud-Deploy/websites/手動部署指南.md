# 博弈網站手動部署指南

## 方法：使用 AWS 控制台部署

### 步驟 1：登入 AWS 控制台
1. 前往：https://console.aws.amazon.com/
2. 搜尋並進入 "EC2"
3. 點擊左側 "實例"

### 步驟 2：連線到第一台服務器（35.74.79.10）
1. 選擇實例（IP: 35.74.79.10）
2. 點擊右上角「連線」按鈕
3. 選擇「EC2 Instance Connect」標籤
4. 點擊「連線」按鈕（會在瀏覽器開啟終端）

### 步驟 3：備份並建立新網站
在終端中執行以下命令：

```bash
# 備份現有網站
sudo cp /var/www/html/index.html /var/www/html/index.html.backup

# 建立新網站文件
sudo nano /var/www/html/index.html
```

### 步驟 4：貼上網站內容
1. 在 nano 編輯器中，按 `Ctrl+K` 多次刪除所有內容
2. 從 Cursor 開啟檔案：`websites/site2-gaming/aws.html`
3. 全選並複製所有內容（Cmd+A, Cmd+C）
4. 回到瀏覽器終端，按右鍵貼上
5. 按 `Ctrl+X`，然後按 `Y`，然後按 `Enter` 儲存

### 步驟 5：重啟服務
```bash
sudo systemctl restart httpd
```

### 步驟 6：驗證
在瀏覽器開啟：http://35.74.79.10
應該會看到金色的「👑 皇冠娛樂城 👑」

### 步驟 7：重複步驟 2-6 在第二台服務器
對 IP: 35.78.244.92 重複相同步驟

## 快速驗證命令

部署完成後，在終端執行：

```bash
# 檢查內容

# 檢查服務狀態

```

## 如果遇到問題

### 問題 1：找不到 /var/www/html 目錄
```bash
# 建立目錄
sudo mkdir -p /var/www/html
sudo chown -R nginx:nginx /var/www/html
```

### 問題 2：服務無法啟動
```bash
# 檢查錯誤
sudo journalctl -u httpd -n 50

# 或檢查 nginx
sudo systemctl status nginx
```

### 問題 3：網站無法訪問
1. 檢查 AWS Security Group 是否開放 80 埠
2. 檢查防火牆：`sudo firewall-cmd --list-all`
3. 檢查服務：`sudo systemctl status httpd`
