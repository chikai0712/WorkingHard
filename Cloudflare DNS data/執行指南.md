## 📁 輸出檔案

腳本執行完成後會產生以下檔案：

### 1. 主要結果檔案
- **檔名格式**：`cloudflare_metrics_{timestamp}.csv`
- **內容**：所有域名的完整數據
- **欄位**：
  - 帳戶
  - 域名
  - 狀態
  - 計劃
  - 總請求數
  - 總流量(MB)
  - 緩存流量(MB)
  - 緩存請求數
  - DNS查詢數

### 2. 排序結果檔案
- **檔名格式**：`cloudflare_all_sorted_{timestamp}.csv`
- **內容**：按總流量降序排列的所有結果

### 3. 失敗記錄檔案（如果有失敗）
- **檔名格式**：`cloudflare_failed_domains_{timestamp}.csv`
- **內容**：處理失敗的域名及原因

### 範例檔名
```
cloudflare_metrics_20250102_120000.csv
cloudflare_all_sorted_20250102_120000.csv
cloudflare_failed_domains_20250102_120000.csv
```

- **單一域名**：約 5-7 秒（5 個指標 + 延遲）
- **10 個域名批次**：約 1-2 分鐘

## ⚠️ 注意事項

### 1. 速率限制
- 腳本已內建速率限制處理
- 如果遇到速率限制，會自動重試
- 批次間會自動休息，避免觸發限制

### 2. 計劃限制
- Free 計劃的 DNS Analytics 可能不支援
- 不支援的域名會標記為 `"N/A (計劃不支援)"`
- 不會中斷整個處理流程

### 3. 網路連線
- 確保網路連線穩定
- 如果中斷，可以重新執行（已處理的數據不會重複）

### 4. 數據範圍
- 目前設定為查詢 **過去 7 天** 的數據
- 如需修改，請編輯腳本第 35 行：
  ```python
  start_date = end_date - datetime.timedelta(days=7)  # 改為需要的天數
  ```

## 🔍 故障排除

### 問題 1：ModuleNotFoundError

**錯誤訊息**：
```
ModuleNotFoundError: No module named 'requests'
```

**解決方法**：
```bash
pip3 install requests pandas
```

### 問題 2：API 認證失敗

**錯誤訊息**：
```
獲取域名列表失敗: [{'code': 6003, 'message': 'Invalid request headers'}]
```

**解決方法**：
- 檢查 API Key 是否正確
- 確認 Email 和 API Key 是否匹配
- 確認 Account ID 是否正確

### 問題 3：速率限制

**現象**：
- 出現 "遇到速率限制" 訊息
- 腳本會自動等待並重試

**處理**：
- 腳本會自動處理，無需手動介入
- 如果持續出現，可以增加批次間的休息時間

### 問題 4：DNS Analytics 不支援

**現象**：
- 出現 `[跳過] 域名 xxx: DNS Analytics 不支援` 訊息

**說明**：
- 這是正常情況，某些計劃不支援 DNS Analytics
- 該域名的 DNS 查詢數會顯示為 `"N/A (計劃不支援)"`
