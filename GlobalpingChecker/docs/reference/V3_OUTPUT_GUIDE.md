# v3.0 輸出欄位說明

## 📊 完整輸出範例

```
🔍 檢測域名 [1/2]: 7plusmm.com ...
  📍 BIZNET NETWORKS (AS17451) [Jakarta]
     🔌 節點IP: 202.169.224.46  | 🎯 目標IP: 107.167.190.204  | [CLEAN] ✅ 正常連通 (HTTP 308)
  📍 Media Sarana Data (AS45727) [Surabaya]
     🔌 節點IP: 103.49.221.211  | 🎯 目標IP: 107.167.190.204  | [CLEAN] ✅ 正常連通 (HTTP 308)
  📍 XL Axiata (AS24203) [Jakarta]
     🔌 節點IP: 202.152.0.2     | 🎯 目標IP: 107.167.190.204  | [CLEAN] ✅ 正常連通 (HTTP 308)
  -> 整體狀態: CLEAN
------------------------------------------------
```

---

## 🔍 欄位詳細說明

### 1. 檢測域名行

```
🔍 檢測域名 [1/2]: 7plusmm.com ...
```

| 欄位 | 說明 | 範例 |
|------|------|------|
| `[1/2]` | 當前進度（第1個/共2個） | `[1/2]` |
| `7plusmm.com` | 被檢測的域名 | `7plusmm.com` |

---

### 2. 節點信息行

```
📍 BIZNET NETWORKS (AS17451) [Jakarta]
```

| 欄位 | 說明 | 範例 | 用途 |
|------|------|------|------|
| `📍` | 節點標記 | - | 視覺標識 |
| `BIZNET NETWORKS` | **ISP 名稱** | BIZNET NETWORKS | 識別測試節點的網絡提供商 |
| `(AS17451)` | **ASN 號碼** | AS17451 | 自治系統編號，唯一識別 ISP |
| `[Jakarta]` | **城市** | Jakarta | 測試節點所在城市 |

**ASN 說明**：
- AS17451 = BIZNET（印尼商業網絡）
- AS45727 = Media Sarana Data（印尼 ISP）
- AS24203 = XL Axiata（印尼電信運營商）

---

### 3. 測試結果行

```
🔌 節點IP: 202.169.224.46  | 🎯 目標IP: 107.167.190.204  | [CLEAN] ✅ 正常連通 (HTTP 308)
```

#### 3.1 節點 IP

```
🔌 節點IP: 202.169.224.46
```

| 欄位 | 說明 | 範例 | 用途 |
|------|------|------|------|
| `🔌 節點IP` | 測試機器的 IP | 202.169.224.46 | 識別具體的測試節點 |

**說明**：
- 這是 Globalping 測試節點的公網 IP
- 代表從哪個 IP 發起測試
- 可能顯示為 `private`（隱私保護）

---

#### 3.2 目標 IP

```
🎯 目標IP: 107.167.190.204
```

| 欄位 | 說明 | 範例 | 用途 |
|------|------|------|------|
| `🎯 目標IP` | 域名解析後的 IP | 107.167.190.204 | 域名實際指向的服務器 IP |

**重要性**：
- ✅ **正常 IP**：域名正確解析到真實服務器
- ❌ **污染 IP**：`36.86.63.185`、`10.x.x.x`、`127.0.0.1`
- ⚠️ **無解析**：顯示為空或 `無解析`

**範例**：
```
正常：🎯 目標IP: 107.167.190.204  ✅
污染：🎯 目標IP: 36.86.63.185     ❌ DNS 污染
超時：🎯 目標IP: 無解析            ⚠️ 無法連接
```

---

#### 3.3 狀態標記

```
[CLEAN] ✅ 正常連通 (HTTP 308)
```

| 欄位 | 說明 | 含義 |
|------|------|------|
| `[CLEAN]` | 狀態代碼 | 正常連通 |
| `✅` | 視覺標記 | 成功 |
| `正常連通` | 狀態描述 | 可以正常訪問 |
| `(HTTP 308)` | HTTP 狀態碼 | 永久重定向 |

---

## 📋 所有狀態類型

### 1. CLEAN（正常連通）✅

```
[CLEAN] ✅ 正常連通 (HTTP 200)
[CLEAN] ✅ 正常連通 (HTTP 301)
[CLEAN] ✅ 正常連通 (HTTP 308)
[CLEAN] ✅ 正常連通 (HTTP 403)
```

**含義**：
- 域名可以正常解析
- 服務器正常響應
- HTTP 狀態碼為 2xx、3xx 或 403

**HTTP 狀態碼說明**：
- `200 OK` - 正常訪問
- `301 Moved Permanently` - 永久重定向（正常）
- `308 Permanent Redirect` - 永久重定向（正常）
- `403 Forbidden` - 禁止訪問（但服務器存在）

---

### 2. BLOCKED（DNS 污染）🚨

```
[BLOCKED] 🚨 DNS 污染
```

**含義**：
- 域名被解析到錯誤的 IP
- 常見污染 IP：
  - `36.86.63.185`（印尼常見污染 IP）
  - `10.x.x.x`（內網 IP）
  - `127.0.0.1`（本地回環）
  - `0.0.0.0`（無效 IP）

**範例**：
```
🎯 目標IP: 36.86.63.185  | [BLOCKED] 🚨 DNS 污染
```

---

### 3. TIMEOUT（完全超時）⚠️

```
[TIMEOUT] ⚠️  阻斷/無回應
```

**含義**：
- 無法解析域名
- 或無法連接到服務器
- 可能被防火牆阻斷

**範例**：
```
🎯 目標IP: 無解析  | [TIMEOUT] ⚠️  阻斷/無回應
```

---

### 4. WARNING（服務異常）⚠️

```
[WARNING] ⚠️  異常狀態 (HTTP 500)
[WARNING] ⚠️  異常狀態 (HTTP 502)
```

**含義**：
- 域名可以解析
- 但服務器返回錯誤狀態碼
- HTTP 4xx（除了 403）或 5xx

**HTTP 狀態碼說明**：
- `404 Not Found` - 頁面不存在
- `500 Internal Server Error` - 服務器內部錯誤
- `502 Bad Gateway` - 網關錯誤
- `503 Service Unavailable` - 服務不可用

---

### 5. PARTIAL（部分異常）🔄

```
-> 整體狀態: PARTIAL
```

**含義**：
- 3 個節點測試結果不一致
- 例如：2 個 CLEAN + 1 個 TIMEOUT
- 需要二次檢測確認

**範例**：
```
節點1: [CLEAN] ✅
節點2: [CLEAN] ✅
節點3: [TIMEOUT] ⚠️
-> 整體狀態: PARTIAL
```

---

### 6. API_ERROR（檢測失敗）❌

```
[API_ERROR] ❌ API 請求失敗
```

**含義**：
- API 配額用完
- 網絡連接問題
- API 服務異常

## 📊 整體狀態判斷邏輯

### 判斷規則

```
3 個節點全部 CLEAN     → 整體狀態: CLEAN
3 個節點全部 BLOCKED   → 整體狀態: BLOCKED
3 個節點全部 TIMEOUT   → 整體狀態: TIMEOUT
3 個節點全部 WARNING   → 整體狀態: WARNING
節點結果不一致          → 整體狀態: PARTIAL
```

### 範例

#### 範例 1: CLEAN（全部正常）

```
節點1: [CLEAN] ✅ 正常連通 (HTTP 308)
節點2: [CLEAN] ✅ 正常連通 (HTTP 308)
節點3: [CLEAN] ✅ 正常連通 (HTTP 308)
-> 整體狀態: CLEAN
```

#### 範例 2: BLOCKED（全部污染）

```
節點1: [BLOCKED] 🚨 DNS 污染
節點2: [BLOCKED] 🚨 DNS 污染
節點3: [BLOCKED] 🚨 DNS 污染
-> 整體狀態: BLOCKED
```

#### 範例 3: PARTIAL（部分異常）

```
節點1: [CLEAN] ✅ 正常連通 (HTTP 308)
節點2: [CLEAN] ✅ 正常連通 (HTTP 308)
節點3: [TIMEOUT] ⚠️  阻斷/無回應
-> 整體狀態: PARTIAL
```

## 💡 如何解讀結果

### ✅ CLEAN - 正常

**含義**：域名在印尼可以正常訪問

**行動**：無需處理

---

### 🚨 BLOCKED - DNS 污染

**含義**：域名被 DNS 污染，無法正常訪問

**行動**：
1. 確認域名是否真的被封鎖
2. 考慮更換域名
3. 或使用 CDN 繞過

---

### ⚠️ TIMEOUT - 超時

**含義**：可能被防火牆阻斷

**行動**：
1. 檢查服務器是否正常
2. 檢查防火牆設置
3. 可能需要更換 IP

---

### 🔄 PARTIAL - 部分異常

**含義**：不同 ISP 結果不一致，需要進一步確認

**行動**：
1. v3.0 會自動進行二次檢測
2. 查看二次檢測結果
3. 根據最終結果決定
