# V4.1 部署指南

## 📋 更新內容

### 1. API_ERROR 處理機制
- 遇到 HTTP 429 (Too Many Requests) 時，自動等待 60 分鐘
- 遇到 HTTP 500+ (服務器錯誤) 時，自動等待 60 分鐘
- 遇到未知異常時，自動等待 60 分鐘
- 記錄暫停和恢復時間

### 2. 時區設置
- 系統時區：GMT+8 (Asia/Taipei)
- PostgreSQL 時區：GMT+8
- 所有時間戳使用台北時間

### 3. 智能循環檢測
- 異常區檢測：每天 AM 1:00 開始
- 正常區檢測：每天 AM 9:00 開始
- 檢測間隔：90 分鐘
- 最大迭代：10 次

## 🚀 快速部署

### 方法 1：使用自動化腳本（推薦）

```bash
cd ~/Desktop/Project/GlobalpingChecker
./deploy-v4.1-to-aws.sh
```

這個腳本會自動：
1. ✅ 打包 V4.1 代碼
2. ✅ 上傳到 AWS
3. ✅ 停止舊服務
4. ✅ 備份舊版本
5. ✅ 部署新版本
6. ✅ 啟動服務
7. ✅ 驗證部署

### 方法 2：手動部署

#### 步驟 1：打包代碼
```bash
cd ~/Desktop/Project/GlobalpingChecker
tar -czf v4.1-deploy.tar.gz v4.1/
```

#### 步驟 2：上傳到 AWS
```bash
scp -i ~/.ssh/globalping-checker-key.pem v4.1-deploy.tar.gz ubuntu@54.238.247.106:~/
```

#### 步驟 3：SSH 到 AWS
```bash
ssh -i ~/.ssh/globalping-checker-key.pem ubuntu@54.238.247.106
```

#### 步驟 4：在 AWS 上執行
```bash
# 停止當前服務
cd ~/v4.1
docker-compose down

# 備份
cd ~
mv v4.1 v4.1-backup-$(date +%Y%m%d)

# 解壓新版本
tar -xzf v4.1-deploy.tar.gz
cd v4.1

# 確認 .env 文件（如果沒有，創建一個）
cat > .env << 'EOF'
GLOBALPING_TOKEN=uh5vlg4ttg3v5gwby5zgtqrciimahql5
POSTGRES_PASSWORD=globalping_secure_password
CHECK_INTERVAL_MINUTES=90
ABNORMAL_CHECK_HOUR=1
NORMAL_CHECK_HOUR=9
MAX_ITERATIONS=10
EOF

# 啟動服務
docker-compose up -d --build

# 查看日誌
docker-compose logs -f
```

## 🔧 故障排除

### 問題 1：容器無法啟動
```bash
# 查看詳細錯誤
docker-compose logs

# 檢查端口佔用
sudo netstat -tlnp | grep 8000

# 重新構建
docker-compose down
docker-compose up -d --build --force-recreate
```

### 問題 2：API_ERROR 過多
```bash
# 檢查 Token 配額
curl -H "Authorization: Bearer uh5vlg4ttg3v5gwby5zgtqrciimahql5" \
  https://api.globalping.io/v1/limits

# 查看日誌中的錯誤
docker-compose logs web | grep "API_ERROR"
```

### 問題 3：時區不正確
```bash
# 檢查容器時區
docker-compose exec web date
docker-compose exec web cat /etc/timezone

# 如果不正確，重新構建
docker-compose down
docker-compose up -d --build
```

### 問題 4：數據庫連接失敗
```bash
# 檢查 PostgreSQL 狀態
docker-compose exec postgres pg_isready -U globalping

# 重啟數據庫
docker-compose restart postgres

# 等待 10 秒後重啟 web
sleep 10
docker-compose restart web
```
