## ✅ 改進項目

### 1. **DNS 查詢錯誤處理改進** 🛡️

**問題**：
- 當域名計劃不支援 DNS Analytics 時，會拋出錯誤並中斷處理
- 無法區分「計劃不支援」和「真正的錯誤」

**改進**：
- 新增 `SkipZone` 自定義異常類別
- 自動識別 DNS Analytics 400 錯誤（計劃不支援）
- 對於不支援的計劃，標記為 `"N/A (計劃不支援)"` 並繼續處理
- 記錄失敗原因但不中斷整個流程

**程式碼位置**：
```python
# 第 60-62 行：新增 SkipZone 異常類別
class SkipZone(Exception):
    """非致命錯誤，用於跳過不支援的域名"""
    pass

# 第 347-389 行：改進的 DNS 查詢錯誤處理
```

### 2. **批次寫入優化** 💾

**問題**：
- 原本是整個批次處理完才寫入，如果中途中斷會遺失數據
- 無法即時看到處理進度

**改進**：
- 每處理 10 筆數據就立即寫入 CSV
- 使用追加模式（append）避免數據遺失
- 顯示累計處理筆數

**程式碼位置**：
```python
# 第 679-686 行：改進的批次寫入邏輯
if len(all_results) == len(batch_results):  # 第一批
    save_results_to_csv(batch_results, results_csv, is_append=False)
    print(f"💾 已保存第一批 {len(batch_results)} 筆數據")
else:
    save_results_to_csv(batch_results, results_csv, is_append=True)
    print(f"💾 已追加 {len(batch_results)} 筆數據 (累計: {len(all_results)} 筆)")
```

## 🔧 技術細節

### SkipZone 異常處理流程

1. **檢測階段**：
   - HTTP 400 狀態碼 → 可能是計劃不支援
   - GraphQL 錯誤訊息包含 "plan not eligible" → 確認是計劃不支援

2. **處理階段**：
   - 拋出 `SkipZone` 異常（非致命）
   - 記錄失敗原因到 `failed_domains`
   - 返回 `"N/A (計劃不支援)"` 作為 DNS 查詢數值
   - 繼續處理下一個域名

3. **記錄階段**：
   - 失敗記錄包含：帳戶、域名、操作、失敗原因
   - 最終匯出到 `cloudflare_failed_domains_{timestamp}.csv`

## 🎯 使用建議

1. **監控進度**：
   - 觀察每 10 筆的寫入提示，確認數據正常保存
   - 注意 `[跳過]` 訊息，了解哪些域名不支援 DNS Analytics

2. **錯誤處理**：
   - 檢查 `cloudflare_failed_domains_{timestamp}.csv` 了解失敗原因
   - `"N/A (計劃不支援)"` 是正常情況，不需要重試

3. **效能優化**：
   - 批次間休息 20 秒，帳戶間休息 60 秒
   - 每個指標查詢間隔 1 秒，避免觸發速率限制
