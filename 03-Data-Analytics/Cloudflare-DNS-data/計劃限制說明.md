# Cloudflare DNS Analytics API 計劃限制說明

## 各計劃的查詢時間範圍限制

根據 Cloudflare 官方文檔和 API 限制，不同計劃的 DNS Analytics 查詢時間範圍如下：

### Free Website 計劃
- **時間範圍限制**：6 小時
- **說明**：這是官方明確限制，API 會返回 403 錯誤（Code: 1034）如果超過此限制
- **錯誤訊息**：`Maximum queryable time period for the free plan is 6h0m0s`

### Pro 計劃
- **時間範圍限制**：30 天
- **說明**：Pro 計劃通常可以查詢過去 30 天的 DNS Analytics 數據
- **備註**：實際限制可能因 Cloudflare 政策調整而變化

### Business 計劃
- **時間範圍限制**：30 天
- **說明**：Business 計劃通常可以查詢過去 30 天的 DNS Analytics 數據
- **備註**：實際限制可能因 Cloudflare 政策調整而變化

### Enterprise 計劃
- **時間範圍限制**：90 天（或更長）
- **說明**：Enterprise 計劃可以查詢更長時間的歷史數據，通常為 90 天
- **備註**：Enterprise 計劃可能可以自定義查詢時間範圍，請聯繫 Cloudflare 確認具體限制

## 腳本自動調整機制

腳本會根據檢測到的計劃類型自動調整查詢時間範圍：

1. **Free Website**：自動設定為 6 小時
2. **Pro**：自動設定為 30 天
3. **Business**：自動設定為 30 天
4. **Enterprise**：自動設定為 90 天
5. **未知計劃**：預設使用 7 天（保守設定）

## 注意事項

1. 如果查詢時間範圍超過計劃限制，API 會返回 403 錯誤
2. 腳本會自動處理這些錯誤，並標記為 "N/A (計劃限制)"
3. 建議定期檢查 Cloudflare 官方文檔以獲取最新的限制資訊
4. Enterprise 計劃的實際限制可能需要聯繫 Cloudflare 確認
