# Mlytics API 測試說明

## 📋 概述

`test_mlytics_api.py` 是一個用於測試 Mlytics API 連接、認證和端點的測試腳本。

## 🚀 使用方法

### 1. 執行測試腳本

```bash
cd "/Users/ckchiu/Desktop/Project/Cloudflare DNS data"
python3 test_mlytics_api.py
```

### 2. 測試內容

腳本會執行以下三個階段的測試：

#### 階段 1: 測試基礎連接和認證
- 測試 API 根端點 (`/`)
- 測試健康檢查端點 (`/health`, `/status`)
- 確認 API 基礎 URL 和連接是否正常

#### 階段 2: 測試不同的認證方式
- **X-API-Key Header**: 使用 `X-API-Key` Header 傳遞 API Key
- **Bearer Token**: 使用 `Authorization: Bearer {token}` 方式
- 找出哪種認證方式有效

#### 階段 3: 測試常見的 API 端點
測試以下常見的端點：
- `/domains` - 獲取域名列表
- `/zones` - 獲取 Zones 列表
- `/projects` - 獲取項目列表
- `/sites` - 獲取站點列表
- `/analytics` - 獲取統計數據
- `/stats` - 獲取統計資訊
- `/dns` - DNS 相關端點
- `/account` - 獲取帳戶資訊
- 等等...

## 🔍 如何解讀測試結果

### 1. 找到可用的端點

如果測試成功，您會看到：
- ✅ 成功的測試列表
- 實際的 API 響應格式（JSON 結構）

**下一步**：
1. 查看響應的 JSON 結構
2. 確認數據欄位的名稱（如 `data`, `results`, `domains` 等）
3. 根據實際響應格式調整 `mlytics_report.py` 中的解析邏輯

### 2. 認證失敗

如果所有測試都失敗，常見錯誤：

- **401 Unauthorized**: API Key 無效或認證方式不正確
  - 解決方案：檢查 API Key 是否正確，嘗試不同的認證方式

- **403 Forbidden**: API Key 有效但無權限
  - 解決方案：確認帳戶是否有權限訪問該端點

- **404 Not Found**: 端點路徑不正確
  - 解決方案：根據 Mlytics API 文檔確認正確的端點路徑

- **連接錯誤**: API Base URL 不正確或網路問題
  - 解決方案：檢查 API Base URL，確認網路連接

## 🔧 自訂測試

如果需要測試特定的端點，可以修改腳本中的 `common_endpoints` 列表：

```python
common_endpoints = [
    ("/your-custom-endpoint", "GET", None, None, "您的自訂端點描述"),
    # 添加更多端點...
]
```

## 📝 注意事項

1. **速率限制**: 腳本會在請求之間添加延遲，避免觸發 API 速率限制
2. **超時設定**: 每個請求的超時時間為 30 秒
3. **響應大小**: 為了便於閱讀，JSON 響應會被限制在 2000 字元以內
4. **API Key 安全**: 測試腳本中包含 API Key，請確保不要將測試結果分享給他人

## 🎯 測試完成後的下一步

1. **記錄成功的端點**: 記下哪些端點可以成功訪問
2. **記錄響應格式**: 保存實際的 JSON 響應結構
3. **更新主腳本**: 根據測試結果更新 `mlytics_report.py` 中的：
   - API 端點路徑
   - 響應解析邏輯
   - 認證方式（如果發現需要使用不同的認證方式）
