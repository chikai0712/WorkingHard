# Cloudflare 數據收集腳本分析報告

## 📋 腳本概述

**檔案名稱**: `cloudflare_report_Global_Key_final (1).py`  
**主要功能**: 從多個 Cloudflare 帳戶收集域名統計數據（HTTP 流量、DNS 查詢等）並匯出為 CSV

---

## 🎯 核心功能

### 1. **多帳戶支援**
- 支援同時處理多個 Cloudflare 帳戶
- 使用 Global API Key 認證（`X-Auth-Email` + `X-Auth-Key`）
- 每個帳戶獨立處理，避免混淆

### 2. **數據收集指標**
腳本會收集以下 6 種指標：
- ✅ **計劃類型** (Plan Type)
- ✅ **總請求數** (Total Requests)
- ✅ **總流量** (Total Bytes, 轉換為 MB)
- ✅ **緩存流量** (Cached Bytes, 轉換為 MB)
- ✅ **緩存請求數** (Cached Requests)
- ✅ **DNS 查詢數** (DNS Query Count)

### 3. **時間範圍處理**
- **預設查詢範圍**: 過去 30 天
- **Free 計劃限制**: 僅能查詢過去 7 天的 DNS 數據
- **自動調整**: 根據計劃類型動態調整查詢時間窗口

---

## 🔧 技術架構

### API 使用方式

#### 1. **REST API** (用於基本操作)
- `GET /zones` - 獲取域名列表
- `GET /zones/{zone_id}` - 獲取域名計劃信息

#### 2. **GraphQL API** (用於指標查詢)
- **HTTP 指標**: `httpRequests1dGroups` 查詢
  - 指標: `bytes`, `requests`, `cachedBytes`, `cachedRequests`
- **DNS 指標**: `dnsAnalyticsAdaptiveGroups` 查詢
  - 指標: DNS 查詢總數

### 批次處理機制
- **批次大小**: 每 10 個域名為一批次
- **批次間休息**: 20 秒
- **帳戶間休息**: 60 秒
- **請求間延遲**: 每個指標查詢間隔 1 秒

---

## 🛡️ 錯誤處理與重試機制

### 速率限制處理
```python
# 指數退避策略
backoff_time = initial_backoff * (2 ** (retries - 1)) + random.uniform(1, 5)
```

- **重試次數**: 最多 3 次
- **初始退避**: 10 秒（速率限制時）
- **隨機抖動**: 1-5 秒，避免同時重試

### 失敗請求處理
1. **即時失敗**: 記錄到 `failed_domains` 列表
2. **速率限制失敗**: 記錄到 `rate_limited_requests` 列表
3. **延遲重試**: 等待 2 分鐘後重新處理速率限制的請求

## 📊 輸出檔案

### 1. **主要結果檔案**
- `cloudflare_metrics_{timestamp}.csv`
  - 包含所有域名的完整數據
  - 批次追加模式（每 10 筆寫入一次）

### 2. **排序結果檔案**
- `cloudflare_all_sorted_{timestamp}.csv`
  - 按總流量降序排列
  - 所有帳戶合併後的完整結果

### 3. **失敗記錄檔案**
- `cloudflare_failed_domains_{timestamp}.csv`
  - 記錄所有處理失敗的域名及原因

## ⚠️ 潛在問題與改進建議

### 1. **安全性問題** ⚠️
- ❌ **硬編碼 API Key**: 帳戶認證信息直接寫在程式碼中
- ✅ **建議**: 改用環境變數或 `.env` 檔案（如 `cloudflare_dns_query_count.py` 的做法）

### 2. **錯誤處理**
- ⚠️ 部分錯誤訊息為中文，可能影響日誌解析
- ⚠️ 失敗請求的重試邏輯較複雜，可能導致部分數據遺失

### 3. **效能優化**
- ⚠️ 批次間休息時間較長（20-60 秒），處理大量域名時耗時較久
- ✅ 已實作批次寫入（每 10 筆），避免數據遺失

### 4. **數據完整性**
- ⚠️ DNS 查詢數使用 `dnsAnalyticsAdaptiveGroups`，可能受計劃限制影響
- ⚠️ Free 計劃只能查 7 天，可能導致數據不完整

## 📝 使用建議

### 1. **遷移到環境變數**
建議將 `accounts` 列表改為從環境變數讀取：
```python
import os
import json

accounts_json = os.getenv("CF_ACCOUNTS_JSON")
if accounts_json:
    accounts = json.loads(accounts_json)
```

### 2. **統一錯誤處理**
建議統一使用英文錯誤訊息，方便日誌分析

### 3. **合併功能**

- `cloudflare_dns_query_count.py` 專注於 DNS 查詢數
