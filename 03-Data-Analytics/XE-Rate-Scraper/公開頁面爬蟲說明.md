### 完整參數

```bash
python3 xe_table_scraper.py [選項]

選項：
  --base BASE           基準貨幣（預設：TWD）
                        例如：USD, EUR, GBP, JPY, CNY 等
  
  --date DATE           日期（格式：YYYY-MM-DD）
                        預設為今天

  --days DAYS           🆕 爬取多天並計算平均值
                        例如：--days 30 表示爬取過去30天並計算平均
                        會自動計算：平均值、最小值、最大值、標準差
  
  --end-date DATE       🆕 多天爬取的結束日期（格式：YYYY-MM-DD）
                        預設為今天
                        例如：--days 30 --end-date 2025-12-28
  
  --format {json,csv,excel,all}
                        輸出格式（預設：all）
  
  --output OUTPUT       輸出檔案名稱（不含副檔名）
```

## 🆕 30天平均匯率功能

### 為什麼需要平均匯率？

單日匯率可能受到市場波動影響，而**平均匯率**能更準確反映長期趨勢：
- 📊 **更穩定**：平滑短期波動
- 📈 **更準確**：反映真實市場價值
- 📉 **風險評估**：標準差顯示波動性
- 📅 **趨勢分析**：了解匯率變化範圍

### 使用範例

```bash
# 最常用：30天平均匯率
./run_table.sh --days 30

# 7天平均（短期趨勢）
./run_table.sh --days 7

# 90天平均（長期趨勢）
./run_table.sh --days 90 --base USD

# 指定結束日期
./run_table.sh --days 30 --end-date 2025-12-28
```

### 輸出數據說明

當使用 `--days` 參數時，輸出包含以下統計資訊：
- `average_rate`: 平均匯率
- `min_rate`: 期間最低匯率
- `max_rate`: 期間最高匯率
- `std_deviation`: 標準差（波動性指標，數值越大表示波動越大）
- `data_points`: 成功爬取的數據點數量
- `date_range_start`: 日期範圍開始
- `date_range_end`: 日期範圍結束

## ⚠️ 注意事項

1. **請求頻率**：
   - 建議不要過於頻繁地請求
   - 適當添加延遲，避免被封鎖

2. **日期格式**：
   - 必須使用 `YYYY-MM-DD` 格式
   - 例如：`2025-12-28`

3. **貨幣代碼**：
   - 使用標準的三字母貨幣代碼
   - 例如：USD, EUR, TWD, JPY 等

4. **數據更新**：
   - 頁面數據可能不是即時的
   - 通常每天更新一次
