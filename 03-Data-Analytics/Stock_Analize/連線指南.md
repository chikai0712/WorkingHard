# 🔌 連線指南

## 📍 服務端點

系統啟動後，您可以透過以下網址訪問：

| 服務 | 網址 | 說明 |
|------|------|------|
| **前端儀表板** | http://localhost:5173 | 主要的網頁介面 |
| **後端 API** | http://localhost:8000 | RESTful API 服務 |
| **API 文件** | http://localhost:8000/docs | 互動式 API 文件（Swagger） |
| **健康檢查** | http://localhost:8000/health | 檢查後端服務狀態 |

---

## 🚀 啟動步驟

### 方法一：使用啟動腳本（最簡單）

```bash
cd Stock_Analize
./start.sh
```

腳本會自動：
1. ✅ 檢查環境（Python、Node.js）
2. ✅ 安裝依賴（如果需要）
3. ✅ 啟動後端服務（端口 8000）
4. ✅ 啟動前端服務（端口 5173）

啟動成功後，訪問：**http://localhost:5173**

停止服務：
```bash
./stop.sh
```

---

### 方法二：手動啟動（兩個終端）

#### 終端 1：啟動後端

```bash
cd Stock_Analize/backend

# 建立並啟動虛擬環境（第一次）
python3 -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 安裝依賴（第一次）
pip install -r requirements.txt

# 啟動服務
python main.py
```

看到以下訊息表示後端啟動成功：
```
🚀 啟動股票資訊儀表板後端服務...
✅ 資料庫初始化完成
✅ 定時任務已啟動
INFO:     Uvicorn running on http://0.0.0.0:8000
```

#### 終端 2：啟動前端

```bash
cd Stock_Analize/frontend

# 安裝依賴（第一次）
npm install

# 啟動開發伺服器
npm run dev
```

看到以下訊息表示前端啟動成功：
```
  VITE v5.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: use --host to expose
```

## 🆘 連線問題排除

### 問題 1：無法訪問 http://localhost:5173

**可能原因**：
- 前端服務未啟動
- 端口被占用

**解決方法**：
1. 檢查前端終端是否有錯誤訊息
2. 確認端口 5173 是否被其他程式使用
3. 嘗試修改 `frontend/vite.config.js` 中的端口號

### 問題 2：前端顯示「無法連接 API」

**可能原因**：
- 後端服務未啟動
- API URL 設定錯誤
- CORS 設定問題

**解決方法**：
1. 確認後端服務正在運行（訪問 http://localhost:8000/health）
2. 檢查 `frontend/.env` 中的 `VITE_API_BASE_URL` 是否為 `http://localhost:8000/api`
3. 檢查後端 `backend/.env` 中的 `CORS_ORIGINS` 是否包含 `http://localhost:5173`

### 問題 3：後端啟動失敗

**可能原因**：
- Python 版本不符（需要 3.10+）
- 依賴未安裝
- 端口被占用

**解決方法**：
1. 檢查 Python 版本：`python3 --version`
2. 重新安裝依賴：`pip install -r requirements.txt`
3. 檢查端口 8000 是否被占用
4. 查看錯誤訊息並對應處理

### 問題 4：資料庫錯誤

**可能原因**：
- 資料庫檔案權限問題
- SQLite 版本問題

**解決方法**：
1. 檢查 `backend/stock_data.db` 的權限
2. 刪除資料庫檔案，讓系統重新建立：
   ```bash
   cd backend
   rm stock_data.db
   python main.py  # 會自動重新建立
   ```

## 📱 移動裝置訪問

如果要從同一網路的其他裝置訪問：

1. 找到電腦的 IP 位址：
   ```bash
   # macOS/Linux
   ifconfig | grep "inet "
   
   # Windows
   ipconfig
   ```

2. 啟動時加入 `--host` 參數：
   ```bash
   # 前端
   cd frontend
   npm run dev -- --host
   
   # 後端（修改 main.py 或使用環境變數）
   # HOST=0.0.0.0
   ```

3. 從移動裝置訪問：`http://<電腦IP>:5173`
