# 定時更新設定說明

## 📋 概述

Stock_Analize 系統使用 **APScheduler** 實現自動定時更新功能，可以自動抓取股票和期貨資料。

---

## ⚙️ 當前定時任務配置

### 1. 股票價格更新任務

#### 任務 1: 定期更新（交易時間內）
- **觸發方式**: 間隔觸發（Interval）
- **更新頻率**: 每 5 分鐘（可在 `.env` 設定）
- **執行時間**: 僅在交易時間內（09:00-13:30，週一到週五）
- **任務 ID**: `update_stock_prices`
- **功能**: 更新所有股票的價格資料

#### 任務 2: 開盤前更新
- **觸發方式**: Cron 觸發
- **執行時間**: 每天 08:50（週一到週五）
- **任務 ID**: `pre_market_update`
- **功能**: 開盤前預先更新股票資料

#### 任務 3: 收盤後完整更新
- **觸發方式**: Cron 觸發
- **執行時間**: 每天 15:30（週一到週五）
- **任務 ID**: `daily_update_after_close`
- **功能**: 收盤後執行完整資料更新

---

### 2. 期貨資料更新任務

#### 任務 1: 每日期貨資料更新
- **觸發方式**: Cron 觸發
- **執行時間**: 每天 15:30（週一到週五，收盤後）
- **任務 ID**: `daily_futures_update`
- **功能**: 抓取當天的期貨相關資料
  - 融資融券資料
  - 三大法人買賣超
  - 期貨每日資料
  - 選擇權資料

#### 任務 2: 清理舊資料
- **觸發方式**: Cron 觸發
- **執行時間**: 每天 02:00
- **任務 ID**: `cleanup_old_futures_data`
- **功能**: 自動刪除超過30天的舊資料

---

## 🔧 配置方式

### 方法一：使用環境變數（推薦）

在 `backend/.env` 文件中設定：

```env
# 定時任務開關（預設: True）
SCHEDULER_ENABLED=True

# 更新間隔（分鐘，預設: 5）
UPDATE_INTERVAL_MINUTES=5

# 是否僅在交易時間更新（預設: True）
TRADING_HOURS_ONLY=True
```

### 方法二：修改代碼

編輯 `backend/scheduler/tasks.py` 和 `backend/scheduler/futures_tasks.py` 來調整定時任務。

---

## 📝 詳細設定說明

### UPDATE_INTERVAL_MINUTES

控制股票價格更新的頻率。

**建議值**:
- `1` - 每分鐘更新（最即時，但可能造成 API 限制）
- `5` - 每5分鐘更新（預設，平衡即時性和穩定性）
- `10` - 每10分鐘更新（較穩定，適合測試環境）
- `15` - 每15分鐘更新（最穩定，適合低頻更新需求）

**範例**:
```env
UPDATE_INTERVAL_MINUTES=5
```

---

### TRADING_HOURS_ONLY

控制是否僅在交易時間內更新股票價格。

**設定值**:
- `True` - 僅在交易時間（09:00-13:30，週一到週五）更新
- `False` - 全天候更新（不建議，會浪費資源）

**範例**:
```env
TRADING_HOURS_ONLY=True
```

---

### SCHEDULER_ENABLED

控制是否啟用定時任務。

**設定值**:
- `True` - 啟用定時任務（預設）
- `False` - 停用定時任務（僅手動更新）

**使用場景**:
- 開發測試時可設為 `False` 避免自動更新
- 生產環境應設為 `True`

**範例**:
```env
SCHEDULER_ENABLED=True
```

## 🕐 定時任務時間表

### 週一到週五（交易日）

| 時間 | 任務 | 說明 |
|------|------|------|
| 02:00 | 清理舊資料 | 刪除超過30天的舊資料 |
| 08:50 | 開盤前更新 | 預先更新股票資料 |
| 09:00-13:30 | 定期更新 | 每 N 分鐘更新一次（依設定） |
| 15:30 | 收盤後更新 | 完整更新股票和期貨資料 |

### 週六、週日

- 不執行任何定時任務

如果需要手動觸發更新，可以使用以下方式：

```bash

curl -X POST http://localhost:8000/api/stocks/update
```

### 2. 使用 Python 腳本

```bash

```

### 3. 使用 fetch_recent_data.py

```bash
cd backend
python scripts/fetch_recent_data.py
```

## ⚠️ 注意事項

### 1. 時區設定

所有定時任務使用 **Asia/Taipei (UTC+8)** 時區。

### 2. 交易日判斷

系統會自動判斷是否為交易日（週一到週五），週末不會執行任務。

### 3. API 限制

注意資料來源的 API 限制：
- 過於頻繁的請求可能被限制
- 建議更新間隔至少 5 分鐘

### 4. 資料一致性

- 期貨資料在收盤後（15:30）才會完整更新
- 盤中可能只有部分資料可用

### 5. 錯誤處理

如果定時任務執行失敗：
- 錯誤會記錄在日誌中
- 不會影響系統運行
- 下次定時任務會繼續執行

## 🔄 修改定時任務時間

### 修改股票更新頻率

編輯 `backend/scheduler/tasks.py`:

```python
# 修改更新間隔
interval_minutes = int(os.getenv("UPDATE_INTERVAL_MINUTES", "5"))
```

### 修改期貨更新時間

編輯 `backend/scheduler/futures_tasks.py`:

```python
# 修改收盤後更新時間（例如改為 15:35）
scheduler.add_job(
    func=update_futures_data,
    trigger=CronTrigger(hour=15, minute=35, day_of_week='mon-fri', timezone=taiwan_tz),
    ...
)
```

### 修改清理時間

編輯 `backend/scheduler/futures_tasks.py`:

```python
# 修改清理時間（例如改為 03:00）
scheduler.add_job(
    func=cleanup_old_data,
    trigger=CronTrigger(hour=3, minute=0, timezone=taiwan_tz),
    ...
)
```

## 🆘 故障排除

### 問題：定時任務沒有執行

1. **檢查 SCHEDULER_ENABLED**
   ```bash
   # 確認 .env 文件中設定為 True
   grep SCHEDULER_ENABLED backend/.env
   ```

2. **檢查後端是否運行**
   ```bash
   ps aux | grep python | grep main.py
   ```

3. **查看日誌**
   ```bash
   tail -f logs/backend.log
   ```

### 問題：更新頻率不對

1. **檢查 UPDATE_INTERVAL_MINUTES 設定**
   ```bash
   grep UPDATE_INTERVAL_MINUTES backend/.env
   ```

2. **重啟後端服務**
   ```bash
   ./stop.sh
   ./start.sh
   ```

### 問題：期貨資料沒有更新

1. **確認執行時間**
   - 期貨資料在收盤後（15:30）才會更新
   - 盤中不會自動更新

2. **手動觸發測試**
   ```bash
   cd backend
   python scripts/fetch_recent_data.py
   ```
