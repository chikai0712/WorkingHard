# 日誌說明

## 📋 日誌位置

### 1. 服務日誌（使用 start.sh 啟動時）

當使用 `./start.sh` 啟動服務時，日誌會保存在：

```
Stock_Analize/logs/
├── backend.log      # 後端服務日誌（包含定時任務日誌）
├── frontend.log     # 前端服務日誌
├── backend.pid      # 後端進程 ID
└── frontend.pid     # 前端進程 ID
```

**查看日誌**:
```bash
# 查看後端日誌（實時）
tail -f logs/backend.log

# 查看後端日誌（最近100行）
tail -n 100 logs/backend.log

# 查看前端日誌
tail -f logs/frontend.log

# 查看所有日誌
tail -f logs/*.log
```

---

### 2. 手動更新腳本日誌

手動執行更新腳本時，輸出會直接顯示在終端機上，不會自動保存到文件。

**保存日誌到文件**:

```bash
# 方法一：重定向輸出
cd backend
source venv/bin/activate
python scripts/update_all.py > ../logs/update_$(date +%Y%m%d_%H%M%S).log 2>&1
deactivate

# 方法二：同時顯示和保存
python scripts/update_all.py | tee ../logs/update_$(date +%Y%m%d_%H%M%S).log

# 方法三：使用腳本（自動保存）
./update.sh | tee ../logs/update_$(date +%Y%m%d_%H%M%S).log
```

---

### 3. 定時任務日誌

定時任務的日誌會記錄在 `logs/backend.log` 中（如果使用 start.sh 啟動服務）。

**查看定時任務日誌**:
```bash
# 查看所有定時任務相關日誌
grep "定時任務" logs/backend.log

# 查看最近的定時任務日誌
tail -f logs/backend.log | grep "定時任務"

# 查看今天的定時任務日誌
grep "$(date +%Y-%m-%d)" logs/backend.log | grep "定時任務"
```

## 🔍 查看日誌的方式

### 即時查看日誌（推薦）

```bash
# 查看後端日誌（實時更新）
tail -f logs/backend.log

# 查看包含錯誤的日誌
tail -f logs/backend.log | grep -i error

# 查看包含警告的日誌
tail -f logs/backend.log | grep -i warning
```

### 搜索日誌

```bash
# 搜索關鍵字
grep "更新股票" logs/backend.log

# 搜索錯誤
grep -i "error\|失敗\|exception" logs/backend.log

# 搜索特定日期
grep "2025-12-22" logs/backend.log

# 搜索最近的更新
grep "更新" logs/backend.log | tail -20
```

### 查看日誌統計

```bash
# 統計錯誤數量
grep -i "error\|失敗" logs/backend.log | wc -l

# 查看最新的錯誤
grep -i "error\|失敗" logs/backend.log | tail -10

# 查看更新成功的記錄
grep "更新完成" logs/backend.log | tail -20
```

## 📝 日誌內容說明

### 後端日誌 (backend.log)

包含：
- ✅ 服務啟動/關閉訊息
- ✅ 定時任務執行記錄
- ✅ API 請求記錄
- ✅ 資料更新記錄
- ✅ 錯誤和異常訊息

**範例**:
```
🚀 啟動股票資訊儀表板後端服務...
✅ 資料庫初始化完成
✅ 定時任務已啟動
🔄 開始執行定時任務：更新股票價格 - 2025-12-22 16:43:09
✅ 更新完成：成功 10，失敗 0
```

### 前端日誌 (frontend.log)

包含：
- ✅ 前端服務啟動訊息
- ✅ 建置錯誤和警告
- ✅ 開發伺服器訊息

## ⚠️ 常見問題

### 問題 1: 找不到 logs 目錄

**原因**: 沒有使用 `start.sh` 啟動服務，或目錄未創建

**解決方法**:
```bash
# 手動創建 logs 目錄
mkdir -p logs

# 或使用 start.sh 啟動服務（會自動創建）
./start.sh
```

---

### 問題 2: 日誌文件太大

**原因**: 長時間運行，日誌累積過多

**解決方法**:
```bash
# 清理舊日誌（保留最近1000行）
tail -n 1000 logs/backend.log > logs/backend.log.tmp
mv logs/backend.log.tmp logs/backend.log

# 或使用 logrotate（Linux）
# 配置 logrotate 定期輪轉日誌文件
```

---

### 問題 3: 手動更新沒有日誌

**原因**: 手動執行腳本時，輸出直接在終端顯示

**解決方法**: 使用重定向保存日誌（見上方「保存日誌到文件」）

---

### 問題 4: 找不到錯誤訊息

**解決方法**:
```bash
# 搜索所有錯誤相關關鍵字
grep -i "error\|exception\|失敗\|failed\|traceback" logs/backend.log

# 查看最後的錯誤
tail -100 logs/backend.log | grep -i error

# 查看完整的錯誤堆疊
grep -A 20 "Traceback" logs/backend.log
```

---

## 💡 建議

1. **定期檢查日誌**: 每天檢查一次，確認系統正常運行
2. **保存重要日誌**: 更新失敗時，保存相關日誌以便排查
3. **設定日誌輪轉**: 避免日誌文件過大（生產環境）
4. **監控錯誤**: 設定自動監控錯誤日誌（生產環境）
