# 部署指南

## 📦 生產環境部署

### 系統需求

- **作業系統**：Linux (Ubuntu 20.04+ 推薦)
- **Python**：3.10+
- **Node.js**：16+
- **資料庫**：PostgreSQL 12+（推薦）或 SQLite（開發/小型部署）
- **Web 伺服器**：Nginx（推薦）

---

## 🔧 後端部署

### 1. 安裝系統依賴

```bash
sudo apt update
sudo apt install -y python3-pip python3-venv nginx postgresql
```

### 2. 設置資料庫

```bash
# 建立資料庫和用戶
sudo -u postgres psql
```

```sql
CREATE DATABASE stock_db;
CREATE USER stock_user WITH PASSWORD 'your_secure_password';
GRANT ALL PRIVILEGES ON DATABASE stock_db TO stock_user;
\q
```

### 3. 設置應用程式

```bash
# 複製專案到伺服器
cd /opt
git clone <repository-url> stock-dashboard
cd stock-dashboard/Stock_Analize/backend

# 建立虛擬環境
python3 -m venv venv
source venv/bin/activate

# 安裝依賴
pip install -r requirements.txt

# 設置環境變數
cp .env.example .env
nano .env
```

更新 `.env` 檔案：
```env
DATABASE_URL=postgresql://stock_user:your_secure_password@localhost:5432/stock_db
DEBUG=False
HOST=127.0.0.1
PORT=8000
```

### 4. 初始化資料庫

```bash
python -c "from database.session import init_db; init_db()"
```

### 5. 使用 systemd 管理服務

建立服務檔案 `/etc/systemd/system/stock-api.service`：

```ini
[Unit]
Description=Stock Dashboard API
After=network.target postgresql.service

[Service]
Type=simple
User=www-data
WorkingDirectory=/opt/stock-dashboard/Stock_Analize/backend
Environment="PATH=/opt/stock-dashboard/Stock_Analize/backend/venv/bin"
ExecStart=/opt/stock-dashboard/Stock_Analize/backend/venv/bin/uvicorn main:app --host 127.0.0.1 --port 8000
Restart=always

[Install]
WantedBy=multi-user.target
```

啟動服務：
```bash
sudo systemctl daemon-reload
sudo systemctl enable stock-api
sudo systemctl start stock-api
sudo systemctl status stock-api
```

## 🎨 前端部署

### 1. 建置前端

```bash
cd Stock_Analize/frontend
npm install
npm run build
```

建置產物會在 `dist/` 目錄。

### 2. 設置 Nginx

建立 Nginx 配置檔案 `/etc/nginx/sites-available/stock-dashboard`：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端靜態檔案
    root /opt/stock-dashboard/Stock_Analize/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

啟用配置：
```bash
sudo ln -s /etc/nginx/sites-available/stock-dashboard /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 3. 設置 SSL（可選，使用 Let's Encrypt）

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com
```

## 📊 監控與日誌

### 查看後端日誌

```bash
sudo journalctl -u stock-api -f
```

### 查看 Nginx 日誌

```bash
sudo tail -f /var/log/nginx/access.log
sudo tail -f /var/log/nginx/error.log
```

### 設定日誌輪轉

建立 `/etc/logrotate.d/stock-api`：

```
/opt/stock-dashboard/Stock_Analize/backend/logs/*.log {
    daily
    missingok
    rotate 14
    compress
    delaycompress
    notifempty
    create 0640 www-data www-data
    sharedscripts
}
```

## 🔒 安全建議

1. **防火牆設定**
   ```bash
   sudo ufw allow 22/tcp
   sudo ufw allow 80/tcp
   sudo ufw allow 443/tcp
   sudo ufw enable
   ```

2. **資料庫安全**
   - 使用強密碼
   - 限制資料庫只能從 localhost 連接
   - 定期備份資料庫

3. **應用程式安全**
   - 保持依賴套件更新
   - 使用環境變數管理敏感資訊
   - 啟用 HTTPS

4. **系統安全**
   - 定期更新系統
   - 使用非 root 用戶運行應用程式
   - 設定適當的檔案權限

## 💾 備份

### 資料庫備份

```bash
# 建立備份腳本
cat > /opt/stock-dashboard/backup.sh << 'EOF'
#!/bin/bash
BACKUP_DIR="/opt/backups/stock-dashboard"
DATE=$(date +%Y%m%d_%H%M%S)
mkdir -p $BACKUP_DIR

# 備份資料庫
pg_dump -U stock_user stock_db > $BACKUP_DIR/db_$DATE.sql

# 保留最近 30 天的備份
find $BACKUP_DIR -name "db_*.sql" -mtime +30 -delete
EOF

chmod +x /opt/stock-dashboard/backup.sh

# 設定定時備份（每天 2:00）
echo "0 2 * * * /opt/stock-dashboard/backup.sh" | sudo crontab -
```
