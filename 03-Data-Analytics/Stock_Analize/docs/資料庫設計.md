## 資料表設計

### 1. stocks（股票基本資料表）

儲存監控股票的基本資訊。

```sql
CREATE TABLE stocks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    symbol VARCHAR(20) NOT NULL UNIQUE,
    market VARCHAR(10) NOT NULL,  -- 'TW', 'US', etc.
    name VARCHAR(100) NOT NULL,
    full_name VARCHAR(200),
    industry VARCHAR(50),
    sector VARCHAR(50),
    website VARCHAR(255),
    description TEXT,
    currency VARCHAR(10) DEFAULT 'TWD',
    is_active BOOLEAN DEFAULT 1,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_stocks_symbol ON stocks(symbol);
CREATE INDEX idx_stocks_market ON stocks(market);
CREATE INDEX idx_stocks_active ON stocks(is_active);
```

### 2. stock_prices（股票價格資料表）

儲存每日的開高低收價格與成交量。

```sql
CREATE TABLE stock_prices (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    stock_id INTEGER NOT NULL,
    date DATE NOT NULL,
    open DECIMAL(10, 2) NOT NULL,
    high DECIMAL(10, 2) NOT NULL,
    low DECIMAL(10, 2) NOT NULL,
    close DECIMAL(10, 2) NOT NULL,
    volume BIGINT DEFAULT 0,
    turnover DECIMAL(20, 2),  -- 成交額
    change_amount DECIMAL(10, 2),  -- 漲跌金額
    change_percent DECIMAL(5, 2),  -- 漲跌幅百分比
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (stock_id) REFERENCES stocks(id),
    UNIQUE(stock_id, date)
);

CREATE INDEX idx_stock_prices_stock_id ON stock_prices(stock_id);
CREATE INDEX idx_stock_prices_date ON stock_prices(date);
CREATE INDEX idx_stock_prices_stock_date ON stock_prices(stock_id, date);
```

### 3. technical_indicators（技術指標表）

儲存計算出的技術指標。

```sql
CREATE TABLE technical_indicators (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    stock_id INTEGER NOT NULL,
    date DATE NOT NULL,
    indicator_type VARCHAR(20) NOT NULL,  -- 'MA', 'RSI', 'MACD', etc.
    indicator_name VARCHAR(50) NOT NULL,  -- 'MA5', 'RSI14', etc.
    value DECIMAL(10, 4) NOT NULL,
    period INTEGER,  -- 計算週期，例如 MA5 的 5
    extra_data JSON,  -- 額外資料（如 MACD 的 signal, hist）
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (stock_id) REFERENCES stocks(id),
    UNIQUE(stock_id, date, indicator_type, indicator_name)
);

CREATE INDEX idx_technical_indicators_stock_id ON technical_indicators(stock_id);
CREATE INDEX idx_technical_indicators_date ON technical_indicators(date);
CREATE INDEX idx_technical_indicators_type ON technical_indicators(indicator_type);
```

### 4. stock_news（股票新聞表）

儲存股票相關新聞（可選功能）。

```sql
CREATE TABLE stock_news (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    stock_id INTEGER,
    title VARCHAR(255) NOT NULL,
    content TEXT,
    source VARCHAR(100),
    url VARCHAR(500),
    published_at DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (stock_id) REFERENCES stocks(id)
);

CREATE INDEX idx_stock_news_stock_id ON stock_news(stock_id);
CREATE INDEX idx_stock_news_published_at ON stock_news(published_at);
```

### 5. data_sync_log（資料同步日誌表）

記錄資料抓取的日誌。

```sql
CREATE TABLE data_sync_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    stock_id INTEGER,
    sync_type VARCHAR(20) NOT NULL,  -- 'price', 'indicator', etc.
    status VARCHAR(20) NOT NULL,  -- 'success', 'failed', 'partial'
    records_count INTEGER DEFAULT 0,
    error_message TEXT,
    started_at DATETIME NOT NULL,
    completed_at DATETIME,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (stock_id) REFERENCES stocks(id)
);

CREATE INDEX idx_data_sync_log_stock_id ON data_sync_log(stock_id);
CREATE INDEX idx_data_sync_log_status ON data_sync_log(status);
CREATE INDEX idx_data_sync_log_started_at ON data_sync_log(started_at);
```

## 效能優化建議

### 1. 索引優化
- 已在常用查詢欄位建立索引
- 定期檢查索引使用情況

### 2. 分表策略
當資料量過大時，可考慮按年份分表：
- `stock_prices_2024`
- `stock_prices_2025`
- ...

### 3. 資料歸檔
- 歸檔超過 2 年的舊資料
- 保留摘要統計資料

### 4. 查詢優化
- 使用適當的 WHERE 條件
- 避免 SELECT *
- 使用 LIMIT 限制結果數量

## 資料完整性

### 外鍵約束
- 所有外鍵都已設定
- 確保資料一致性

### 唯一性約束
- `stocks.symbol` - 唯一
- `stock_prices(stock_id, date)` - 唯一組合
- `technical_indicators(stock_id, date, indicator_type, indicator_name)` - 唯一組合

### 檢查約束
建議在應用層驗證：
- 價格必須 > 0
- 日期必須在合理範圍內
- 百分比必須在 -100 到 +100 之間
