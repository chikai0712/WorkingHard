# 每小時更新設定說明

## 📋 概述

為了滿足「電腦不會隨時開著」但仍需要定期更新的需求，系統已設定為**每小時自動更新期貨資料**。

---

## ⚙️ 運作方式

### 1. 自動更新機制

- **更新頻率**：每小時執行一次
- **更新內容**：期貨相關資料（融資融券、三大法人、未平倉量等）
- **執行條件**：只要後端服務運行中，就會每小時自動執行

### 2. 電腦開關機處理

#### ✅ 當電腦開機時：
- 後端服務啟動後，定時任務會自動開始運作
- 從啟動時間開始，每小時執行一次更新
- **不需要手動操作**

#### ⏸️ 當電腦關機時：
- 後端服務停止，定時任務暫停
- **錯過的更新不會執行**（這是正常的）
- 不會造成任何問題

#### 🔄 當電腦重新開機時：
- 後端服務重新啟動後，定時任務會繼續運作
- 從重新啟動時間開始，每小時執行一次更新

---

## 🔧 設定方法

### 方法一：使用環境變數（推薦）

在 `backend/.env` 文件中設定：

```env
# 是否啟用每小時更新（預設: True）
HOURLY_FUTURES_UPDATE=True
```

**設定值說明**：
- `True` - 每小時更新一次（適合電腦不會隨時開著的情況）
- `False` - 只在每天收盤後更新一次（適合電腦一直開著的情況）

### 方法二：停用每小時更新

如果想回到原來的每天更新一次模式，設定：

```env
HOURLY_FUTURES_UPDATE=False
```

## 🚀 自動啟動設定（可選）

為了讓電腦開機時自動啟動後端服務，可以使用以下方法：

### macOS 使用 LaunchAgent

1. **創建 plist 文件**：

```bash
# 創建目錄（如果不存在）
mkdir -p ~/Library/LaunchAgents

# 創建 plist 文件
cat > ~/Library/LaunchAgents/com.stockanalize.backend.plist << 'EOF'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>Label</key>
    <string>com.stockanalize.backend</string>
    <key>ProgramArguments</key>
    <array>
        <string>/bin/bash</string>
        <string>-c</string>
        <string>cd /Users/ckchiu/Desktop/Project/Stock_Analize && ./start.sh</string>
    </array>
    <key>RunAtLoad</key>
    <true/>
    <key>KeepAlive</key>
    <false/>
    <key>StandardOutPath</key>
    <string>/Users/ckchiu/Desktop/Project/Stock_Analize/logs/launchd.log</string>
    <key>StandardErrorPath</key>
    <string>/Users/ckchiu/Desktop/Project/Stock_Analize/logs/launchd.error.log</string>
</dict>
</plist>
EOF
```

2. **載入服務**：

```bash
launchctl load ~/Library/LaunchAgents/com.stockanalize.backend.plist
```

3. **啟動服務**：

```bash
launchctl start com.stockanalize.backend
```

4. **檢查狀態**：

```bash
launchctl list | grep stockanalize
```

5. **如果需要停止**：

```bash
launchctl unload ~/Library/LaunchAgents/com.stockanalize.backend.plist
```

### Linux 使用 systemd

1. **創建 service 文件**：

```bash
sudo cat > /etc/systemd/system/stockanalize-backend.service << 'EOF'
[Unit]
Description=Stock Analize Backend Service
After=network.target

[Service]
Type=simple
User=your_username
WorkingDirectory=/path/to/Stock_Analize
ExecStart=/bin/bash start.sh
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF
```

2. **啟用並啟動服務**：

```bash
sudo systemctl daemon-reload
sudo systemctl enable stockanalize-backend.service
sudo systemctl start stockanalize-backend.service
```

3. **檢查狀態**：

```bash
sudo systemctl status stockanalize-backend.service
```

### Windows 使用工作排程器

1. 開啟「工作排程器」
2. 建立基本工作
3. 觸發程序：當電腦啟動時
4. 動作：啟動程式
5. 程式或指令碼：`C:\path\to\Stock_Analize\start.bat`

## 📊 更新時間表

### 每小時更新模式（HOURLY_FUTURES_UPDATE=True）

| 時間 | 說明 |
|------|------|
| 每小時整點 | 更新期貨資料 |
| 02:00 | 清理超過30天的舊資料 |

**範例**：
- 如果電腦在 09:15 開機，後端服務啟動
- 第一次更新：10:00
- 第二次更新：11:00
- 第三次更新：12:00
- ...以此類推

### 每日更新模式（HOURLY_FUTURES_UPDATE=False）

| 時間 | 說明 |
|------|------|
| 15:30（交易日） | 更新期貨資料（收盤後） |
| 02:00 | 清理超過30天的舊資料 |

## ⚠️ 注意事項

### 1. 電腦關機時

- **錯過的更新不會執行**：這是正常的行為
- **不會影響系統運作**：當電腦重新開機後，更新會繼續進行
- **資料不會丟失**：已經儲存的資料會保留

### 2. 資料完整性

- 如果電腦在交易日中間關機，可能會錯過某些時間點的資料
- 這是可以接受的，因為期貨資料主要在收盤後才會完整
- 建議在交易時間內保持電腦開機，或至少在收盤後（15:30）保持開機

### 3. 資源使用

- 每小時更新一次會產生一定的系統負載
- 如果電腦效能較低，可以考慮改為每 2 小時或更長時間更新一次
- 修改方式：編輯 `backend/scheduler/futures_tasks.py`，將 `IntervalTrigger(hours=1)` 改為 `IntervalTrigger(hours=2)`

---

## 🛠️ 自訂更新頻率

如果想設定為其他更新頻率（例如每 2 小時、每 30 分鐘），可以修改 `backend/scheduler/futures_tasks.py`：

```python
# 每 2 小時更新一次
scheduler.add_job(
    func=update_futures_data,
    trigger=IntervalTrigger(hours=2),
    ...
)

# 每 30 分鐘更新一次
scheduler.add_job(
    func=update_futures_data,
    trigger=IntervalTrigger(minutes=30),
    ...
)
```
