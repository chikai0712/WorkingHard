# 借券賣出資料問題說明

## 問題確認

**是的，是因為沒有抓取借券賣出的資料，所以才顯示為 0。**

## 詳細分析

### 1. 代碼狀態

```python
# backend/crawler/twse_api.py
result = {
    'securities_lending_sell': 0,      # ← 初始化為 0
    'securities_lending_change': 0,    # ← 初始化為 0
    # ... 沒有邏輯填充這兩個欄位
}
```

**結論**：代碼中確實沒有實現借券賣出資料的抓取邏輯。

### 2. API 限制

**TWSE 融資融券 API** (`/marginTrading/MI_MARGN`)
- ✅ 提供：融資餘額、融券餘額
- ❌ **不提供**：借券賣出餘額

**TWSE 借券 API** (`/lending/t13sa710`)
- ✅ 提供：歷史借券成交明細
- ❌ **不提供**：借券賣出餘額（需要從明細計算）

### 3. 資料庫狀態

```
securities_lending_sell: 0
securities_lending_change: 0
```

這兩個欄位都是 0，因為沒有資料來源。

---

## 解決方案

### 方案 1: 從成交明細計算餘額（複雜）

需要：
1. 抓取所有借券成交明細
2. 累加借券數量
3. 減去還券數量
4. 計算餘額和增減

**優點**：可以取得資料  
**缺點**：複雜度高，需要處理大量明細資料

### 方案 2: 保持現狀（建議）

**原因**：
- TWSE 官方 API 不直接提供借券賣出餘額
- 從明細計算餘額複雜且可能有誤差
- 借券賣出餘額可能不是核心資料

**做法**：
- 在前端標註「資料來源限制」
- 或隱藏該欄位

---

## 建議

**暫時保持現狀**，因為：
1. TWSE 官方 API 不提供借券賣出餘額
2. 如果需要這個資料，需要從成交明細計算，複雜度高
3. 目前融資、融券資料已完整，借券賣出可能不是必須

如果需要實現，可以：
- 添加說明標註
- 或考慮未來實現從明細計算的邏輯
