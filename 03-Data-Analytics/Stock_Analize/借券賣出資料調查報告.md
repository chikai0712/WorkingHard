# 借券賣出資料調查報告

## 問題分析

用戶詢問：「是因為沒有抓前面的資料所以才沒顯示嗎？」

## 調查結果

### 1. 代碼檢查結果

從 `backend/crawler/twse_api.py` 的代碼可以看到：

```python
result = {
    'date': target_date,
    'market_type': 'weighted',
    'margin_balance': 0,
    'margin_change': 0,
    'short_selling_balance': 0,
    'short_selling_change': 0,
    'securities_lending_sell': 0,      # ← 初始化為 0
    'securities_lending_change': 0,    # ← 初始化為 0
    'index_price': 0,
    'index_change_percent': 0
}
```

**發現**：
- ✅ `securities_lending_sell` 和 `securities_lending_change` 在代碼中**初始化為 0**
- ❌ **沒有任何邏輯去填充這兩個欄位**
- ❌ 代碼中只處理了融資和融券資料，完全沒有處理借券賣出

### 2. API 回應檢查結果

從 TWSE 融資融券 API (`/marginTrading/MI_MARGN`) 的回應結構：

**Table 1: 信用交易統計**
- 欄位：['項目', '買進', '賣出', '現金(券)償還', '前日餘額', '今日餘額']
- 資料：只有 3 筆
  - 融資(交易單位)
  - 融券(交易單位)
  - 融資金額(仟元)
- **沒有借券相關的資料**

### 3. 資料庫檢查結果

```
securities_lending_sell: 0
securities_lending_change: 0
```

這兩個欄位都是 0，因為代碼中沒有填充邏輯。

---

## 結論

**是的，是因為沒有抓取借券賣出的資料，所以顯示為 0。**

### 原因

1. **API 限制**：TWSE 融資融券 API (`/marginTrading/MI_MARGN`) **不包含借券賣出資料**
2. **代碼缺失**：代碼中沒有實現借券賣出資料的抓取邏輯
3. **資料來源**：借券賣出可能需要從其他 API 取得（如 `/lending/t13sa710`）

---

## 解決方案

### 方案 1: 從借券 API 取得資料

需要：
1. 測試 `/lending/t13sa710` API 的完整回應結構
2. 檢查是否有借券賣出餘額資料
3. 如果有，實現抓取和解析邏輯
4. 整合到現有的資料抓取流程

### 方案 2: 保持現狀

如果：
- TWSE 官方確實不提供借券賣出餘額 API
- 或資料來源確實無法取得

則可以：
- 在前端標註「資料來源限制」
- 或隱藏該欄位

---

## 建議

1. **先測試借券 API**
   - 完整測試 `/lending/t13sa710` API
   - 確認是否有餘額資料

2. **如果 API 有資料**
   - 實現抓取邏輯
   - 整合到 `twse_api.py`

3. **如果 API 沒有資料**
   - 在前端標註說明
   - 或考慮從其他資料來源取得
