# 前端測試指南

## 啟動服務

### 1. 啟動後端服務

```bash
cd Stock_Analize/backend
source venv/bin/activate
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

後端服務應該在 `http://localhost:8000` 運行

### 2. 啟動前端服務

```bash
cd Stock_Analize/frontend
npm install  # 如果還沒安裝依賴
npm run dev
```

前端服務應該在 `http://localhost:5173` 運行（或 Vite 顯示的端口）

---

## 測試步驟

### 1. 確認後端 API 正常

訪問以下 URL 確認 API 正常運作：

```bash
# 測試三大法人 API
curl "http://localhost:8000/api/futures/institutional?market_type=weighted&limit=7"

# 測試融資融券 API
curl "http://localhost:8000/api/futures/trading?market_type=weighted&limit=7"
```

預期回應：
- 狀態碼：200
- 包含 `status: "success"`
- 包含 `data` 陣列，至少有 1 筆資料

### 2. 訪問前端頁面

在瀏覽器中訪問：
- 主頁：`http://localhost:5173/`
- 期貨儀表板：`http://localhost:5173/futures`

### 3. 檢查前端顯示

在期貨儀表板頁面應該看到：

1. **三大法人買賣超卡片**
   - 顯示最新日期的三大法人資料
   - 外資、投信、自營商的買進、賣出、淨買賣超

2. **三大法人歷史表格**
   - 顯示最近 7 天的歷史資料
   - 包含外資、投信、自營商的淨買賣超

3. **融資融券餘額卡片**
   - 顯示最新日期的融資融券資料
   - 融資餘額、融券餘額、借券賣出

4. **融資融券歷史表格**
   - 顯示最近 7 天的歷史資料

## 可能的問題和解決方案

### 問題 1: API 回傳 404

**原因**：後端服務未啟動或路由錯誤

**解決**：
- 確認後端服務正在運行
- 檢查 `main.py` 中的路由配置
- 確認 API 路徑是否正確

### 問題 2: CORS 錯誤

**原因**：前端和後端的跨域設定問題

**解決**：
- 檢查 `main.py` 中的 CORS 設定
- 確認允許的來源包含前端 URL

### 問題 3: 資料顯示為空

**原因**：資料庫中沒有資料

**解決**：
- 執行資料抓取腳本：`python backend/scripts/fetch_recent_data.py`
- 確認資料庫中有資料
- 檢查 API 回應是否包含資料

### 問題 4: 前端顯示錯誤

**原因**：前端元件問題或 API 回應格式不符

**解決**：
- 檢查瀏覽器控制台的錯誤訊息
- 確認 API 回應格式是否符合前端預期
- 檢查 `futuresApi.js` 中的 API 端點是否正確
