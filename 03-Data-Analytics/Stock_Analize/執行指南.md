# 執行指南 - 每小時自動更新

## 🚀 快速開始

### 第一步：確認設定（可選）

如果要啟用每小時更新（預設已啟用），檢查 `backend/.env` 文件：

```bash
cd Stock_Analize/backend
# 如果沒有 .env 文件，可以創建一個
cat .env | grep HOURLY_FUTURES_UPDATE
```

如果沒有設定，系統預設會每小時更新。

### 第二步：啟動系統

在專案根目錄執行：

```bash
cd Stock_Analize
./start.sh
```

### 第三步：查看啟動日誌

啟動後，應該會看到：

```
✅ 期貨資料定時任務已設定
   - 每小時更新期貨資料
   - 每日 02:00 清理超過30天的舊資料
```

## 📋 執行步驟詳解

### 1. 啟動後端服務

後端服務會自動啟動定時任務。查看日誌確認：

```bash
# 查看後端日誌
tail -f logs/backend.log
```

應該會看到類似以下的訊息：

```
🚀 啟動股票資訊儀表板後端服務...
✅ 資料庫初始化完成
✅ 定時任務已啟動
✅ 期貨資料定時任務已設定
   - 每小時更新期貨資料
   - 每日 02:00 清理超過30天的舊資料
✅ 後端服務已啟動在 http://localhost:8000
```

### 2. 確認定時任務運行

#### 方法一：查看日誌

```bash
# 查看實時日誌
tail -f logs/backend.log | grep "更新期貨資料"
```

#### 方法二：檢查資料庫

```bash
cd backend
source venv/bin/activate
python -c "
from database.session import SessionLocal
from database.models_futures import FuturesOpenInterest
from datetime import datetime, timedelta

db = SessionLocal()
try:
    latest = db.query(FuturesOpenInterest).order_by(FuturesOpenInterest.updated_at.desc()).first()
    if latest:
        time_diff = datetime.utcnow() - latest.updated_at.replace(tzinfo=None)
        print(f'最新資料日期: {latest.date}')
        print(f'最後更新: {latest.updated_at}')
        print(f'距離現在: {time_diff}')
    else:
        print('資料庫中沒有資料')
finally:
    db.close()
"
```

### 3. 等待首次更新

- 系統啟動後，會在**下一個整點**執行第一次更新
- 例如：如果 09:15 啟動，第一次更新會在 10:00
- 之後每小時整點執行一次

```bash

```

應該會看到類似：

```

✅ 期貨資料更新完成：成功 3/4
```

### 方法二：檢查資料庫最新資料

```bash
cd backend
source venv/bin/activate
python -c "
from database.session import SessionLocal
from database.models_futures import FuturesOpenInterest
from datetime import date

db = SessionLocal()
try:
    latest = db.query(FuturesOpenInterest).filter(
        FuturesOpenInterest.date == date.today()
    ).first()
    
    if latest:
        print(f'✅ 今日資料已更新')
        print(f'   日期: {latest.date}')
        print(f'   外資未平倉: {latest.foreign_oi:,}')
        print(f'   投信未平倉: {latest.trust_oi:,}')
        print(f'   自營未平倉: {latest.dealer_oi:,}')
        print(f'   最後更新: {latest.updated_at}')
    else:
        print('❌ 今日資料尚未更新')
finally:
    db.close()
"
```

## ⏰ 更新時間表

### 每小時更新

| 時間 | 說明 |
|------|------|
| 每小時整點（00分） | 更新期貨資料 |
| 例如：10:00, 11:00, 12:00... | 自動執行 |

### 其他定時任務

| 時間 | 任務 |
|------|------|
| 02:00（每天） | 清理超過30天的舊資料 |
| 08:50（交易日） | 開盤前更新股票資料 |
| 09:00-13:30（交易日） | 每 N 分鐘更新股票價格 |
| 15:30（交易日） | 收盤後完整更新 |

## 🔧 常見問題

### Q1: 啟動後沒有看到更新記錄？

**A**: 等待下一個整點。系統會在整點執行更新。

**驗證方法**：
```bash
# 查看定時任務是否已設定
tail -50 logs/backend.log | grep "定時任務已設定"
```

### Q2: 如何確認定時任務正在運行？

**A**: 查看後端啟動日誌，應該會看到：

```
✅ 期貨資料定時任務已設定
   - 每小時更新期貨資料
```

### Q3: 電腦關機後重新開機，定時任務會繼續嗎？

**A**: 會的。重新啟動 `./start.sh` 後，定時任務會自動恢復。

### Q4: 如何改變更新頻率（例如每 2 小時）？

**A**: 編輯 `backend/scheduler/futures_tasks.py`：

```python
# 將 hours=1 改為 hours=2
trigger=IntervalTrigger(hours=2),
```

然後重啟服務。

### Q5: 如何停用每小時更新，改回每天更新一次？

**A**: 在 `backend/.env` 中設定：

```env
HOURLY_FUTURES_UPDATE=False
```

然後重啟服務。

## 🛑 停止服務

當不需要運行時：

```bash
./stop.sh
```

或者手動停止：

```bash
# 查看 PID
cat logs/backend.pid
cat logs/frontend.pid

# 停止進程
kill $(cat logs/backend.pid)
kill $(cat logs/frontend.pid)
```
