# V4.1 AWS 部署指南

## 🚀 快速部署

### 方法 1：使用更新腳本（推薦）

```bash
# 1. 添加執行權限
chmod +x AWS-deploy/update-v4.1.sh

# 2. 執行更新
cd ~/Desktop/Project
./AWS-deploy/update-v4.1.sh
```

### 方法 2：手動指定 IP（如果 AWS CLI 有問題）

```bash
# 設置 IP 環境變數
export AWS_IP=54.238.247.106

# 執行更新
./AWS-deploy/update-v4.1.sh
```

### 方法 3：完全手動部署

```bash
# 1. 打包代碼
cd ~/Desktop/Project/GlobalpingChecker
tar -czf v4.1-update.tar.gz v4.1/

# 2. 上傳到 AWS
scp -i ~/.ssh/globalping-checker-key.pem v4.1-update.tar.gz ubuntu@54.238.247.106:~/

# 3. SSH 到 AWS
ssh -i ~/.ssh/globalping-checker-key.pem ubuntu@54.238.247.106

# 4. 在 AWS 上執行以下命令：

# 停止服務
cd ~/v4.1
docker-compose down

# 備份
cd ~
cp v4.1/.env /tmp/v4.1.env.backup
cp v4.1/domains.txt /tmp/v4.1.domains.backup
mv v4.1 v4.1-backup-$(date +%Y%m%d)

# 解壓新版本
tar -xzf v4.1-update.tar.gz
cd v4.1

# 恢復配置
cp /tmp/v4.1.env.backup .env
cp /tmp/v4.1.domains.backup domains.txt

# 啟動服務
docker-compose up -d --build

# 查看日誌
docker-compose logs -f
```

## ✅ 更新內容

1. **API_ERROR 處理**
   - 遇到 HTTP 429/500+ 時自動等待 60 分鐘
   - 記錄暫停和恢復時間

2. **時區設置**
   - 系統時區：GMT+8 (Asia/Taipei)
   - PostgreSQL 時區：GMT+8
   - 檢測時間：AM 1:00 (異常區), AM 9:00 (正常區)

3. **智能循環檢測**
   - 自動區域管理
   - 每 90 分鐘檢測一次
   - 最多 10 次迭代

## 🔧 故障排除

### SSH 連線失敗
```bash
# 檢查密鑰權限
chmod 400 ~/.ssh/globalping-checker-key.pem

# 測試連線
ssh -i ~/.ssh/globalping-checker-key.pem ubuntu@54.238.247.106 "echo 'OK'"
```

### 服務無法啟動
```bash
# 查看詳細日誌
docker-compose logs

# 重新構建
docker-compose down
docker-compose up -d --build --force-recreate
```

### 時區不正確
```bash
# 檢查容器時區
docker-compose exec web date
docker-compose exec postgres psql -U globalping -d globalping_db -c "SELECT now();"

# 應該顯示 GMT+8 時間
```
