## 🐛 如果還有問題

### 問題 1：setup.sh 執行失敗

**解決方案**：
```bash
# 確認腳本有執行權限
chmod +x setup.sh run.sh

# 重新執行
bash setup.sh
```

### 問題 2：虛擬環境創建失敗

**解決方案**：
```bash
# 刪除舊的虛擬環境
rm -rf venv

# 重新執行安裝
bash setup.sh
```

### 問題 3：Pygame 安裝失敗

**可能原因**：網路問題

**解決方案**：
```bash
# 啟動虛擬環境
source venv/bin/activate

# 手動安裝 Pygame
pip install pygame

# 驗證安裝
python -c "import pygame; print(pygame.version.ver)"
```

## 📝 常見問題

### Q1: 為什麼不直接用 sudo pip install？
**A**: 使用 sudo 會修改系統 Python，可能導致系統問題。虛擬環境更安全。

### Q2: 虛擬環境會佔用很多空間嗎？
**A**: 不會，大約 20-30 MB，而且只在這個專案使用。

### Q3: 每次都要啟動虛擬環境嗎？
**A**: 是的，但我已經創建了 `run.sh` 腳本，會自動處理。

### Q4: 可以刪除虛擬環境嗎？
**A**: 可以，直接刪除 `venv/` 目錄，然後重新執行 `bash setup.sh`。

### Q5: 沒有圖片可以運行嗎？
**A**: 可以！遊戲會使用灰色方塊代替圖片，邏輯完全正常。
