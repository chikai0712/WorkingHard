# 遞迴解析器託管選項分析

## 問題：遞迴解析能否託管給 AWS 或 Google？

### 簡短答案

**AWS 和 Google 都不提供商業化的公開遞迴解析器託管服務**，主要原因是：
1. 遞迴解析器需要處理大量查詢，成本高
2. 容易被濫用（DDoS攻擊）
3. 需要大量資源和維護成本

## 各服務商提供的服務

### 1. AWS Route 53 Resolver

**提供的服務**：
- ✅ **VPC 內部遞迴解析**：用於VPC內資源查詢公共DNS記錄
- ✅ **轉發規則**：可以配置轉發規則解析內部DNS名稱
- ❌ **公開遞迴解析器**：不提供對外公開的遞迴解析器服務

**適用場景**：
- VPC內部的DNS查詢
- 混合雲環境的DNS解析
- 內部域名解析

**不適用場景**：
- 對外提供DNS服務（如方案一的需求）
- 處理來自互聯網的DNS查詢請求

### 2. Google Cloud DNS

**提供的服務**：
- ✅ **權威DNS服務**：託管DNS區域和記錄
- ✅ **VPC 內部DNS解析**：VPC內虛擬機可以解析內部和外部DNS名稱
- ❌ **公開遞迴解析器託管**：不提供商業化的公開遞迴解析器服務

**適用場景**：
- 權威DNS託管
- VPC內部DNS解析

**不適用場景**：
- 對外提供遞迴解析服務
- 處理來自互聯網的高QPS查詢

### 3. Google Public DNS (8.8.8.8)

**提供的服務**：
- ✅ **公開遞迴解析器**：任何人都可以使用
- ❌ **商業化託管**：不是給服務提供商使用的
- ❌ **自定義控制**：無法自定義限流、防禦規則
- ❌ **SLA保證**：沒有商業SLA

**適用場景**：
- 終端用戶的DNS查詢
- 個人或小型企業使用

**不適用場景**：
- 商業化DNS服務
- 需要自定義控制的場景
- 高QPS的商業需求

## 為什麼需要自建遞迴解析器？

### 1. 商業化需求
- 需要處理高QPS（20k+）
- 需要SLA保證
- 需要自定義限流和防禦規則

### 2. 成本考量
- 第三方遞迴解析器服務（如NS1、Dyn）成本極高
- 自建在高QPS情況下更經濟

### 3. 控制需求
- 需要完全控制查詢過程
- 需要快速恢復能力（1-2小時）
- 需要自定義監控和告警

## 替代方案分析

### 方案A：使用第三方遞迴解析器服務

**服務商**：
- NS1
- Dyn (Oracle)
- UltraDNS (Neustar)

**優點**：
- 無需維護基礎設施
- 專業服務和支援
- 高可用性

**缺點**：
- **成本極高**：20k QPS可能超過$50,000/月
- 無法自定義控制
- 依賴第三方服務

**費用估算（20k QPS）**：
- NS1: 約 $30,000-50,000/月
- Dyn: 約 $40,000-60,000/月
- UltraDNS: 約 $35,000-55,000/月

### 方案B：自建遞迴解析器（當前方案）

**優點**：
- **成本低**：$1,315/月（含折扣）
- 完全控制
- 快速恢復（1-2小時）
- 可享受雲端折扣

**缺點**：
- 需要維護基礎設施
- 需要專業運維團隊
- 初期投入較高

**費用估算（20k QPS）**：
- 自建成本：$1,315/月（含折扣）
- 比第三方服務便宜 **95%以上**

### 方案C：混合方案

**架構**：
- 權威DNS：使用 AWS Route53 或 Google Cloud DNS
- 遞迴解析器：自建

**優點**：
- 減少權威DNS的維護成本
- 保留遞迴解析器的控制權

**缺點**：
- 失去對權威DNS的控制
- 恢復時間依賴第三方

**費用估算**：
- 權威DNS：$0.5-0.7/月（託管區域費用）
- 遞迴解析器：$1,315/月（自建）
- 總計：約 $1,316/月

## 結論

1. **AWS 和 Google 都不提供商業化的公開遞迴解析器託管服務**
2. **自建遞迴解析器是最經濟的選擇**（比第三方服務便宜95%以上）
3. **如果需要託管服務**，可以考慮：
   - 權威DNS：使用 AWS Route53 或 Google Cloud DNS（$0.5-0.7/月）
   - 遞迴解析器：自建（$1,315/月）

## 建議

對於20k QPS的環境：
- ✅ **推薦**：自建遞迴解析器（當前方案）
- ❌ **不推薦**：使用第三方遞迴解析器服務（成本過高）
- ✅ **可選**：權威DNS使用雲服務商（減少維護成本）
