# 多供應商架構費用計算說明

## 概述

本文檔提供兩種多供應商架構策略的費用計算和對比：

1. **方案一 (WL)**: 自建DNS架構
2. **方案二 (API)**: 第三方DNS服務商架構

## 方案一：自建DNS架構 (WL)

### 架構說明

**重要澄清**：本架構為**純權威DNS架構**，只託管自己的域名，不進行遞迴查詢。

### 架構組成

- **域名註冊商**: Godaddy（主域名）
- **自建權威DNS**: BIND DNS Master（自建VM）- 對外提供權威DNS查詢服務
- **多CDN層**: Cloudflare, Mlytics, CDNetwork, Byte Plus CDN, Cloudfront

### 架構流程說明

```
Player (玩家)
    ↓ DNS查詢請求（查詢您的域名）
Recursive DNS Server (遞迴DNS服務器，如 8.8.8.8)
    ↓ 查詢權威記錄
自建權威DNS - BIND DNS Master
    ↓ 返回權威記錄
Player (玩家)
```

**自建權威DNS (BIND DNS Master) 的作用**：
- 作為**權威DNS服務器**（Authoritative DNS Server）
- 從域名註冊商（Godaddy）同步DNS記錄
- 作為Master服務器，管理DNS記錄的權威來源
- **直接對外提供權威DNS查詢服務**
- 只回應自己託管的域名查詢
- **不進行遞迴查詢**（不會查詢其他域名）

**流量說明**：
- **只有權威DNS回應流量**：當玩家查詢您的域名時，才會產生回應流量
- **不會產生遞迴查詢流量**：因為不查詢其他域名，所以沒有遞迴查詢的流量費用
- **流量大幅降低**：相比遞迴解析器架構，流量費用可降低 80-90%

### 費用明細（20k QPS）

| 項目 | 規格 | 數量 | 月費 (USD) |
|------|------|------|-----------|
| 域名註冊 | Godaddy | 1 | $15 |
| 權威DNS VM | n2-standard-4 (4vCPU/16GB) | 1-2 | $138-276 |
| 監控VM | n2-standard-4 (4vCPU/16GB) | 1 | $138 |
| Bastion VM | e2-medium (2vCPU/4GB) | 1 | $34 |
| Load Balancer | Global TCP LB | 1 | $20 |
| 磁碟費用 | 系統碟+日誌碟 | - | $30 |
| DNS出站流量 | 4,147 GB/月（權威DNS回應） | - | $474 |
| 監控流量 | 70 GB/月 | - | $8 |
| CDN費用 | 5個CDN服務商 | 5 | $535 |
| **小計（不含折扣）** | | | **$1,373-1,511** |
| **折扣後總計** | 1年承諾使用折扣30% | | **$961-1,058** |

**說明**：
- **權威DNS VM**：中型VM即可（n2-standard-4），單台可處理20k QPS
- **可選高可用**：2台Active-Active架構，提高可用性（+$138/月）
- **流量費用**：只有權威DNS回應流量，沒有遞迴查詢流量
- **成本降低**：相比遞迴解析器架構，節省約 $300-400/月

### 優勢

1. **完全控制**: 自建權威DNS可完全控制恢復時間（1-2小時）
2. **成本優化**: 
   - 運算資源可享受承諾使用折扣（省30%）
   - 不需要遞迴解析器，大幅降低VM成本
   - 只有權威DNS回應流量，流量費用降低
3. **長期成本**: 高QPS情況下，比第三方服務便宜95%以上
4. **靈活性**: 可根據需求調整架構和資源
5. **簡化架構**: 單一角色（權威DNS），維護簡單

### 劣勢

1. **維護成本**: 需要專業運維團隊維護基礎設施
2. **初期投入**: 需要較高的初期投入和設置時間
3. **技術要求**: 需要DNS和基礎設施管理專業知識

## 方案二：第三方DNS服務商架構 (API)

### 架構組成

- **域名註冊商**: Godaddy（主域名）+ Namecheap（備用域名）
- **NameServer**: Cloudflare Free + AWS Route53
- **DNS服務**: AWS Route53（配對Godaddy）+ Google Cloud DNS（配對Namecheap）
- **多CDN層**: Cloudflare, Mlytics, CDNetwork, Akamai, Cloudfront

### 費用明細（20k QPS）

| 項目 | 規格 | 數量 | 月費 (USD) |
|------|------|------|-----------|
| 域名註冊 | Godaddy + Namecheap | 2 | $27 |
| NameServer | Cloudflare Free + Route53 | 2 | $0.5 |
| DNS查詢費用 | AWS Route53 | - | $12,400 |
| DNS查詢費用 | Google Cloud DNS | - | $12,400 |
| DNS託管區域 | Route53 + GCP DNS | 2 | $0.7 |
| DNS健康檢查 | Route53 + GCP DNS | 4 | $2 |
| CDN費用 | 5個CDN服務商 | 5 | $835 |
| **總計** | | | **$24,866** |

### 優勢

1. **無需維護**: 無需維護基礎設施，專注業務
2. **高可用性**: 雙DNS服務商備援，自動故障轉移
3. **快速部署**: 可快速部署和擴展
4. **專業服務**: 享受雲服務商的專業維護和支援

### 劣勢

1. **查詢費用高**: DNS查詢費用佔總成本90%以上
2. **無法折扣**: 第三方服務無法享受承諾使用折扣
3. **依賴性**: 依賴第三方服務的可用性和定價
4. **恢復時間**: 依賴第三方服務的恢復時間

## 費用對比（20k QPS）

| 項目 | 方案一 (WL) | 方案二 (API) | 差異 |
|------|------------|-------------|------|
| **月費（不含折扣）** | $1,373-1,511 | $24,866 | +$23,355-23,493 |
| **月費（含折扣）** | $961-1,058 | $24,866 | +$23,808-23,905 |
| **域名註冊** | $15 | $27 | +$12 |
| **DNS服務** | $0 | $24,800 | +$24,800 |
| **CDN費用** | $535 | $835 | +$300 |
| **基礎設施** | $330-468 | $0 | -$330-468 |

### 關鍵發現

1. **DNS查詢費用是主要成本**: 在方案二中，DNS查詢費用（$24,800）佔總成本的99.7%
2. **AWS Route53 vs Google Cloud DNS**: AWS Route53和GCP都使用階梯定價，大流量時價格相同（前10億次$0.40/百萬次，超過$0.20/百萬次）
3. **方案一更經濟**: 在高QPS（20k+）情況下，方案一明顯更經濟（便宜97%以上）
4. **純權威DNS架構優勢**: 
   - 不需要遞迴解析器，VM成本降低約$400/月
   - 只有權威DNS回應流量，流量費用不變但架構更簡單
   - 總成本降低約$300-400/月
5. **方案二適合低QPS**: 如果QPS較低（<5k），方案二可能更經濟
6. **折扣影響**: 方案一可享受折扣，進一步降低成本

## 建議

### 選擇方案一的條件

- QPS > 10k
- 需要快速恢復能力（1-2小時）
- 有專業運維團隊
- 重視長期成本優化
- 需要完全控制DNS架構

### 選擇方案二的條件

- QPS < 5k
- 無專業運維團隊
- 需要快速部署
- 重視高可用性和自動故障轉移
- 預算充足

## 成本優化建議

### 方案一優化

1. **使用承諾使用折扣**: 可節省30%運算和磁碟費用
2. **純權威DNS架構**: 不需要遞迴解析器，VM成本降低約$400/月
3. **流量優化**: 權威DNS回應流量已是最小化（只有回應，沒有遞迴查詢）
4. **優化CDN配置**: 選擇性啟用CDN，避免重複費用
5. **監控優化**: 優化監控數據收集，減少流量
6. **高可用選項**: 可選2台Active-Active架構（+$138/月），提高可用性

### 方案二優化

1. **選擇合適的DNS服務商**: 比較AWS Route53和Google Cloud DNS的價格
2. **優化查詢模式**: 提高快取命中率，減少查詢次數
3. **CDN選擇**: 根據實際流量選擇最經濟的CDN組合
4. **域名管理**: 合理使用備用域名，避免不必要的查詢費用

## 注意事項

1. **QPS變化**: 費用會隨QPS變化而變化，需要定期評估
2. **流量波動**: 實際流量可能與估算有差異，需要監控和調整
3. **CDN費用**: CDN費用依實際流量計費，需要根據實際使用情況調整
4. **折扣條件**: 承諾使用折扣需要1年或3年承諾，需要評估長期需求
