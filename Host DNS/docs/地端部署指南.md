## 1. 部署架構

### 1.1 網路拓撲

```
Internet
   │
   ├── 主線路 (10 Gbps)
   │   └── 防火牆 1
   │
   └── 備援線路 (10 Gbps)
       └── 防火牆 2
           │
           ├── 核心交換機 (堆疊)
           │
           ├── HAProxy 1 (VIP: 203.0.113.10)
           ├── HAProxy 2 (備援)
           │
           ├── DNS1 (10.0.1.10)
           ├── DNS2 (10.0.1.11)
           └── DNS3 (10.0.1.12) - 備援站點
```

### 1.2 IP 規劃

| 設備 | IP 地址 | 說明 |
|------|---------|------|
| **VIP** | 203.0.113.10 | 對外服務 IP（HAProxy） |
| DNS1 | 10.0.1.10 | 主節點 1 |
| DNS2 | 10.0.1.11 | 主節點 2 |
| DNS3 | 10.0.1.12 | 備援節點 |
| HAProxy1 | 10.0.1.20 | 負載均衡器 1 |
| HAProxy2 | 10.0.1.21 | 負載均衡器 2 |
| Prometheus | 10.0.1.30 | 監控節點 |
| ClickHouse | 10.0.1.40 | 日誌存儲 |

## 2. 系統準備

### 2.1 作業系統安裝

所有節點安裝 Ubuntu Server 22.04 LTS：

```bash
# 最小化安裝，選擇：
# - OpenSSH server
# - 標準系統工具
```

### 2.2 基礎配置

```bash
# 更新系統
sudo apt update && sudo apt upgrade -y

# 安裝基礎工具
sudo apt install -y curl wget vim git htop net-tools

# 配置主機名
sudo hostnamectl set-hostname dns1
# 或 dns2, dns3, haproxy1, haproxy2 等

# 配置 hosts 文件
sudo vim /etc/hosts
# 添加：
# 10.0.1.10 dns1
# 10.0.1.11 dns2
# 10.0.1.12 dns3
# 10.0.1.20 haproxy1
# 10.0.1.21 haproxy2
```

### 2.3 網路配置

```bash
# 配置靜態 IP（以 DNS1 為例）
sudo vim /etc/netplan/00-installer-config.yaml

network:
  version: 2
  renderer: networkd
  ethernets:
    enp1s0:
      addresses:
        - 10.0.1.10/24
      gateway4: 10.0.1.1
      nameservers:
        addresses:
          - 8.8.8.8
          - 8.8.4.4

# 應用配置
sudo netplan apply
```

### 2.4 防火牆配置

```bash
# 安裝 UFW
sudo apt install -y ufw

# 允許 SSH
sudo ufw allow 22/tcp

# 允許 DNS
sudo ufw allow 53/udp
sudo ufw allow 53/tcp

# 啟用防火牆
sudo ufw enable
```

### 2.5 系統優化

```bash
# 編輯 sysctl 配置
sudo vim /etc/sysctl.d/99-dns-optimization.conf

# 添加以下內容：
net.netfilter.nf_conntrack_max = 9000000
net.netfilter.nf_conntrack_udp_timeout = 10
net.netfilter.nf_conntrack_udp_timeout_stream = 30
net.core.rmem_max = 134217728
net.core.wmem_max = 134217728
net.core.rmem_default = 8388608
net.core.wmem_default = 8388608
net.ipv4.udp_mem = 8388608 16777216 33554432
net.core.netdev_max_backlog = 5000
net.core.somaxconn = 4096

# 應用配置
sudo sysctl -p /etc/sysctl.d/99-dns-optimization.conf
```

## 3. 負載均衡器配置

### 3.1 安裝 HAProxy

```bash
# 在 HAProxy1 和 HAProxy2 上執行
sudo apt update
sudo apt install -y haproxy keepalived
```

### 3.2 HAProxy 配置

```bash
# 編輯 HAProxy 配置
sudo vim /etc/haproxy/haproxy.cfg

global
    log /dev/log local0
    log /dev/log local1 notice
    chroot /var/lib/haproxy
    stats socket /run/haproxy/admin.sock mode 660 level admin
    stats timeout 30s
    user haproxy
    group haproxy
    daemon
    maxconn 100000

defaults
    log global
    mode tcp
    option tcplog
    option dontlognull
    timeout connect 5000ms
    timeout client 50000ms
    timeout server 50000ms
    maxconn 100000

# DNS UDP 負載均衡
frontend dns_udp
    bind *:53
    mode udp
    default_backend dns_servers_udp

backend dns_servers_udp
    mode udp
    balance roundrobin
    option log-health-checks
    server dns1 10.0.1.10:53 check
    server dns2 10.0.1.11:53 check
    server dns3 10.0.1.12:53 check backup

# DNS TCP 負載均衡
frontend dns_tcp
    bind *:53
    mode tcp
    default_backend dns_servers_tcp

backend dns_servers_tcp
    mode tcp
    balance roundrobin
    option log-health-checks
    server dns1 10.0.1.10:53 check
    server dns2 10.0.1.11:53 check
    server dns3 10.0.1.12:53 check backup

# 統計頁面
frontend stats
    bind *:8404
    stats enable
    stats uri /stats
    stats refresh 30s
    stats admin if TRUE

# 重啟服務
sudo systemctl restart haproxy
sudo systemctl enable haproxy
```

### 3.3 Keepalived 配置（VRRP）

**HAProxy1 配置：**

```bash
sudo vim /etc/keepalived/keepalived.conf

vrrp_script chk_haproxy {
    script "/usr/bin/killall -0 haproxy"
    interval 2
    weight -2
    fall 3
    rise 2
}

vrrp_instance VI_1 {
    state MASTER
    interface enp1s0
    virtual_router_id 51
    priority 101
    advert_int 1
    authentication {
        auth_type PASS
        auth_pass your_secure_password
    }
    virtual_ipaddress {
        203.0.113.10/24
    }
    track_script {
        chk_haproxy
    }
}
```

**HAProxy2 配置：**

```bash
sudo vim /etc/keepalived/keepalived.conf

vrrp_script chk_haproxy {
    script "/usr/bin/killall -0 haproxy"
    interval 2
    weight -2
    fall 3
    rise 2
}

vrrp_instance VI_1 {
    state BACKUP
    interface enp1s0
    virtual_router_id 51
    priority 100
    advert_int 1
    authentication {
        auth_type PASS
        auth_pass your_secure_password
    }
    virtual_ipaddress {
        203.0.113.10/24
    }
    track_script {
        chk_haproxy
    }
}
```

**啟動服務：**

```bash
sudo systemctl restart keepalived
sudo systemctl enable keepalived
```

## 4. DNS 節點配置

### 4.1 安裝 dnsdist 和 Unbound

```bash
# 在 DNS1, DNS2, DNS3 上執行

# 添加 PowerDNS 倉庫
curl -L https://repo.powerdns.com/FD380FBB-pub.asc | sudo apt-key add -
echo "deb [arch=amd64] http://repo.powerdns.com/ubuntu jammy main" | sudo tee /etc/apt/sources.list.d/pdns.list

# 添加 Unbound 倉庫
sudo apt install -y software-properties-common
sudo add-apt-repository -y ppa:unbound/ppa

# 更新並安裝
sudo apt update
sudo apt install -y dnsdist unbound
```

### 4.2 dnsdist 配置

```bash
# 使用項目中的配置文件
sudo cp /path/to/config/dnsdist.conf /etc/dnsdist/dnsdist.conf

# 或手動配置
sudo vim /etc/dnsdist/dnsdist.conf

# 內容參考項目中的 config/dnsdist.conf
# 注意：將後端地址改為本地 Unbound
# address="127.0.0.1:5353"
```

### 4.3 Unbound 配置

```bash
# 使用項目中的配置文件
sudo cp /path/to/config/unbound.conf /etc/unbound/unbound.conf

# 下載 root hints 和 root key
sudo wget -O /etc/unbound/root.hints https://www.internic.net/domain/named.cache
sudo wget -O /etc/unbound/root.key https://data.iana.org/root-anchors/root-anchors.xml

# 設置權限
sudo chown -R unbound:unbound /etc/unbound
```

### 4.4 啟動服務

```bash
# 啟動 Unbound
sudo systemctl start unbound
sudo systemctl enable unbound

# 啟動 dnsdist
sudo systemctl start dnsdist
sudo systemctl enable dnsdist

# 檢查狀態
sudo systemctl status unbound
sudo systemctl status dnsdist
```

## 5. 監控系統配置

### 5.1 安裝 Prometheus

```bash
# 在監控節點上執行
cd /opt
wget https://github.com/prometheus/prometheus/releases/download/v2.45.0/prometheus-2.45.0.linux-amd64.tar.gz
tar xvf prometheus-2.45.0.linux-amd64.tar.gz
sudo mv prometheus-2.45.0.linux-amd64 /opt/prometheus

# 創建 systemd 服務
sudo vim /etc/systemd/system/prometheus.service

[Unit]
Description=Prometheus
After=network.target

[Service]
Type=simple
User=prometheus
ExecStart=/opt/prometheus/prometheus --config.file=/etc/prometheus/prometheus.yml
Restart=always

[Install]
WantedBy=multi-user.target

# 創建配置目錄
sudo mkdir -p /etc/prometheus
sudo cp /opt/prometheus/prometheus.yml /etc/prometheus/

# 創建用戶
sudo useradd -r -s /bin/false prometheus
sudo chown -R prometheus:prometheus /opt/prometheus /etc/prometheus

# 啟動服務
sudo systemctl daemon-reload
sudo systemctl start prometheus
sudo systemctl enable prometheus
```

### 5.2 Prometheus 配置

```bash
sudo vim /etc/prometheus/prometheus.yml

global:
  scrape_interval: 15s
  evaluation_interval: 15s

scrape_configs:
  - job_name: 'dnsdist'
    static_configs:
      - targets: ['10.0.1.10:8083', '10.0.1.11:8083', '10.0.1.12:8083']
  
  - job_name: 'unbound'
    static_configs:
      - targets: ['10.0.1.10:8953', '10.0.1.11:8953', '10.0.1.12:8953']

# 重啟服務
sudo systemctl restart prometheus
```

## 6. 高可用性配置

### 6.1 健康檢查腳本

```bash
# 在 HAProxy 節點上創建健康檢查腳本
sudo vim /usr/local/bin/check_dns.sh

#!/bin/bash
dig @127.0.0.1 google.com +timeout=2 +tries=1 > /dev/null 2>&1
exit $?

# 設置執行權限
sudo chmod +x /usr/local/bin/check_dns.sh
```

### 6.2 自動故障轉移測試

```bash
# 測試 HAProxy 故障轉移
# 在 HAProxy1 上停止服務
sudo systemctl stop haproxy

# 檢查 VIP 是否轉移到 HAProxy2
ip addr show | grep 203.0.113.10

# 恢復服務
sudo systemctl start haproxy
```

## 7. 測試與驗證

### 7.1 基本功能測試

```bash
# 測試 DNS 解析
dig @203.0.113.10 google.com
dig @203.0.113.10 google.com +tcp

# 測試負載均衡
for i in {1..10}; do
    dig @203.0.113.10 google.com +short
    sleep 1
done

# 檢查 HAProxy 統計
curl http://203.0.113.10:8404/stats
```

### 7.2 性能測試

```bash
# 安裝 dnsperf
sudo apt install -y dnsperf

# 創建查詢文件
cat > queryfile.txt << EOF
google.com A
example.com A
github.com A
EOF

# 執行性能測試
dnsperf -s 203.0.113.10 -d queryfile.txt -Q 1000 -l 30
```

### 7.3 故障轉移測試

```bash
# 1. 停止 DNS1
ssh dns1 "sudo systemctl stop dnsdist"

# 2. 驗證服務仍可用
dig @203.0.113.10 google.com

# 3. 檢查 HAProxy 日誌
sudo tail -f /var/log/haproxy.log

# 4. 恢復服務
ssh dns1 "sudo systemctl start dnsdist"
```

## 8. 維護與運維

### 8.1 日誌管理

```bash
# 配置日誌輪轉
sudo vim /etc/logrotate.d/dnsdist

/var/log/dnsdist/*.log {
    daily
    rotate 7
    compress
    delaycompress
    missingok
    notifempty
    create 0640 dnsdist dnsdist
}
```

### 8.2 備份配置

```bash
# 創建備份腳本
sudo vim /usr/local/bin/backup_dns.sh

#!/bin/bash
BACKUP_DIR="/backup/dns"
DATE=$(date +%Y%m%d)

mkdir -p $BACKUP_DIR/$DATE

# 備份配置文件
cp /etc/dnsdist/dnsdist.conf $BACKUP_DIR/$DATE/
cp /etc/unbound/unbound.conf $BACKUP_DIR/$DATE/
cp /etc/haproxy/haproxy.cfg $BACKUP_DIR/$DATE/

# 壓縮備份
tar -czf $BACKUP_DIR/dns_backup_$DATE.tar.gz $BACKUP_DIR/$DATE/

# 保留 30 天備份
find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete

# 設置定時任務
sudo chmod +x /usr/local/bin/backup_dns.sh
sudo crontab -e
# 添加：0 2 * * * /usr/local/bin/backup_dns.sh
```

## 10. 故障排除

### 10.1 常見問題

**問題：DNS 查詢超時**

```bash
# 檢查服務狀態
sudo systemctl status dnsdist
sudo systemctl status unbound

# 檢查日誌
sudo journalctl -u dnsdist -f
sudo journalctl -u unbound -f

# 檢查網路連接
sudo ss -tuln | grep 53
```

**問題：負載均衡不工作**

```bash
# 檢查 HAProxy 狀態
sudo systemctl status haproxy
sudo haproxy -c -f /etc/haproxy/haproxy.cfg

# 檢查後端健康狀態
curl http://localhost:8404/stats
```

**問題：VIP 不切換**

```bash
# 檢查 Keepalived 狀態
sudo systemctl status keepalived
sudo journalctl -u keepalived -f

# 檢查 VRRP 狀態
ip addr show
```
