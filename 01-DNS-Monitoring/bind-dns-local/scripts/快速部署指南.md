# AWS EC2 網站快速部署指南

本指南協助你快速在 AWS 上部署兩台 EC2，用於 DNS failover 測試。

## 前置需求

### 1. 安裝 AWS CLI

```bash
# macOS
brew install awscli

# 或使用 pip
pip3 install awscli

# 驗證
aws --version
```

### 2. 配置 AWS 憑證

```bash
aws configure
```

需要輸入：
- **AWS Access Key ID**: 你的 Access Key
- **AWS Secret Access Key**: 你的 Secret Key
- **Default region**: 例如 `ap-northeast-1`（東京）
- **Default output format**: `json`

### 3. 驗證配置

```bash
aws sts get-caller-identity
```

應該顯示你的 AWS 帳號資訊。

## 快速部署

### 方法 1：使用自動部署腳本（推薦）

```bash
cd bind-dns-local/scripts
bash ./deploy-ec2-websites.sh
```

腳本會自動：
1. ✅ 檢查並建立 Key Pair（如果不存在）
2. ✅ 檢查並建立 Security Group（如果不存在）
3. ✅ 自動選擇最新的 Amazon Linux 2 AMI
4. ✅ 啟動兩台 EC2 實例
5. ✅ 自動部署網站內容
6. ✅ 顯示實例資訊和 IP

### 方法 2：自訂配置

```bash
# 設定環境變數
export AWS_REGION=ap-northeast-1
export INSTANCE_TYPE=t2.micro
export KEY_NAME=my-key-pair
export SECURITY_GROUP_NAME=my-sg

# 執行部署
bash ./deploy-ec2-websites.sh
```

## 驗證部署

### 1. 檢查網站是否可訪問

```bash
# 測試 AWS EC2
curl http://<AWS_EC2_IP>

# 測試 Google EC2
curl http://<Google_EC2_IP>
```

應該看到 HTML 內容，包含 "我是 AWS" 或 "我是 Google"。

### 2. 在瀏覽器中訪問

打開瀏覽器，訪問：
- `http://<AWS_EC2_IP>` - 應顯示紫色背景的 "我是 AWS 3.3.3.3"
- `http://<Google_EC2_IP>` - 應顯示粉色背景的 "我是 Google 2.2.2.2"

## 下一步：配置 DNS

### 1. AWS Route53 配置

1. 登入 AWS Console → Route53
2. 建立或選擇 Hosted Zone
3. 建立 A 記錄：
   - **名稱**: `www`（或你的子域名）
   - **類型**: A
   - **值**: `<AWS_EC2_IP>`（從部署結果取得）
   - **TTL**: 60

### 2. Google Cloud DNS 配置

1. 登入 Google Cloud Console → Cloud DNS
2. 建立或選擇 Zone
3. 建立 A 記錄：
   - **DNS 名稱**: `www.yourdomain.com`
   - **資源記錄類型**: A
   - **IPv4 地址**: `<Google_EC2_IP>`（從部署結果取得）
   - **TTL**: 60

### 3. 執行 DNS Failover 測試

```bash
cd bind-dns-local/scripts
sudo bash ./dns-failover-test.sh www.yourdomain.com <AWS_EC2_IP> <Google_EC2_IP>
```

## 故障排除

### 問題 1: AWS CLI 未配置

```bash
# 檢查配置
aws configure list

# 重新配置
aws configure
```

### 問題 2: 權限不足

確保你的 AWS 帳號有以下權限：
- `ec2:RunInstances`
- `ec2:CreateKeyPair`
- `ec2:CreateSecurityGroup`
- `ec2:DescribeInstances`
- `ec2:DescribeImages`

### 問題 3: 實例無法訪問

1. 檢查 Security Group 是否開放 80 埠
2. 檢查實例狀態：`aws ec2 describe-instances --instance-ids <instance-id>`
3. 檢查實例日誌：在 EC2 Console 中查看 System Log

### 問題 4: 網站內容未更新

網站內容通過 User Data 自動部署，需要約 30-60 秒。如果超過 2 分鐘仍未更新，可以手動檢查：

```bash
# SSH 連線到實例
ssh -i ~/.ssh/<key-name>.pem ec2-user@<EC2_IP>

# 檢查 Nginx 狀態
sudo systemctl status nginx

# 檢查網站內容
cat /usr/share/nginx/html/index.html
```

## 清理資源

測試完成後，記得刪除 EC2 實例以節省費用：

```bash
# 列出實例
aws ec2 describe-instances \
    --filters "Name=tag:Name,Values=DNS-Test-*" \
    --query 'Reservations[*].Instances[*].[InstanceId,Tags[?Key==`Name`].Value|[0],PublicIpAddress]' \
    --output table

# 終止實例（替換為實際的實例 ID）
aws ec2 terminate-instances \
    --instance-ids i-xxxxxxxxxxxxx i-yyyyyyyyyyyyy \
    --region ap-northeast-1
```

## 費用估算

- **t2.micro**: 約 $0.0116/小時（約 $8.5/月）
- **兩台實例**: 約 $17/月
- **建議**: 測試完成後立即刪除實例

## 注意事項

1. ⚠️ **費用**: EC2 實例會產生費用，測試完成後請及時刪除
2. ⚠️ **IP 地址**: 實例重啟後 IP 可能會改變，建議使用 Elastic IP
3. ⚠️ **安全**: 測試環境建議使用限制性 Security Group，只允許必要的流量
4. ⚠️ **HTTPS**: 如需測試 HTTPS，請配置 SSL 憑證（可使用 Let's Encrypt）
