# Multi-NS BIND DNS 容器執行指南

## 🚀 快速啟動

### 方法一：使用啟動腳本（最簡單）

#### macOS/Linux
```bash
cd Multi-NS
./start.sh
```

#### Windows
```cmd
cd Multi-NS
start.bat
```

### 方法二：使用 Makefile（推薦）

```bash
cd Multi-NS
make start
```

### 方法三：直接使用 Docker Compose

```bash
cd Multi-NS
docker-compose up -d
```

---

## ✅ 啟動前檢查

### 1. 確認 Docker 已安裝並運行

```bash
# 檢查 Docker 是否運行
docker info
```

如果顯示錯誤，請先啟動 Docker Desktop 或 Docker daemon。

### 2. 確認必要目錄存在

啟動腳本會自動創建，但也可以手動檢查：

```bash
ls -la bind/
```

應該看到：
- `bind/config/` - 配置文件目錄
- `bind/zones/` - DNS Zone 文件目錄
- `bind/cache/` - 緩存目錄
- `bind/logs/` - 日誌目錄

### 1. 檢查容器狀態

```bash
# 使用 Makefile
make status

# 或直接查看

```

dig @127.0.0.1 example.com

docker-compose logs -f bind9

## 🔍 詳細操作說明

### 添加新域名（Zone）

1. **使用輔助腳本**（推薦）：
```bash
./add-zone.sh yourdomain.com 192.0.2.100
```

2. **手動添加**：
   - 編輯 `bind/config/named.conf.local`
   - 創建 Zone 文件在 `bind/zones/`
   - 重啟容器：`docker-compose restart bind9`

### 檢查配置語法

```bash
make checkconf
```

### 進入容器內部

```bash
make shell
```

然後可以執行 BIND 相關命令：
```bash
named-checkconf
named-checkzone example.com /var/lib/bind/zones/db.example.com
rndc status
```

## 📝 測試 DNS 查詢

### 基本查詢

```bash
# 查詢 A 記錄
dig @127.0.0.1 example.com

# 查詢 A 記錄（簡短輸出）
dig @127.0.0.1 example.com +short

# 反向查詢（PTR）
dig @127.0.0.1 -x 192.0.2.1
```

### 測試遞迴查詢

```bash
# 測試遞迴查詢（查詢外部域名）
dig @127.0.0.1 google.com
```

### 使用 nslookup（Windows/macOS/Linux）

```bash
nslookup example.com 127.0.0.1
```

## 🔧 故障排除

### 問題：容器無法啟動

1. **檢查 Docker 是否運行**
```bash
docker info
```

2. **檢查端口是否被占用**
```bash
# macOS/Linux
lsof -i :53

# Windows
netstat -ano | findstr :53
```

如果端口被占用，可以：
- 停止占用端口的服務
- 或修改 `docker-compose.yml` 中的端口映射

3. **查看錯誤日誌**
```bash
docker-compose logs bind9
```

### 問題：DNS 查詢失敗

1. **確認容器正在運行**
```bash
docker ps | grep bind9-dns
```

2. **檢查配置語法**
```bash
make checkconf
```

3. **檢查 Zone 文件語法**
```bash
docker exec -it bind9-dns named-checkzone example.com /var/lib/bind/zones/db.example.com
```

4. **查看容器日誌**
```bash
docker-compose logs bind9
```

### 問題：配置文件修改後不生效

1. **重啟容器**
```bash
docker-compose restart bind9
```

2. **或重新載入配置**（進入容器後）
```bash
docker exec -it bind9-dns rndc reload
```

## 💡 提示

1. **首次啟動**：系統會自動創建必要的目錄和配置文件
2. **配置修改**：修改配置文件後需要重啟容器才會生效
3. **日誌位置**：日誌文件保存在 `bind/logs/` 目錄
4. **數據持久化**：所有配置和數據都保存在本地 `bind/` 目錄
