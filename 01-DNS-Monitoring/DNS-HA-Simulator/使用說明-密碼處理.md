# DNS Port Forward 修復 - 密碼處理說明

## 為什麼需要密碼？

修復 DNS port forward 需要：
- 停止 macOS 系統服務（mDNSResponder）
- 綁定系統保留端口（53）
- 這些操作需要 **root 權限**（sudo）

## 安全說明

### ⚠️ 為什麼不在腳本中存儲密碼？

**絕對不要**在腳本中硬編碼密碼，原因：
1. **安全風險**：密碼可能被洩露
2. **版本控制**：如果提交到 Git，密碼會被記錄
3. **權限管理**：違反 macOS 安全原則

### ✅ 推薦做法

1. **使用 sudo 權限緩存**：
   ```bash
   # 在腳本開始時緩存權限
   sudo -v
   # 之後 5 分鐘內不需要再輸入密碼
   ```

2. **使用交互式輸入**：
   - 腳本執行時提示輸入
   - 密碼不會顯示在螢幕上
   - 不會保存到文件

3. **使用 macOS Keychain**（進階）：
   - 可以將 sudo 權限保存到 Keychain
   - 但需要額外配置，且安全性較低

## 常見問題

### Q: 可以設置免密碼 sudo 嗎？

**可以，但不推薦**。如果必須設置：

```bash
# 編輯 sudoers 文件（需要非常小心！）
sudo visudo

# 添加以下行（替換 YOUR_USERNAME）
YOUR_USERNAME ALL=(ALL) NOPASSWD: /path/to/fix-dns-port.sh
```

**風險**：
- 如果腳本被惡意修改，可能造成安全問題
- 違反最小權限原則

### Q: 為什麼每次都要輸入密碼？

這是 macOS 的安全機制：
- 保護系統服務不被隨意修改
- 防止未授權的端口綁定
- 符合安全最佳實踐

### Q: 可以自動輸入密碼嗎？

**不推薦**，但可以使用 `expect`（需要安裝）：

```bash
# 安裝 expect
brew install expect

# 創建 expect 腳本
# （但這會將密碼以某種形式存儲，不安全）
```

## 最佳實踐

1. **使用簡化腳本**：`./scripts/fix-dns-port-easy.sh`
   - 只需輸入一次密碼
   - 權限會自動緩存

2. **測試完成後恢復**：
   ```bash
   # 停止 port forward
   sudo ./scripts/setup-dns-forward.sh stop
   
   # 恢復 mDNSResponder
   sudo launchctl load -w /System/Library/LaunchDaemons/com.apple.mDNSResponder.plist
   ```

3. **如果不想處理密碼**：
   - 使用備用方案：在 Docker 容器內測試
   - 直接使用 5300 端口測試（不需要 sudo）

## 備用方案（不需要 sudo）

如果不想處理密碼問題，可以使用：

### 方案 1: 直接測試 5300 端口

```bash
# 不需要 sudo，直接測試
dig @127.0.0.1 -p 5300 app.example.com
```

### 方案 2: 在 Docker 容器內測試

```bash
# 進入 client 容器（不需要 sudo）
docker exec -it dns-client sh

# 在容器內測試
dig @172.20.0.101 app.example.com
curl http://app.example.com
```

這些方案可以完整測試 DNS HA 功能，且不需要修改 macOS 系統設定。
