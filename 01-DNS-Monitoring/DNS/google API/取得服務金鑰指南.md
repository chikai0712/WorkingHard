# 如何取得 Google Cloud 服務帳號金鑰

本指南將協助你取得 Google Cloud 服務帳號金鑰，以便使用 DNS 管理工具。

## 前置需求

1. 擁有 Google 帳號
2. 已建立或可存取 Google Cloud Platform (GCP) 專案
3. 具有專案的管理員權限（或可建立服務帳號的權限）

---

## 步驟 1: 登入 Google Cloud Console

1. 前往 [Google Cloud Console](https://console.cloud.google.com/)
2. 使用你的 Google 帳號登入
3. 選擇或建立一個專案
   - 如果還沒有專案，點擊頂部專案選擇器，然後點擊「新增專案」
   - 輸入專案名稱，記下專案 ID（稍後會用到）
lets crypt
---

## 步驟 2: 啟用 Cloud DNS API

1. 在 Google Cloud Console 中，前往 [API 和服務 > 程式庫](https://console.cloud.google.com/apis/library)
2. 在搜尋框中輸入「Cloud DNS API」
3. 點擊「Cloud DNS API」結果
4. 點擊「啟用」按鈕
5. 等待 API 啟用完成（通常幾秒鐘）

---

## 步驟 3: 建立服務帳號

1. 前往 [IAM 和管理 > 服務帳號](https://console.cloud.google.com/iam-admin/serviceaccounts)
2. 點擊頂部的「建立服務帳號」按鈕
3. 填寫服務帳號資訊：
   - **服務帳號名稱**：例如 `dns-manager` 或 `cloud-dns-service`
   - **服務帳號 ID**：會自動產生（可保持預設）
   - **說明**：例如「用於管理 Cloud DNS 的服務帳號」（選填）
4. 點擊「建立並繼續」

---

## 步驟 4: 授予權限

1. 在「授予此服務帳號存取專案的權限」步驟中：
   - 在「選取角色」下拉選單中，選擇「Cloud DNS > Cloud DNS Admin」
   - 或者搜尋「dns.admin」來快速找到
2. 點擊「繼續」
3. 在「授予使用者存取此服務帳號的權限」步驟中：
   - 通常不需要額外設定，直接點擊「完成」即可

---

## 步驟 5: 建立並下載金鑰

1. 在服務帳號列表中，找到剛才建立的服務帳號
2. 點擊該服務帳號的電子郵件地址（進入詳細頁面）
3. 切換到「金鑰」分頁
4. 點擊「新增金鑰」>「建立新金鑰」
5. 在彈出視窗中：
   - 選擇「JSON」格式（**重要：必須選擇 JSON**）
   - 點擊「建立」
6. 金鑰檔案會自動下載到你的電腦（通常是 `Downloads` 資料夾）
   - 檔案名稱類似：`your-project-id-xxxxxxxxxxxx.json`

---

## 步驟 6: 儲存金鑰檔案

1. 將下載的 JSON 檔案移動到專案目錄中
   - 建議放在：`/Users/ckchiu/Desktop/Project/DNS/google API/`
   - 可以重新命名為：`service-account-key.json`（方便識別）
2. **重要：請妥善保管此檔案，不要上傳到公開的 Git 儲存庫！**

---

## 步驟 7: 更新設定檔

編輯 `example.py`，更新以下參數：

```python
PROJECT_ID = "your-project-id"  # 替換為你的 GCP 專案 ID
SERVICE_ACCOUNT_KEY = "service-account-key.json"  # 金鑰檔案路徑（如果放在同目錄）
ZONE_NAME = "example-zone"  # 你的 Managed Zone 名稱
```

---

## 安全建議

1. **不要將金鑰檔案加入 Git**
   - 在專案根目錄建立 `.gitignore` 檔案
   - 加入：`*.json` 或 `service-account-key.json`

2. **限制金鑰權限**
   - 只授予必要的權限（Cloud DNS Admin）
   - 不要授予過度寬鬆的權限（如 Owner）

3. **定期輪換金鑰**
   - 建議每 90 天更換一次服務帳號金鑰
   - 舊金鑰刪除前，先確認新金鑰正常運作

## 常見問題

### Q: 找不到「服務帳號」選項？
A: 確認你已選擇正確的專案，並且具有專案的管理員權限。

### Q: 下載的金鑰檔案格式不對？
A: 必須選擇「JSON」格式，不要選擇「P12」格式。

### Q: 權限不足錯誤？
A: 確認服務帳號已授予「Cloud DNS Admin」角色，並且已啟用 Cloud DNS API。

### Q: 找不到專案 ID？
A: 在 Google Cloud Console 頂部的專案選擇器中，專案 ID 會顯示在專案名稱旁邊。

2. 查看錯誤訊息，通常在錯誤訊息中會指出具體問題
3. 確認所有步驟都已正確完成
