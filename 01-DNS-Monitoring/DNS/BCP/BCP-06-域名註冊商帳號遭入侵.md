# BCP-06: 域名註冊商帳號遭入侵與域名劫持應變

## 📋 異常發現

### 用戶端現象
- **網站被導向惡意頁面**：瀏覽器顯示釣魚網站或未知內容
- **無法登入註冊商後台**：帳號密碼被竄改，Email 驗證失效
- **NS 記錄被修改**：域名指向未知的惡意 DNS 伺服器
- **域名狀態異常**：WHOIS 顯示域名鎖被解除或轉移狀態異常

### 觸發情境
域名註冊商（如 GoDaddy、Namecheap）管理帳號遭入侵，攻擊者：
- 修改域名 NS 記錄指向惡意伺服器
- 竄改帳號登入憑據（Email、密碼）
- 解除域名鎖（Domain Lock）
- 嘗試將域名轉移至其他註冊商

**嚴重性評估**：🔴 **P0 級別** - 企業級資安災難，可能導致：
- 用戶流量被導向釣魚網站
- 品牌信譽受損
- 業務中斷
- 客戶資料外洩風險

## 🔍 偵測與驗證流程

### 自動偵測
- **DNS 監控告警**：監控服務偵測到 NS 記錄異常變更
- **網站可用性監控**：網站無法正常訪問或返回異常內容
- **SSL 憑證告警**：憑證與預期不符

### 手動驗證步驟

#### 1. 驗證 DNS 記錄異常
```bash
# 檢查當前 NS 記錄
dig NS example.com +short

# 檢查 A 記錄是否指向預期 IP
dig A example.com +short

# 追蹤完整解析路徑
dig +trace example.com

# 使用多個公共 DNS 驗證
dig @1.1.1.1 example.com +short
dig @8.8.8.8 example.com +short
dig @168.95.1.1 example.com +short  # 台灣 ISP DNS
```

#### 2. 檢查 WHOIS 資訊
```bash
# 查詢域名狀態
whois example.com | grep -i "status\|name server\|registrar"

# 重點檢查：
# - Status 是否包含 "clientTransferProhibited"（域名鎖）
# - Name Server 是否為預期的伺服器
# - Registrar 是否異常變更
```

#### 3. 驗證註冊商帳號狀態
- 嘗試登入註冊商後台
- 檢查 Email 是否收到異常登入通知
- 驗證 2FA 設定是否被移除
- 檢查帳號活動記錄（如 GoDaddy 的 Account Activity）

#### 4. 確認影響範圍
```bash
# 使用 DNS 檢測工具驗證全球解析狀態
# https://dnschecker.org
# https://www.whatsmydns.net

# 檢查所有相關域名
for domain in example.com www.example.com api.example.com; do
  echo "檢查 $domain:"
  dig $domain +short
done
```

### 判斷標準
- ⚠️ **疑似異常**：NS 記錄變更但可登入後台
- 🔴 **確認入侵**：無法登入後台 + NS 記錄異常 + WHOIS 顯示異常狀態
- 🔴 **嚴重**：多個域名同時受影響 + 域名鎖被解除

## 🚨 應變流程

### 階段一：損害管制 (Damage Control) - T+0~T+30 分鐘

#### 1. 確認異常與影響範圍（0-5 分鐘）

1. **快速驗證異常**
   ```bash
   # 檢查 NS 記錄
   dig NS example.com +short
   
   # 檢查 A 記錄
   dig A example.com +short
   
   # 檢查 WHOIS
   whois example.com | grep -i "name server\|status"
   
   # 使用線上工具驗證全球解析
   # https://dnschecker.org/#NS/example.com
   ```

2. **確認無法登入註冊商**
   - 嘗試登入 GoDaddy/Namecheap 後台
   - 檢查 Email 是否收到異常登入通知
   - 驗證 2FA 是否失效

3. **評估影響範圍**
   - 列出所有受影響的域名
   - 確認是否有備援域名可用
   - 評估業務影響（用戶數量、服務類型）

#### 2. 啟動事件通報（5 分鐘內）

1. **內部通報**
   - Slack `#bcp-infra-alert` @HOST @SecOps @BCP-DNS-team @Legal
   - 觸發 **TP0 告警**（DOMAIN-HIJACK）
   - 電話通知事件指揮官、資安團隊、法務
   - 記錄異常開始時間、受影響域名清單、初步影響評估

2. **對外公告（5 分鐘內）**
   - 更新狀態頁：發布「DNS 異常，正在緊急處理中」
   - 社群媒體公告：告知用戶目前遭遇 DNS 劫持，暫時勿點擊任何連結
   - APP 推播：通知用戶暫時避免使用受影響服務
   - 使用「狀態頁更新模板」：
     ```
     ⚠️ 緊急公告：DNS 異常
     我們偵測到域名解析異常，正在緊急處理中。
     請用戶暫時避免點擊任何可疑連結。
     預計恢復時間：待評估
     ```

#### 3. 聯繫解析服務商（10 分鐘內）

1. **聯繫 Cloudflare / AWS Route53**
   - 確認目前 NS 是否仍有部分流量
   - 雖然 NS 被改了，但全球 DNS 緩存尚未完全更新
   - 請求鎖定現有的 DNS 區域設定，防止進一步連鎖反應
   - 取得 DNS 記錄備份（如有）

2. **檢查備援 DNS 設定**
   - 確認備援 DNS 提供商（如 Route53）的記錄是否完整
   - 準備快速切換方案

### 階段二：奪回控制權 (Account Recovery) - T+30 分鐘~T+2 小時

#### 1. 聯繫註冊商緊急資安小組（30 分鐘內）

1. **準備身份驗證文件**
   - ✅ 公司營業執照（或組織登記證明）
   - ✅ 原註冊人身分證明
   - ✅ 最近一次域名續約的信用卡帳單
   - ✅ 域名註冊時的原始 Email 記錄
   - ✅ 域名註冊時的原始地址證明
   - ✅ 最近一次成功登入的時間記錄（如有）

2. **聯繫 GoDaddy 24/7 客服**
   - **電話**：+1-480-505-8877（國際）或當地客服電話
   - **Email**：security@godaddy.com（資安事件專用）
   - **線上客服**：https://www.godaddy.com/help（選擇「Security」類別）
   
   **通話腳本範例**：
   ```
   您好，我是 [公司名稱] 的 [姓名]。
   我們的域名 [example.com] 管理帳號遭入侵，
   無法登入後台，且 NS 記錄被惡意修改。
   這屬於緊急資安事件，需要立即處理。
    
   我已準備好以下身份驗證文件：
   1. 公司營業執照
   2. 域名註冊人身分證明
   3. 最近一次續約的信用卡帳單
    
    請問需要提供哪些額外文件來驗證身份？
   ```

3. **聯繫 Namecheap 客服**（如使用 Namecheap）
   - **電話**：+1-661-310-2107
   - **Email**：abuse@namecheap.com（緊急事件）
   - **線上客服**：https://www.namecheap.com/support/

#### 2. 檢查域名鎖狀態（同時進行）

1. **查詢 WHOIS 確認域名鎖狀態**
   ```bash
   whois example.com | grep -i "status"
   ```

2. **確認狀態碼**
   - ✅ **clientTransferProhibited**：域名鎖已開啟（好消息，駭客無法立即轉移）
   - ❌ **無此狀態**：域名鎖被解除（需立即請求重新啟用）
   - ⚠️ **pendingTransfer**：域名正在轉移中（最嚴重，需立即阻止）

3. **請求重新啟用域名鎖**
   - 在聯繫註冊商時，明確要求：
     - 重新啟用 Domain Lock（域名鎖）
     - 啟用 Registry Lock（註冊局鎖，如可用）
     - 凍結所有域名變更操作，直到身份驗證完成

#### 3. 2FA 與帳號安全恢復

1. **請求重置帳號權限**
   - 提供身份驗證文件
   - 請求重置 Email 和密碼
   - 重新啟用 2FA（建議使用硬體 Key 如 Yubikey）

2. **檢查帳號活動記錄**
   - 要求註冊商提供異常登入記錄
   - 確認攻擊時間點和來源 IP
   - 記錄作為證據保全

### 階段三：緊急轉移至備援註冊商 (Emergency Transfer) - T+2 小時~T+24 小時

**觸發條件**：如果註冊商處理緩慢（超過 4 小時無回應），需準備將域名強行轉出或恢復至備援註冊商。

#### 1. 向備援註冊商發起移入請求（2-4 小時）

1. **準備 Namecheap 移入請求**
   - 登入 Namecheap 帳號（確保已完成 KYC 身份認證）
   - 進入 `Domain Transfer` 頁面
   - 輸入域名和轉移碼（Auth Code）

2. **取得轉移碼的替代方案**
   - **正常流程**：從原註冊商後台取得 Auth Code
   - **異常情況**：如果拿不到轉移碼，需透過以下方式：
     - 請求原註冊商手動提供（需身份驗證）
     - 透過 ICANN 申訴程序
     - 聯繫註冊局（Registry）直接處理

#### 2. ICANN 域名爭議申訴（如無法取得轉移碼）

1. **填寫 ICANN Inter-Registrar Transfer Complaint**
   - **網址**：https://www.icann.org/compliance/complaint
   - **適用情況**：
     - 註冊商拒絕提供轉移碼
     - 註冊商無回應（超過 5 個工作天）
     - 域名被非法轉移

2. **申訴所需資訊**
   - 域名資訊
   - 註冊商名稱
   - 問題描述（帳號遭入侵、無法取得轉移碼）
   - 身份驗證文件
   - 與註冊商的通訊記錄

3. **申訴模板範例**
   ```
   主題：域名轉移爭議 - 帳號遭入侵
   
   域名：example.com
   註冊商：GoDaddy
   問題：我們的註冊商帳號遭入侵，無法登入後台取得轉移碼。
   我們已提供身份驗證文件給註冊商，但超過 5 個工作天未獲回應。
   
   請求：協助取得轉移碼或直接處理域名轉移。
   
   附件：
   1. 公司營業執照
   2. 域名註冊證明
   3. 與註冊商的通訊記錄
   ```

#### 3. 緊急 DNS 恢復方案（同時進行）

如果域名轉移需要時間，但需要立即恢復服務：

1. **使用備援域名**
   - 切換至預先準備的備援域名（如 example-backup.com）
   - 更新 CDN 和應用程式配置
   - 通知用戶使用新域名

2. **直接使用 IP 訪問**（臨時方案）
   - 在應用程式內提供直接 IP 訪問選項
   - 更新 App 推播，提供臨時訪問方式
   - 僅作為最後手段，不建議長期使用

### 階段四：DNS 記錄恢復與驗證 - T+24 小時~T+48 小時

#### 1. 恢復正確的 NS 記錄

1. **登入恢復後的註冊商帳號**
   - 確認帳號權限已恢復
   - 驗證 2FA 已重新啟用
   - 檢查所有安全設定

2. **恢復 NS 記錄**
   - 將 NS 記錄改回正確的 DNS 伺服器（如 Cloudflare、Route53）
   - 確認域名鎖已重新啟用
   - 記錄變更時間

3. **驗證 DNS 記錄完整性**
   ```bash

   dig NS example.com +short
   
   # 檢查 A 記錄
   dig A example.com +short
   
   # 檢查 MX 記錄（如有）
   dig MX example.com +short
   
   # 使用線上工具驗證全球傳播
   # https://dnschecker.org
   ```

#### 2. 全球 DNS 傳播驗證（24-48 小時）

1. **持續監控 DNS 傳播**

   - 記錄恢復時間點

2. **驗證服務可用性**

   - 驗證 API 端點正常

## 💡 進階建議

### Registry Lock（註冊局鎖）

如果您管理的域名價值極高，可以考慮向註冊商購買 **Registry Lock（註冊局鎖）**。

**Registry Lock vs Domain Lock：**
- **Domain Lock**：註冊商層級的鎖定，可透過註冊商後台解除
- **Registry Lock**：註冊局層級的鎖定，要修改 NS 必須透過：
  - 人工電話驗證
  - 多重簽署
  - 註冊局直接處理

**優點：**
- 即使帳號密碼被盜，駭客也無法修改 NS
- 提供最高等級的域名保護

**缺點：**
- 需要額外費用（通常每年 $50-200 USD）
- 變更 DNS 記錄需要更長時間（需人工處理）

**適用場景：**
- 高價值域名（品牌域名、主要業務域名）
- 金融、醫療等高度監管行業
- 曾經遭遇過域名劫持的組織
