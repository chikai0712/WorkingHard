# 方案一 (WL) 架構角色說明

## 架構說明

**重要澄清**：本架構為**純權威DNS架構**，只託管自己的域名，不進行遞迴查詢。

## 架構流程

```
Player (玩家)
    ↓ DNS查詢請求（查詢您的域名）
Recursive DNS Server (遞迴DNS服務器，如 8.8.8.8)
    ↓ 查詢權威記錄
自建權威DNS - BIND DNS Master
    ↓ 返回權威記錄
Player (玩家)
```

**說明**：
- 玩家查詢您的域名時，會透過遞迴DNS服務器（如 Google 8.8.8.8）查詢
- 遞迴DNS服務器會查詢您的權威DNS服務器
- 您的權威DNS只返回自己託管的域名記錄
- **不會產生遞迴查詢其他域名的流量**

## 架構角色

### 自建權威DNS (BIND DNS Master)

**角色**: Authoritative DNS Server（權威DNS服務器）

**作用**:
- 從域名註冊商（Godaddy）同步DNS記錄
- 作為Master服務器，管理DNS記錄的權威來源
- 存儲和管理您的域名的DNS記錄（A記錄、AAAA記錄、CNAME等）
- **直接對外提供權威DNS查詢服務**
- 只回應自己託管的域名查詢
- **不進行遞迴查詢**（不會查詢其他域名）

**為什麼需要自建權威DNS？**
- 完全控制DNS記錄管理
- 可以將恢復時間掌握在自己手裡（1-2小時內恢復異常）
- 作為備援機制，當域名註冊商的DNS出現問題時，可以快速切換
- 高QPS情況下成本更低（相比第三方DNS服務如 Route53）
- 可以享受雲端折扣（30%）

**技術規格**:
- 使用 BIND DNS 作為權威DNS服務器
- 中型VM即可（需要處理20k QPS的查詢請求）
- 建議規格：n2-standard-4 (4vCPU/16GB) 或更高
- 可選：2台Active-Active架構，跨區域部署（提高可用性）

**流量說明**:
- **只有權威DNS回應流量**：當玩家查詢您的域名時，才會產生回應流量
- **不會產生遞迴查詢流量**：因為不查詢其他域名，所以沒有遞迴查詢的流量費用
- **流量大幅降低**：相比遞迴解析器架構，流量費用可降低 80-90%

## 流量與費用說明

### 流量產生機制

1. **只有權威DNS回應流量**：
   - 當玩家查詢您的域名時，遞迴DNS服務器（如 8.8.8.8）會查詢您的權威DNS
   - 您的權威DNS只返回自己託管的域名記錄
   - **不會查詢其他域名**，所以沒有遞迴查詢的流量

2. **流量計算**：
   - 查詢流量：玩家查詢您的域名 → 遞迴DNS服務器查詢您的權威DNS（入站，通常免費）
   - 回應流量：權威DNS返回記錄給遞迴DNS服務器（出站，產生費用）
   - **只有回應流量會產生費用**

3. **流量優化**：
   - 權威DNS回應通常很小（80-100 bytes）
   - 20k QPS 約產生 4,147 GB/月的出站流量
   - 相比遞迴解析器架構，流量費用大幅降低

### 架構優勢

1. **成本優化**:
   - 不需要遞迴解析器（dnsdist + Unbound）
   - 不需要高併發處理的VM（n2-standard-8）
   - 只需要中型VM（n2-standard-4）即可
   - 流量費用大幅降低（只有權威DNS回應，沒有遞迴查詢）

2. **簡化架構**:
   - 單一角色：權威DNS服務器
   - 職責清晰：只託管自己的域名
   - 維護簡單：不需要複雜的遞迴解析配置

3. **快速恢復**:
   - 完全控制DNS記錄
   - 可以在1-2小時內恢復異常
   - 不依賴第三方服務

## 為什麼選擇自建權威DNS？

### 與第三方權威DNS服務對比

#### AWS Route53 / Google Cloud DNS
- ✅ 提供權威DNS服務
- ❌ **查詢費用極高**：20k QPS 約 $20,000-33,000/月
- ❌ 無法享受折扣
- ❌ 恢復時間依賴第三方

#### 自建權威DNS
- ✅ **成本低**：約 $200-400/月（含VM和流量）
- ✅ 可以享受雲端折扣（30%）
- ✅ 完全控制DNS記錄
- ✅ 快速恢復（1-2小時）
- ✅ 不依賴第三方服務

**結論**：對於高QPS（>10k）環境，自建權威DNS比第三方服務便宜 **95%以上**。

## 建議

對於20k QPS的環境，建議使用自建權威DNS架構：
- **權威DNS**: 中型VM（n2-standard-4），處理查詢請求
- **可選高可用**: 2台Active-Active架構，跨區域部署
- **總成本**: 約 $200-400/月（含VM、流量和折扣）
- **優勢**: 成本低、完全控制、快速恢復、不依賴第三方
