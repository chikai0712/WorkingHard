# GCP 單一平台 Active-Active Master 架構說明

## 🎯 決策概述

選擇 **Google Cloud Platform (GCP) 單一平台**來建置兩台 **Active-Active (AA) Master 主機**，這是一個非常務實且高效的決策！

---

## ✅ 核心優勢

### 1. **極速內網 (VPC Network)**

#### 技術優勢
- **全球光纖骨幹**：兩台主機雖在不同區域（如台灣與東京），但走的是 Google 全球光纖骨幹
- **低延遲同步**：VPC 內網延遲通常 < 5ms，即使跨區域
- **高頻寬**：VPC 內網頻寬不受限制，可達到 10 Gbps 以上
- **穩定可靠**：Google 全球網路基礎設施，99.99% 可用性保證

#### 實際效益
- ✅ **DNS 同步速度極快**：Master 之間的 zone transfer 和配置同步幾乎即時
- ✅ **監控數據傳輸無壓力**：Prometheus 指標、日誌傳輸不佔用外網頻寬
- ✅ **健康檢查即時**：跨區域健康檢查延遲極低

---

### 2. **管理單純化**

#### 統一管理介面
- **統一帳單**：所有資源在同一帳單，費用管理簡單
- **統一 IAM 權限**：一套權限管理系統，無需跨平台配置
- **統一監控介面**：Cloud Operations (原 Stackdriver) 統一監控
- **統一 API**：所有操作使用同一套 GCP API

#### 實際效益
- ✅ **降低管理成本**：無需學習多套平台操作
- ✅ **簡化權限管理**：一套 IAM 政策管理所有資源
- ✅ **統一告警通知**：所有告警集中在 Cloud Monitoring
- ✅ **統一日誌查詢**：Cloud Logging 統一查詢所有日誌

---

### 3. **異地備援 (Disaster Recovery)**

#### 區域選擇建議
- **主區域**：`asia-east1` (台灣)
- **備援區域**：`asia-northeast1` (東京) 或 `asia-southeast1` (新加坡)

#### 災難備援能力
- **物理級別備援**：不同區域 = 不同數據中心 = 不同物理位置
- **網路隔離**：即使某區域網路故障，另一區域仍可運作
- **自動故障轉移**：GCP Load Balancer 可自動偵測故障並轉移流量

#### 實際效益
- ✅ **符合災難備援要求**：達到物理級別的災難備援
- ✅ **自動恢復**：GCP 自動處理區域級故障
- ✅ **零停機時間**：故障轉移幾乎無感知

## 🚀 部署建議

### 1. 區域選擇

#### 推薦配置
- **Master #1**：`asia-east1-a` (台灣，可用區 A)
- **Master #2**：`asia-northeast1-a` (東京，可用區 A)

#### 替代配置
- **Master #1**：`asia-east1` (台灣)
- **Master #2**：`asia-southeast1` (新加坡)

### 2. VPC 網路配置

```yaml
VPC 設定:
  - 名稱: host-dns-vpc
  - CIDR: 10.0.0.0/16
  - 子網路:
    - asia-east1: 10.0.1.0/24
    - asia-northeast1: 10.0.2.0/24
  - 防火牆規則:
    - 允許 VPC 內網通訊
    - 允許 DNS (UDP/TCP 53)
    - 允許健康檢查
```

### 3. Load Balancer 配置

```yaml
Global TCP Load Balancer:
  - 類型: External TCP/UDP Network Load Balancer
  - 後端服務: 2 台 Master VM
  - 健康檢查: HTTP (port 8080/metrics)
  - 會話親和性: None (Active-Active)
  - 分發模式: 均勻分發
```

## 📊 性能指標

### 預期性能

- **內網延遲**：< 5ms（跨區域）
- **同步速度**：> 1 Gbps（VPC 內網）
- **可用性**：99.99%+（GCP SLA）
- **故障轉移時間**：< 30 秒

### 容量規劃

- **單台 Master**：10k QPS
- **兩台 Master**：20k QPS（正常運作）
- **故障後**：10k QPS（單台運作，可能降級）

---

## ⚠️ 注意事項

### 1. 容量考量

- **2 台 Master** 的容量剛好滿足 20k QPS 需求
- **故障時**：只剩 1 台（10k QPS），可能無法應付峰值
- **建議**：如果預算允許，可考慮 3 台 Master（方案 B）

### 2. 監控與告警

- 設定 **Master 故障告警**
- 監控 **VPC 內網延遲**
- 監控 **Load Balancer 健康狀態**

### 3. 備份策略

- 定期備份 DNS zone 配置
- 使用 GCP Persistent Disk Snapshot
- 跨區域備份（自動複製到另一區域）
