# DNS 三種方案完整比較分析

## 概述

本文檔提供 Host DNS 的三種部署方案的全面比較分析：
1. **自己買機器建立**（On-Premises）
2. **雲端自建**（Cloud Self-Hosted）
3. **採購第三方服務**（Third-Party Managed Service）

**幣別說明**：本文檔中所有費用均以**美金（USD）**為單位。如有其他幣別，將統一轉換為 USD（匯率以 2025 年 12 月為準）。

**人力費用說明**：人力成本以台灣市場為基準，假設 DNS 運維工程師年薪 100 萬台幣（約 $31,250 USD，匯率 1 USD = 32 TWD），月薪約 $2,604 USD，時薪約 $15.62 USD/小時（假設年工作時數 2,000 小時）。

## 二、方案一：自己買機器建立（On-Premises）

### 架構說明

在自有機房或租用機房中部署物理伺服器，自行安裝和管理 DNS 軟體（如 BIND DNS）。

### SLA 與可用性

- **SLA 保證**：無（自控，取決於自身架構設計）
- **目標可用性**：99.9% - 99.99%（取決於架構設計和維護能力）
- **異常恢復時間**：1-4 小時（取決於故障類型和團隊能力）
- **多點部署**：需要額外投資（多機房部署）

### DDoS 防護能力

- **防護方式**：
  - 硬體防火牆（需額外投資 $10,000-30,000）
  - 雲端清洗服務（月費 $500-2,000）
  - 自行配置限流規則
- **防護能力**：取決於硬體設備和頻寬，通常可防護 10-100 Gbps 攻擊
- **自動化程度**：需自行配置和監控

### 優點

1. **完全控制權**
   - 完全控制硬體和軟體配置
   - 可根據需求客製化
   - 數據完全在內部，隱私性高

2. **無查詢計量費用**
   - 不依查詢量計費
   - 高 QPS 環境下成本固定

3. **快速恢復能力**
   - 不依賴第三方
   - 可快速修復和恢復

4. **合規性**
   - 適合有嚴格數據駐留要求的場景
   - 可滿足特定行業合規要求

### 缺點

1. **高初始投資**
   - 需要大量前期資本投入
   - 硬體折舊和更新成本

2. **高維護負擔**
   - 需要專業運維團隊
   - 7x24 監控和維護
   - 故障排除和更新責任

3. **擴展性受限**
   - 硬體擴展需要時間和成本
   - 應對流量突增能力有限

4. **DDoS 防護成本高**
   - 需要額外投資防護設備
   - 大規模攻擊防護能力有限

5. **單點故障風險**
   - 需要多機房部署才能實現高可用
   - 災難恢復能力有限

### 適用場景

- ✅ 對數據隱私和合規有嚴格要求
- ✅ 有專業運維團隊
- ✅ 預算充足，可承擔高初始投資
- ✅ 需要完全控制 DNS 服務
- ✅ QPS 較高（>10k），希望固定成本

### 不適用場景

- ❌ 預算有限
- ❌ 無專業運維團隊
- ❌ 需要快速擴展
- ❌ 對 DDoS 防護要求高但預算有限

- 與 Google 生態系統整合較好 

### 3.4 SLA 與可用性

- **SLA 保證**：無（自控，取決於雲端平台 SLA）
- **雲端平台 SLA**：
  - **AWS EC2**：99.99%（多可用區）
  - **GCP Compute Engine**：99.99%（多區域）
- **目標可用性**：99.9% - 99.99%（取決於架構設計）
- **異常恢復時間**：1-2 小時（可快速重建 VM）
- **多點部署**：容易實現（跨區域部署）

### DDoS 防護能力

- **防護方式**：
  - 雲端平台基礎 DDoS 防護（AWS Shield Standard、GCP Cloud Armor）
  - 自行配置限流規則（BIND DNS）
  - 可選：雲端 DDoS 防護服務（AWS Shield Advanced $3,000/月起）
- **防護能力**：
  - AWS Shield Standard：免費，基礎防護
  - AWS Shield Advanced：可防護大規模攻擊（需額外費用）
  - GCP Cloud Armor：基礎防護，高級功能需付費
- **自動化程度**：需自行配置，可通過腳本自動化

### 優點

1. **低初始投資**
   - 無需購買硬體
   - 按需付費，降低前期成本

2. **高擴展性**
   - 可快速擴展 VM 資源
   - 跨區域部署容易
   - 自動擴展能力

3. **完全控制權**
   - 完全控制 DNS 配置
   - 可快速恢復和修復
   - 不依賴第三方 DNS 服務

4. **成本優勢（5億次/月）**
   - 月運營成本 $656-753，比 AWS/GCP 自建便宜約 86%（第三方服務角度）
   - 但需考慮前期建置成本 $13,300-24,900
   - 無查詢計量費用（固定成本）
   - 可享受雲端折扣

5. **靈活性**
   - 可選擇不同雲端平台
   - 可混合使用多雲架構

### 缺點

1. **維護負擔**
   - 需要專業運維團隊
   - 需要監控和維護 VM
   - 需要處理更新和補丁

2. **DDoS 防護需額外配置**
   - 基礎防護有限
   - 高級防護需額外費用
   - 需自行配置限流規則

3. **學習曲線**
   - 需要熟悉雲端平台
   - 需要 DNS 專業知識

4. **流量費用**
   - 出站流量產生費用
   - 高流量時費用增加

### 3.6 適用場景

- ✅ 長期使用（>3 年），可攤平前期建置成本
- ✅ 需要完全控制 DNS 服務
- ✅ 有運維團隊（或可外包）
- ✅ 需要快速擴展能力
- ✅ 查詢量穩定且較高（>20 億次/月）
- ✅ **選擇 AWS**：成本敏感，需要較便宜的流量費用，已使用 AWS 生態系統
- ✅ **選擇 GCP**：已使用 Google Cloud 生態系統，需要 Global Load Balancer

### 3.7 不適用場景

- ❌ 低查詢量環境（<10 億次/月，第三方服務更經濟）
- ❌ 無運維團隊且無法外包
- ❌ 需要零維護
- ❌ 短期專案或不確定長期需求
- ❌ 對 DDoS 防護要求極高但預算有限

## 四、方案三：採購第三方服務（Third-Party Managed Service）

### 4.1 Cloudflare DNS

#### 方案類型

- 如需專職運維工程師維護雲端自建方案，需額外加上約 $24,000/年的人力成本（0.75 名工程師）

| **10k - 20k** | 雲端自建 | 成本優勢明顯（$961-1,058/月） |

|-------|---------|------|

## 八、總結與建議

### 8.1 成本結論（5億次/月，峰值20k QPS）

1. **Cloudflare Free/Pro 最經濟**：
   - Free 方案：完全免費
   - Pro 方案：$20/月，固定價格
   - 適合所有查詢量範圍

2. **AWS Route53 / Google Cloud DNS 次之**：
   - 5 億次/月 + 50個域名：$208.50-216/月
   - 無前期建置成本
   - 第一年比雲端自建便宜約 70-72%（考慮前期建置和50個域名後）
   - 後續年度需視查詢量而定

3. **雲端自建（需考慮前期建置）**：
   - 前期建置：$2,900（一次性，台灣人力）
   - 月運營：$750/月
   - 第一年總成本：$11,900（不含專職人力）
   - 後續年度：$9,000/年（不含專職人力，固定成本優勢）
   - 含專職人力：後續年度 $33,000/年

4. **企業級服務（NS1/Akamai）**：
   - NS1：$5,500/月
   - Akamai：$8,500/月
   - 適合需要高級功能的大型企業

### 8.2 SLA 結論

1. **第三方服務提供 100% SLA**：
   - Cloudflare Pro/Business
   - AWS Route53
   - Google Cloud DNS
   - NS1
   - Akamai Edge DNS

2. **自建方案無 SLA 保證**：
   - 自己買機器建立：取決於架構和團隊能力
   - 雲端自建：取決於架構設計

### 8.3 DDoS 防護結論

1. **最強防護**：
   - Akamai Edge DNS（Prolexic 技術，1.35+ Tbps）
   - Cloudflare（無上限防護）

2. **企業級防護**：
   - NS1（內建防護）
   - AWS Shield Advanced（需額外費用）

3. **基礎防護**：
   - AWS Shield Standard（Route53 內建，免費）
   - Google Cloud DNS（內建基礎防護）
   - 雲端自建 + 基礎防護

### 8.4 最終建議

#### 對於 20 億次/月（峰值 20k QPS）環境：

1. **首選**：**Cloudflare Pro**（$20/月）
   - 成本最低（無前期建置，月費固定）
   - 強大 DDoS 防護
   - 100% SLA
   - 簡單易用

2. **次選**：**AWS Route53 / Google Cloud DNS**（$208.50-216/月，含50個域名）
   - 無前期建置成本
   - 第一年比雲端自建貴約 23-24%（考慮前期建置成本後）
   - 適合需要快速部署、無維護負擔、多域名管理的場景

3. **可選**：**雲端自建**（前期 $2,900，月運營 $750）
   - 完全控制權
   - 後續年度固定成本優勢（$9,000/年，不含專職人力）
   - 適合需要完全控制、長期使用的場景

4. **不推薦**：
   - NS1 / Akamai（費用較高，$3k-12k/月，適合大型企業）
   - 自己買機器建立（初始投資高 $35.8k-88.8k，維護負擔重）

#### 對於低查詢量環境（<10 億次/月）：

1. **首選**：**Cloudflare Free/Pro**
   - Free：完全免費
   - Pro：$20/月，有 SLA

2. **次選**：**第三方服務**（AWS/GCP）
   - 成本隨查詢量線性增長
   - 無需維護負擔
   - 無前期建置成本

#### 對於企業級需求：

1. **需要高級流量管理**：**NS1**
2. **需要最強安全防護**：**Akamai Edge DNS**
3. **AWS/GCP 生態系統用戶**：**AWS Route53 / Google Cloud DNS**
