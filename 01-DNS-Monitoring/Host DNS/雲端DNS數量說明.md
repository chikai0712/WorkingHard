# 為什麼需要四台 DNS VM？雲端 vs 自建差異

## 一、自建環境需要四台的原因

### 1. 高可用性（HA）設計
- **目標**：任一實體主機故障時，服務仍可正常運作
- **分配策略**：
  - 實體主機 #1：2 台 DNS VM
  - 實體主機 #2：2 台 DNS VM
  - 實體主機 #3：備用（可跑 1 台 DNS VM 或關機）
- **結果**：任何 1 台實體主機故障，仍有 2-3 台 DNS VM 運作

### 2. 容量冗餘
- **正常運作**：4 台 × 10k QPS = **40k QPS 能力**
- **目標負載**：20k QPS 峰值
- **安全邊際**：2x 冗餘（40k / 20k = 2倍）

### 3. 故障容錯
- **故障場景**：1 台實體主機故障 → 剩 2-3 台 DNS VM
- **剩餘能力**：20k-30k QPS
- **結論**：仍足夠應付 20k QPS 峰值

---

## 二、雲端環境可以減少數量

### 為什麼雲端不需要四台？

#### 1. 更高的基礎設施可用性
- **AWS / GCP 可用區（AZ）**：99.99% 可用性保證
- **自動恢復**：VM 故障時可自動重啟或遷移
- **多可用區部署**：跨多個 AZ 分散風險

#### 2. 更靈活的擴縮容
- **自動擴縮**：可根據負載自動增加/減少 VM
- **快速啟動**：新 VM 可在數分鐘內啟動
- **按需付費**：不需要預先準備過多資源

#### 3. 更智能的負載均衡
- **AWS NLB / GCP TCP LB**：自動健康檢查與故障轉移
- **跨區域負載分散**：可分散到多個區域
- **智能路由**：自動避開故障節點

---

## 三、雲端建議方案

### 方案 A：最小配置（2 台）⭐ 推薦
- **配置**：2 台 DNS VM，跨 2 個可用區
- **容量**：2 × 10k QPS = 20k QPS（剛好滿足需求）
- **可用性**：1 台故障時，剩 1 台（10k QPS，可能降級）
- **成本**：約 **$560-600 / 月**（AWS/GCP）
- **適用**：預算有限，可接受短暫降級

### 方案 B：標準配置（3 台）⭐⭐ 最推薦
- **配置**：3 台 DNS VM，跨 3 個可用區
- **容量**：3 × 10k QPS = 30k QPS（1.5x 冗餘）
- **可用性**：1 台故障時，剩 2 台（20k QPS，仍可正常運作）
- **成本**：約 **$840-900 / 月**（AWS/GCP）
- **適用**：平衡成本與可用性

### 方案 C：高可用配置（4 台）
- **配置**：4 台 DNS VM，跨 3-4 個可用區
- **容量**：4 × 10k QPS = 40k QPS（2x 冗餘）
- **可用性**：1 台故障時，剩 3 台（30k QPS，完全無影響）
- **成本**：約 **$1,120-1,200 / 月**（AWS/GCP）
- **適用**：高可用性要求，預算充足

---

## 四、方案對比表

| 項目 | 方案 A（2台） | 方案 B（3台）⭐ | 方案 C（4台） |
|------|-------------|---------------|-------------|
| **DNS VM 數量** | 2 | 3 | 4 |
| **正常容量** | 20k QPS | 30k QPS | 40k QPS |
| **故障後容量** | 10k QPS（降級） | 20k QPS（正常） | 30k QPS（正常） |
| **可用區** | 2 個 | 3 個 | 3-4 個 |
| **AWS 月費** | ~$560 | ~$840 | ~$1,120 |
| **GCP 月費** | ~$600 | ~$900 | ~$1,200 |
| **優點** | 成本最低 | 平衡成本與可用性 | 最高可用性 |
| **缺點** | 故障時可能降級 | 無明顯缺點 | 成本較高 |

---

## 五、建議

### 對於 20k QPS 的中型環境

#### 推薦：方案 B（3 台 DNS VM）
- ✅ **成本合理**：比 4 台省約 25% 成本
- ✅ **可用性足夠**：1 台故障時仍有 20k QPS 能力
- ✅ **符合需求**：30k QPS 容量足夠應付峰值
- ✅ **符合雲端最佳實踐**：跨 3 個可用區部署

#### 不推薦：方案 A（2 台）
- ❌ 故障時容量只剩 10k QPS，可能無法應付 20k QPS 峰值
- ❌ 不符合高可用性要求

#### 可選：方案 C（4 台）
- ⚠️ 如果預算充足且要求極高可用性，可考慮
- ⚠️ 但對於 20k QPS 來說，4 台可能過度配置

## 六、更新後的費用估算

### 方案 B（3 台 DNS VM）⭐ 推薦

#### AWS 總月費：約 $1,010 / 月
- DNS VM（3台）：$0.384 × 730 × 3 = **$841**
- 監控 VM（1台）：$140
- Bastion VM（1台）：$30
- 磁碟費用：$108
- **總計**：約 **$1,010 / 月**

#### GCP 總月費：約 $1,080 / 月
- DNS VM（3台）：$0.379 × 730 × 3 = **$830**
- 監控 VM（1台）：$138
- Bastion VM（1台）：$34
- 磁碟費用：$230
- **總計**：約 **$1,080 / 月**

#### 含折扣後（1 年 Reserved / 承諾使用）
- **AWS**：約 **$600-700 / 月**（省 30-40%）
- **GCP**：約 **$650-750 / 月**（省 30-40%）

## 七、結論

**原本四台 DNS VM 的設計是針對自建環境**，因為：
- 實體主機故障風險較高
- 需要預先準備足夠冗餘
- 擴縮容較困難

**在雲端環境中，建議改為 3 台 DNS VM**，因為：
- 雲端基礎設施可用性更高
- 可快速擴縮容
- 負載均衡更智能
- 成本更合理

**如果預算有限，2 台也可以考慮**，但需接受故障時可能降級的風險。
