# Host DNS Resolver 地端部署設備規格清單

## 需求分析

### 1. 性能需求計算

**目標：同時承載 10,000 人查詢**

假設條件：
- 每人平均查詢頻率：1 QPS（每秒 1 次查詢）
- 峰值倍數：3x（考慮突發流量）
- 總 QPS 需求：10,000 × 3 = **30,000 QPS**

**實際建議配置：**
- 正常負載：30,000 QPS
- 峰值負載：50,000 QPS（安全邊際）
- 單節點處理能力：10,000-15,000 QPS

## 3. 設備清單與規格

### 3.1 DNS Resolver 節點（主服務器）

**數量：3 台（主站點 2 台 + 備援站點 1 台）**

#### 規格要求：

| 組件 | 規格 | 說明 |
|------|------|------|
| **CPU** | Intel Xeon E-2388G 或同等級<br/>8 核心 / 16 執行緒 | 支援高併發 DNS 查詢 |
| **記憶體** | 32GB DDR4 ECC | 足夠的快取空間和併發處理 |
| **儲存** | 500GB NVMe SSD<br/>（SATA SSD 備選） | 作業系統和日誌存儲 |
| **網路** | 2 × 10GbE 網卡<br/>（支援 Bonding） | 高頻寬、冗餘網路 |
| **電源** | 雙電源供應器（1+1 備援） | 高可用性 |
| **機架** | 1U 機架式伺服器 | 節省空間 |

**推薦型號：**
- Dell PowerEdge R650
- HPE ProLiant DL325 Gen10 Plus
- Supermicro 1U Server

**單台處理能力：**
- 正常負載：10,000-15,000 QPS
- 峰值負載：20,000 QPS
- 3 台總計：30,000-45,000 QPS（滿足需求）

## 10. 高可用性設計

### 10.1 故障場景與應對

| 故障場景 | 影響 | 應對措施 |
|---------|------|---------|
| **單台 DNS 節點故障** | 無影響 | HAProxy 自動切換到其他節點 |
| **主站點全部故障** | 服務中斷 | 自動切換到備援站點（DNS3） |
| **負載均衡器故障** | 無影響 | Keepalived 自動切換 VIP |
| **網路線路故障** | 無影響 | 備援線路自動啟用 |
| **電源故障** | 無影響 | UPS 供電，發電機備援 |

### 10.2 備援策略

1. **DNS 節點備援**：主站點 2 台 + 備援站點 1 台
2. **負載均衡器備援**：主備模式（VRRP）
3. **網路備援**：雙線路、Bonding
4. **電源備援**：UPS + 發電機
5. **數據備份**：每日自動備份配置和日誌
