# 資料來源 API 說明

#### 1. ✅ 融資融券資料 - TWSE (台灣證券交易所)

#### 2. ⚠️ 三大法人買賣超 - TAIFEX (台灣期貨交易所)

**API 來源**: `https://www.taifex.com.tw/cht/3/futDataDown?down_type=2`

**參數**:
- `down_type`: 2（表示三大法人買賣超）
- `queryStartDate`: 開始日期
- `queryEndDate`: 結束日期

**包含資料**:
- 外資買賣超（買進、賣出、淨買賣超）
- 投信買賣超（買進、賣出、淨買賣超）
- 自營商買賣超（總、自買、避險）
- 三大法人合計

**狀態**: ⚠️ **API 端點已知，但回傳 HTML 而非 CSV，需要實現網頁爬蟲或解析 HTML**

## ⚠️ TAIFEX API 問題

### 發現的問題

1. **API 回傳 HTML 而非 CSV**
   - 測試結果顯示 TAIFEX API 回傳 `text/html` 而非 CSV
   - 可能需要先訪問網頁，再透過 JavaScript 下載 CSV

2. **可能需要 Session**
   - 可能需要先建立 Session 或 Cookie
   - 或需要模擬瀏覽器行為

3. **需要使用網頁爬蟲**
   - 從 HTML 頁面中提取資料
   - 或解析 JavaScript 生成的內容

---

## 🔧 解決方案

### 方案 1: 使用網頁爬蟲（推薦）

**優點**:
- 可以處理 HTML 回應
- 可以模擬瀏覽器行為
- 靈活性高

**實現方式**:
- 使用 `BeautifulSoup` 解析 HTML
- 使用 `Selenium` 處理 JavaScript（如需要）
- 從網頁中提取表格資料

**檔案**: `backend/crawler/taifex_web_scraper.py`

### 方案 2: 尋找正確的 API 端點

**優點**:
- 如果找到直接 API，效率更高
- 資料格式可能更穩定

**需要**:
- 查閱 TAIFEX 官方文件
- 測試不同的參數組合
- 確認是否有其他端點

### 方案 3: 使用瀏覽器自動化

**優點**:
- 可以處理複雜的 JavaScript
- 可以模擬真實使用者行為

**缺點**:
- 效能較低
- 需要額外的依賴（如 Selenium）

---
