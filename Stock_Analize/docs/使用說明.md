# 使用說明

## 🚀 快速開始

### 1. 環境準備

確保已安裝：
- Python 3.10 或更高版本
- Node.js 16 或更高版本
- npm 或 yarn

### 2. 後端設置

```bash
# 進入後端目錄
cd Stock_Analize/backend

# 建立虛擬環境（建議）
python3 -m venv venv

# 啟動虛擬環境
# macOS/Linux:
source venv/bin/activate
# Windows:
# venv\Scripts\activate

# 安裝依賴
pip install -r requirements.txt

# 複製環境變數檔案
cp .env.example .env

# 編輯 .env 檔案（可選，預設值已足夠開發使用）
# nano .env
```

### 3. 前端設置

```bash
# 進入前端目錄
cd Stock_Analize/frontend

# 安裝依賴
npm install

# 複製環境變數檔案
cp .env.example .env

# 編輯 .env 檔案（確認 API 基礎 URL）
# VITE_API_BASE_URL=http://localhost:8000/api
```

### 4. 啟動服務

#### 啟動後端（終端 1）

```bash
cd Stock_Analize/backend

# 啟動 FastAPI 服務
python main.py

# 或使用 uvicorn 直接啟動
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

後端服務將在 `http://localhost:8000` 啟動。

您可以在瀏覽器開啟 `http://localhost:8000/docs` 查看 API 文件。

#### 啟動前端（終端 2）

```bash
cd Stock_Analize/frontend

# 啟動開發伺服器
npm run dev
```

前端服務將在 `http://localhost:5173` 啟動。

### 5. 訪問應用程式

開啟瀏覽器，訪問 `http://localhost:5173` 即可看到股票資訊儀表板。

## 📊 新增股票

### 方法一：透過 API

```bash
curl -X POST "http://localhost:8000/api/stocks" \
  -H "Content-Type: application/json" \
  -d '{
    "symbol": "2330",
    "market": "TW",
    "name": "台積電",
    "industry": "半導體"
  }'
```

### 方法二：透過 API 文件頁面

1. 訪問 `http://localhost:8000/docs`
2. 找到 `POST /api/stocks` 端點
3. 點擊 "Try it out"
4. 輸入股票資訊
5. 點擊 "Execute"

### 方法三：直接使用資料庫（進階）

```python
from database.session import SessionLocal
from services.stock_service import StockService

db = SessionLocal()
stock_service = StockService(db)

stock = stock_service.create_stock(
    symbol="2317",
    market="TW",
    name="鴻海",
    industry="電子製造"
)

db.close()
```

## 🔄 資料更新

系統會自動在以下時間更新資料：

1. **開盤前更新**：每天 08:50（週一至週五）
2. **交易時間更新**：每 5 分鐘（可在 `.env` 設定 `UPDATE_INTERVAL_MINUTES`）
3. **收盤後更新**：每天 15:30（週一至週五）

### 手動觸發更新

```bash
# 使用 Python 腳本
cd Stock_Analize/backend
python -c "
from database.session import SessionLocal
from crawler.stock_crawler import StockCrawler

db = SessionLocal()
crawler = StockCrawler(db)
results = crawler.update_all_stocks()
print(results)
db.close()
"
```

## 🎯 功能說明

### 儀表板頁面

- **總覽統計**：顯示總股票數、總市值、總漲跌、上漲家數
- **股票列表**：顯示所有監控股票的即時資訊
- **自動更新**：每 30 秒自動刷新資料（可在 `.env` 設定）

### 股票詳情頁面

- **價格資訊**：顯示最新價、開盤、最高、最低、收盤
- **漲跌資訊**：顯示漲跌金額和百分比
- **歷史價格**：顯示最近 30 筆歷史價格資料
- **技術指標**：未來版本將加入技術指標顯示

## 🔧 配置說明

### 後端配置（`.env`）

```env
# 資料庫（SQLite，預設）
DATABASE_URL=sqlite:///./stock_data.db

# 更新間隔（分鐘）
UPDATE_INTERVAL_MINUTES=5

# 僅在交易時間更新
TRADING_HOURS_ONLY=True

# 台股交易時間
TAIWAN_MARKET_OPEN=09:00
TAIWAN_MARKET_CLOSE=13:30
```

### 前端配置（`.env`）

```env
# API 基礎 URL
VITE_API_BASE_URL=http://localhost:8000/api

# 自動更新間隔（秒）
VITE_UPDATE_INTERVAL_SECONDS=30
```

## 🐛 常見問題

### 1. 後端啟動失敗

**問題**：`ModuleNotFoundError: No module named 'fastapi'`

**解決**：確認已安裝所有依賴
```bash
cd backend
pip install -r requirements.txt
```

### 2. 前端無法連接到後端

**問題**：前端顯示「無法連接」或 CORS 錯誤

**解決**：
1. 確認後端服務正在運行（訪問 `http://localhost:8000/health`）
2. 檢查前端 `.env` 中的 `VITE_API_BASE_URL` 是否正確
3. 確認後端 `.env` 中的 `CORS_ORIGINS` 包含前端 URL

### 3. 無法取得股票資料

**問題**：股票列表為空或顯示錯誤

**解決**：
1. 確認已新增股票（參考「新增股票」章節）
2. 檢查網路連線（yfinance 需要網路連線）
3. 確認股票代號正確（台股需要正確代號，如 "2330"）

### 4. 資料沒有自動更新

**問題**：定時任務沒有執行

**解決**：
1. 確認 `.env` 中 `SCHEDULER_ENABLED=True`
2. 檢查後端日誌是否有錯誤訊息
3. 確認系統時間正確
