# 手動更新資料指南

## 🚀 快速更新

### 方法一：使用更新腳本（推薦）

#### macOS/Linux
```bash
./update.sh
```

#### Windows
```cmd
update.bat
```

### 方法二：直接執行 Python 腳本

```bash
cd backend
source venv/bin/activate  # Linux/Mac
# 或
venv\Scripts\activate     # Windows

python scripts/update_all.py
```

---

## 📋 更新腳本說明

### 1. 統合更新（推薦）

**腳本**: `backend/scripts/update_all.py`  
**功能**: 同時更新股票和期貨資料  
**執行**:
```bash
cd backend
python scripts/update_all.py
```

**更新內容**:
- ✅ 所有股票的價格資料
- ✅ 今天的期貨相關資料（融資融券、三大法人、期貨每日資料）

---

### 2. 僅更新股票資料

**腳本**: `backend/scripts/manual_update.py`  
**功能**: 只更新股票價格資料  
**執行**:
```bash
cd backend
python scripts/manual_update.py
```

---

### 3. 更新期貨資料（今天）

**腳本**: `backend/scripts/fetch_futures_data.py`  
**功能**: 抓取今天的期貨資料  
**執行**:
```bash
cd backend
python scripts/fetch_futures_data.py
```

**或指定日期**:
```bash
python scripts/fetch_futures_data.py 2025-12-22
```

---

### 4. 更新期貨資料（最近 N 天）

**腳本**: `backend/scripts/fetch_recent_data.py`  
**功能**: 抓取最近 7 天的期貨資料（補齊歷史資料）  
**執行**:
```bash
cd backend
python scripts/fetch_recent_data.py
```

**說明**:
- 預設抓取最近 7 天
- 可修改腳本中的 `days = 7` 來調整天數
- 用於補齊歷史資料或測試

## ⚠️ 常見問題

### 問題 1: 股票資料更新顯示「總數：0」

**原因**: 資料庫中沒有股票

**解決方法**:
```bash
cd backend
python scripts/add_stocks.py
```

或在 API 中添加股票：
```bash
curl -X POST "http://localhost:8000/api/stocks" \
  -H "Content-Type: application/json" \
  -d '{"symbol": "2330", "market": "TW", "name": "台積電"}'
```

---

### 問題 2: 期貨資料更新失敗

**可能原因**:
1. 非交易日（週末或假日）
2. 資料來源 API 暫時無法訪問
3. 收盤前執行（資料尚未發布）

**解決方法**:
- 等待收盤後（15:30 後）再執行
- 檢查網路連線
- 查看錯誤訊息了解詳細原因

---

### 問題 3: 找不到虛擬環境

**錯誤訊息**:
```
❌ 錯誤: 未找到虛擬環境，請先執行 ./start.sh
```

**解決方法**:
```bash
# 啟動服務（會自動建立虛擬環境）
./start.sh

# 或手動建立
cd backend
python3 -m venv venv
source venv/bin/activate  # Linux/Mac
pip install -r requirements.txt
```

## 📝 更新日誌位置

更新過程的日誌會顯示在終端機，如果需要保存：

```bash
# 保存更新日誌
./update.sh > logs/update_$(date +%Y%m%d_%H%M%S).log 2>&1
```

---

## 🔄 與定時任務的關係

- **手動更新**: 可隨時執行，不受定時任務限制
- **定時任務**: 自動在設定的時間執行（需 `SCHEDULER_ENABLED=True`）
- **兩者互不影響**: 手動更新不會影響定時任務，反之亦然

---

## 💡 使用建議

1. **日常使用**: 讓定時任務自動更新即可
2. **測試或補資料**: 使用手動更新腳本
3. **補歷史資料**: 使用 `fetch_recent_data.py`
4. **檢查資料**: 使用 `update_all.py` 快速檢查所有資料是否正常
