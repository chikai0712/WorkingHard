# 自動重試版腳本使用指南

## 📦 新版本特點

**文件名**: `id_globalping_auto_retry.sh`

### ✨ 主要改進

1. **一次性完成** - 無需手動分段操作
2. **自動重試** - 檢測結束後自動重試失敗域名（最多 2 輪）
3. **優化延遲** - 增加延遲時間，大幅降低 API 限制觸發率
4. **智能批次** - 每 40 個域名休息 45 秒
5. **實時進度** - 顯示當前進度和狀態

---

## 🚀 快速使用

### 1. 添加執行權限

```bash
chmod +x ~/Desktop/Project/id_globalping_auto_retry.sh
```

### 2. 運行檢測

```bash
# 使用你的域名文件
~/Desktop/Project/id_globalping_auto_retry.sh ~/domains.txt
```

### 3. 等待完成

腳本會自動：
- ✅ 檢測所有域名
- ✅ 自動重試失敗的域名（最多 2 輪）
- ✅ 生成詳細日誌
- ✅ 顯示最終統計

---

## ⚙️ 配置說明

腳本內置的優化配置：

```bash
DELAY=5              # 每個域名間隔 5 秒（比舊版多 3 秒）
API_ERROR_DELAY=20   # API 錯誤後延遲 20 秒（比舊版多 18 秒）
BATCH_SIZE=40        # 每批 40 個（比舊版少 10 個）
BATCH_DELAY=45       # 批次間休息 45 秒（比舊版多 15 秒）
MAX_RETRY_ROUNDS=2   # 最多重試 2 輪
```

### 如果仍然遇到 API 限制

編輯腳本，調整這些參數：

```bash
nano ~/Desktop/Project/id_globalping_auto_retry.sh

# 修改為更保守的值：
DELAY=8              # 增加到 8 秒
BATCH_SIZE=30        # 減少到 30 個
BATCH_DELAY=60       # 增加到 60 秒
```

```
  ├─ 休息 60 秒後開始
  └─ 最後一次嘗試
  ↓

  ↓
完成
```

## ⏱️ 預估時間

### 計算公式

```
總時間 = (域名數 × 延遲) + (批次數 × 批次延遲) + (重試輪數 × 失敗域名數 × 延遲)
```

### 實際例子（103 個域名）

**第一輪**：
- 103 個域名 × 5 秒 = 515 秒（8.5 分鐘）
- 3 個批次 × 45 秒 = 135 秒（2.3 分鐘）
- **小計**: 約 11 分鐘

**假設 20 個失敗，重試 2 輪**：
- 第 1 輪重試: 20 × 5 秒 + 60 秒休息 = 160 秒（2.7 分鐘）
- 第 2 輪重試: 假設 5 個仍失敗 = 5 × 5 秒 + 60 秒 = 85 秒（1.4 分鐘）
- **小計**: 約 4 分鐘

**總計**: 約 **15-20 分鐘**（取決於失敗數量）

## 🔧 故障排除

### 問題 1: 仍然有很多 API 錯誤

**解決方案**：增加延遲時間

```bash
# 編輯腳本
nano ~/Desktop/Project/id_globalping_auto_retry.sh

# 修改配置
DELAY=8              # 增加到 8 秒
BATCH_SIZE=25        # 減少到 25 個
BATCH_DELAY=60       # 增加到 60 秒
```

### 問題 2: 執行時間太長

**解決方案**：減少重試輪數或分批檢測

```bash
# 修改重試輪數
MAX_RETRY_ROUNDS=1   # 只重試 1 輪

# 或者分批檢測
split -l 50 ~/domains.txt ~/domains_part_
~/Desktop/Project/id_globalping_auto_retry.sh ~/domains_part_aa
~/Desktop/Project/id_globalping_auto_retry.sh ~/domains_part_ab
```

### 問題 3: Python 錯誤

**解決方案**：確認 Python 3 已安裝

```bash
# 檢查 Python
which python3

# 如果沒有，安裝
brew install python3  # macOS
```
